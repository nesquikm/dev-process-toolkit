# Bun (TypeScript)

Stack-specific guidance for projects scaffolded with `bun init` against TypeScript. Use this in conjunction with the generic TypeScript guidance in `examples/typescript-node/`.

## Project layout

Bun's default scaffold (toolkit policy: `src/`-co-located tests, see `docs/patterns.md` § Test Layout Policy):

```
bun.lock                 # lockfile (replaces package-lock.json)
package.json             # `"type": "module"` recommended
tsconfig.json            # strict, ESM, `module: ESNext`
src/
├── index.ts
├── index.test.ts        # co-located unit test (toolkit default)
└── .placeholder.test.ts # zero-match shield until first real test lands (see below)
```

`bun.lock` presence is the canonical Bun-stack marker. `/setup` detects Bun via `bun.lock` OR `package.json` with `"packageManager": "bun@..."` OR `bunfig.toml`.

## Gate command

```bash
bunx tsc --noEmit && bun test
```

Use this exact form in `CLAUDE.md` § Gating rule. `bunx tsc --noEmit` runs the type checker without emit; `bun test` runs the test suite. Both must succeed for the gate to pass.

## Bun zero-match-exit-1 workaround

Bun's `bun test` has **no `--passWithNoTests` equivalent** — when invoked against a directory with no test files, it exits 1 ("no tests found"). On a fresh project just scaffolded by `/setup`, the very first gate-check would fail.

`/setup` step 2c writes a single placeholder file the first time the gate runs, with a marker comment so future readers (and `/gate-check` probe `bun-zero-match-placeholder`) can recognize it as load-bearing scaffolding rather than abandoned dead code:

```typescript
// generated by /dev-process-toolkit:setup — Bun zero-match workaround (see examples/bun-typescript.md)
import { test, expect } from "bun:test";

test("placeholder — delete once the project ships its first real test", () => {
  expect(1 + 1).toBe(2);
});
```

**File path:** `src/.placeholder.test.ts` (toolkit default, co-located). Override to `tests/.placeholder.test.ts` only if the project elected `tests/-mirror` in CLAUDE.md `## Testing Conventions` § Layout. The leading dot keeps it sorted at the top of `ls` output, signalling its scaffolding role.

**When to delete:** the moment a real co-located test file lands in `src/` (or a real `tests/<x>.test.ts` for projects on the mirror layout). The `bun-zero-match-placeholder` probe is vacuous in that case and won't flag the deletion.

**When NOT to delete:** if the project still has zero real tests. Removing the placeholder before a real test exists will break the very next `/gate-check`.

The probe is scoped to Bun projects only (`bun.lock` presence). Non-Bun projects skip the check entirely.

## Permissions allowlist

The canonical Bun-stack `Bash()` permissions live in `templates/permissions.json` under the `bun` stack key. `/setup` merges these into the project's `.claude/settings.json` `permissions.allow`:

```json
{
  "permissions": {
    "allow": [
      "Bash(bun *)",
      "Bash(bunx *)",
      "Bash(git *)",
      "Bash(gh *)",
      "Bash(mkdir *)",
      "Bash(ls *)",
      "Bash(rm *)",
      "Bash(mv *)",
      "Bash(cp *)"
    ]
  }
}
```

User additions (project-specific scripts, `bun run db:migrate`, etc.) are preserved across re-runs — the merge logic in `adapters/_shared/src/setup/merge_settings.ts` dedups the canonical set against the existing array without stripping non-canonical entries.

## TDD patterns

- **Test runner:** `bun test`. APIs (`describe`, `test`, `expect`) auto-import from `"bun:test"`.
- **Mocking:** `mock(...)` and `spyOn(...)` from `bun:test` — closer to Vitest than Jest.
- **Fake timers:** `Bun.sleep()` is real-time; for deterministic time, inject a clock argument and stub it in tests.
- **Schema validation:** Zod or TypeBox both work; both ship ESM-first.

## Common gotchas

- **`bun install` writes to `bun.lock`, not `package-lock.json`.** Don't commit both — pick one.
- **`tsconfig.json` `"module": "ESNext"` + `"moduleResolution": "Bundler"`** is the most permissive combination. Stricter settings work but require explicit extension imports.
- **`bun test` in CI:** add `--bail` (stops on first failure) and `--reporter=junit` (for CI test reporting integrations).
