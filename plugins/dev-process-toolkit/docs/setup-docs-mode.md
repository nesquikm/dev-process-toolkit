# `/setup` Docs Mode Flow (STE-68)

Companion doc for `skills/setup/SKILL.md` Step 7d. Covers the three yes/no
prompts that configure per-project docs generation and the `## Docs`
section `/setup` writes to `CLAUDE.md`. Pointed at from the skill to keep
`SKILL.md` under the NFR-1 300-line budget.

Applies whenever `/setup` runs, regardless of tracker mode. `mode: none`
projects still get the prompts — docs generation is orthogonal to task
tracking. Absent `## Docs` section ≡ all three values `false` (AC-STE-68.3),
which is the backward-compatible default for projects whose CLAUDE.md
predates the docs section.

## Core contract (read first)

- **Three binary prompts.** No mode strings, no free-form. Case-insensitive
  `y`/`yes`/`n`/`no`; other inputs re-prompt with the remedy literal
  `answer y or n`.
- **Absent section ≡ all-false.** No migration prompt on upgrade; existing
  projects continue to work with `/docs` as a no-op until they re-run
  `/setup` (AC-STE-68.3).
- **Both-false is a hard refusal.** If the user declines both
  `user_facing_mode` and `packages_mode`, surface the NFR-10 refusal
  below and re-ask those two prompts only. `changelog_ci_owned` from the
  first pass is preserved across the re-ask (AC-STE-68.4).
- **Atomic re-run writes.** Re-running `/setup` on a project that already
  has a `## Docs` section shows current values inline and overwrites the
  full block in a single file write — no per-line edits (AC-STE-68.5).

## The three prompts

Ask in this exact order, after Step 7c (branch template):

### Prompt 1 — user-facing docs

**First-run:**

> Generate user-facing docs (narrative + mermaid state/flow diagrams)? (y/n)

**Re-run (existing `## Docs` section):**

> Generate user-facing docs? [current: true] (y/n, empty = keep)

Empty input on re-run keeps the current value. Bad input re-prompts with
literal `answer y or n`.

### Prompt 2 — packages-style docs

**First-run:**

> Generate packages-style API reference docs? (typedoc <detected|not found>, ts-morph <bundled>, stack: <ts|other>) (y/n)

**Re-run:**

> Generate packages-style API reference docs? (typedoc <detected|not found>, ts-morph <bundled>, stack: <ts|other>) [current: false] (y/n, empty = keep)

The parenthetical is a signature-extraction probe result (STE-72 AC-STE-72.6). `typedoc` resolves via `<projectRoot>/node_modules/.bin/typedoc` + `PATH`; `ts-morph` is always `bundled` (it ships with `adapters/_shared/package.json`); `stack` is `ts` when `tsconfig.json` exists, otherwise `other`. The probe result is informational — it does not affect the prompt's default or the accepted inputs. It gives the user immediate visibility into which strategy `/docs` will choose at generation time (AC-STE-72.2).

### Prompt 3 — CHANGELOG.md ownership

**First-run:**

> Is CHANGELOG.md generated by CI (if yes, /ship-milestone will not write it)? (y/n)

**Re-run:**

> Is CHANGELOG.md generated by CI? [current: false] (y/n, empty = keep)

## Both-false refusal (AC-STE-68.4)

If the user answers `no` to both prompts 1 and 2, render this NFR-10
canonical-shape error and re-ask prompts 1+2 only (prompt 3's answer is
preserved):

```
/setup: at least one docs mode must be enabled to write the ## Docs section.
Remedy: answer yes to either "user-facing docs?" or "packages API refs?", or decline both to skip docs configuration entirely (the ## Docs section will not be written and /docs will be a no-op).
Context: mode=<tracker-mode>, skill=setup
```

If the user chooses "decline both" on the re-ask, do not write the
`## Docs` section at all. The `mode=<tracker-mode>` field is the active
tracker mode (`none`, `linear`, `jira`, or custom name).

## Writing the section

Place the section immediately after `## Task Tracking` when that section
exists; otherwise append at end of file. Keep a single blank line between
the heading and the first key. Values are lowercase literal `true` /
`false` — no quoting, no `yes`/`no`, no blank lines between keys.

```
## Docs

user_facing_mode: true
packages_mode: false
changelog_ci_owned: true
```

Atomic write: read CLAUDE.md, splice the full `## Docs` block (replacing
any existing one in its entirety), write once. Never partial-line edit
an existing block.

## Reader helper

`adapters/_shared/src/docs_config.ts` exports
`readDocsConfig(claudeMdPath): DocsConfig` where:

```ts
interface DocsConfig {
  userFacingMode: boolean;
  packagesMode: boolean;
  changelogCiOwned: boolean;
}
```

Missing file or missing section ≡ `{ userFacingMode: false, packagesMode: false, changelogCiOwned: false }` (AC-STE-68.3). A key whose value is anything other than literal `true` / `false` throws `MalformedDocsConfigError`, which the calling skill re-renders as NFR-10 canonical shape.

Downstream skills (`/docs`, `/ship-milestone`, `/implement` Phase 4)
consume this helper — never grep the section inline. The helper owns the
parse contract.
