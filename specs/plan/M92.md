## M92: Per-skill token-usage stats {#M92}

**Goal:** Persist per-skill, per-model token usage across the SDD workflow into version-controlled FR + milestone specs — for **process-optimization** (which skill/phase is expensive) and **cost-awareness** (what a milestone cost to build). Grounded on the Claude Code transcript's per-line `attributionSkill` + `message.usage` + `message.model` (empirically verified 2026-07-01): per-skill accounting is a group-by `attributionSkill` × `model`, and `context:fork` child skills roll up into the parent skill automatically. The architecture **decouples capture from render**: a `SessionEnd`/`Stop` hook always appends per-skill tallies to a git-ignored `.dev-process/token-ledger.jsonl`; a sentinel-fenced `## Token Stats` block, projected from that ledger, is rendered into per-FR files + `specs/plan/M<N>.md` **only** when a spec file is already being written/committed. This consciously reverses the repo's prior "no token counts" stance — for measurement, not enforcement (no caps introduced).

**Prerequisites:** None new. Builds on the bundled-hook mechanism (`hooks/hooks.json`, M74), the stdin-transcript hook contract (`templates/hooks/_lib/session.ts`, STE-290), and M91's `## Design References` convention-test precedent.
