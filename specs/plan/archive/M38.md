# M38 — Branch ship-readiness {#M38}

**Release target:** v1.38.0 (provisional minor — clears 4 audit findings, two of which alter behavior the user can observe via `/spec-write`). Codename TBD; finalize at `/ship-milestone`.
**Status:** drafted 2026-04-28 from `/tmp/branch-review-2026-04-27.md` + `/tmp/comprehensive-branch-review-2026-04-27.md` (8 audit rounds, 35 verified findings, 5 consecutive zero-finding rounds).

## Why this milestone exists

The 24-release stretch on `feat/m12-tracker-integration` (v1.15.0 → v1.37.1) shipped 153 archived FRs and a structurally healthy toolkit. The 2026-04-27 audit walked seven different surface categories across two independent reviews and converged on a small set of release blockers.

Of the 35 findings, four are mandatory before merging the branch to `main`:

- **F-1** (STE-135) — `findFRByTrackerRef` always returns null in tracker mode. Silent data-loss path: `/spec-write <tracker-id>` on an existing tracker-mode FR re-imports from the tracker via `writeFileSync(path, body)` and silently overwrites local edits. Reproduced live against `STE-134`. The root cause is a structural inconsistency between the resolver (requires `id:`) and `/gate-check` probe #13 (forbids `id:` in tracker mode); STE-76 (M21) removed `id:` but the resolver was never updated. Cheapest fix is option (c): rewrite skill prose to skip the resolver in tracker mode, since the tracker ID is its own canonical identity.
- **F0** (STE-136) — repo's own `CLAUDE.md` is missing `## Docs` section. `/gate-check` probe #18 fires error severity on the dogfood repo. Hidden because the probe test only uses temp-dir fixtures (no repo-self-check). Two-edit fix.
- **F8-Tier-1 + G0** (STE-137) — shipped templates + examples leak this-repo's internal `STE-N` / `M<N>` namespace into surfaces that COPY into user projects. A user adopting the plugin into a Jira project gets a `CLAUDE.md` peppered with `STE-42`, `Refs: M7` examples they have no context for. G0 (broken `docs/patterns.md` link in `testing-spec.md.template:11`) is the same class of leak; bundle.
- **G20 + G21** (STE-138) — `CLAUDE.md` Release Checklist understates the file count by one (`specs/requirements.md` is the missing 5th) and points at a non-existent `## Structure` section in README (the actual section is `## What's Inside`). Release-process documentation drift: the checklist that exists to prevent release-day forgetfulness is itself silently wrong.

The four FRs are independent, cleanly bisectable, and total ~3-4h of estimated work.
