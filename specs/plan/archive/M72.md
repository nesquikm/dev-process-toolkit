# M72: STE-285 hooks E2E smoke coverage {#M72}

Release target: TBD
Codename: TBD

## Summary

E2E smoke coverage for STE-285 (M71) opt-in toolkit-contract enforcement hooks. STE-285 shipped 4 seeded Process-category hooks (`pre-commit-gate-check`, `pre-pr-spec-review`, `pre-spec-write-brainstorm-reminder`, `pre-commit-tdd-orchestrator`) with per-hook unit tests, but no end-to-end probe that exercises the full pipeline (settings.json install → harness hook dispatch → `$CLAUDE_SESSION_FILE` read → NFR-10 stderr refusal). STE-283's symmetric contract (TDD Orchestrator) already has runtime evidence via `/smoke-test` fixture group 7; STE-285 lacks the parallel probe. M72 closes that gap.

Single-FR milestone for now; M72's theme is STE-285 E2E coverage and future hook-related E2E FRs may extend it.

- [x] STE-286 — Smoke fixture group 8: STE-285 hooks runtime regression

## Risks

- **Risk:** Claude Code harness `$CLAUDE_SESSION_FILE` override behavior is implementation-internal — if the harness overrides the env var when spawning hooks, the smoke driver's env-injection scheme won't isolate scenarios. **Severity:** medium. **Mitigation:** STE-286 AC.7 explicitly requires empirical verification during `/implement` Phase 1 and accepts either the env-injection path OR a documented fallback (Bash wrapper / direct hook-script invocation).
- **Risk:** Fenced-block spawn description in `.claude/skills/smoke-test/SKILL.md` would trip the `auto_approve_marker_in_canonical_spawns` gate-check probe. **Severity:** medium. **Mitigation:** prose-only spawn description per STE-231 group-5b precedent (STE-226 lineage).
