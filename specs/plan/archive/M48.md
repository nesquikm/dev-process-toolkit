# M48 ŌĆö End-User Portability: /ship-milestone, /gate-check, /implement {#M48}

**Release target:** v2.1.0 (minor ŌĆö additive `## Release Files` schema in CLAUDE.md per NFR-2; new TS helper module; behavioral fix to two `/gate-check` probes and one `/implement` error message). Codename: TBD (finalize at `/ship-milestone`).
**Status:** drafted 2026-04-30.

## Why this milestone exists

A user testing the plugin in a non-toolkit project (2026-04-30) hit a fatal error from `/ship-milestone`: the skill expected `plugins/dev-process-toolkit/.claude-plugin/plugin.json`, `.claude-plugin/marketplace.json`, `CHANGELOG.md`, and a toolkit-shaped README "Latest:" line ŌĆö none of which exist outside the toolkit repo itself.

A 2026-04-30 audit of all 14 plugin skills (the 13 end-user skills plus a confirmation that `/smoke-test` is correctly maintainer-only and lives outside the plugin tree) identified three offenders:

- `/ship-milestone` ŌĆö hard-codes the toolkit's four release artifacts as if they were universal. Catastrophic in any non-toolkit project. (STE-167)
- `/gate-check` ŌĆö probes #9b (`plugin.json` version match) and #10 (toolkit template grep) crash on end-user projects where those paths don't exist. (STE-168)
- `/implement` ŌĆö line-265 chain-failure error message points the operator at a toolkit-internal path. (STE-169)

The other 11 skills are clean. `/smoke-test` is maintainer-only by design and excluded from this audit.

After M48 ships:

- `/ship-milestone` reads release-config from CLAUDE.md `## Release Files` block (populated by `/setup` from per-stack defaults under `examples/{typescript,flutter,python,plugin}/release.yml`).
- The toolkit dogfoods the new path via `examples/plugin/release.yml` ŌĆö self-hosting verifies the contract on the first ship through it.
- `/gate-check` probes #9b and #10 emit `n/a` rows in non-toolkit projects, not failures.
- `/implement` chain-failure error message no longer references toolkit-internal paths.
- The plugin is portable end-to-end for end-user projects in the supported stacks.
