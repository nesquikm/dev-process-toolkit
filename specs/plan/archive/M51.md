# M51 — Smoke Findings Cleanup (2026-05-01) {#M51}

**Release target:** v2.4.0 (minor — additive prose + new probes + one probe rename; no Release-Files schema change, no adapter contract break). Codename: TBD at `/ship-milestone` time.
**Status:** drafted 2026-05-01 from the Linear + Jira smoke runs against v2.3.0 (`/tmp/dpt-smoke-findings-2026-05-01-{linear,jira}.md`). Five FRs: three medium-severity plugin issues (STE-179, STE-180, STE-181) + two low-severity cosmetics (STE-182, STE-183).

## Why this milestone exists

The 2026-05-01 smoke runs validated the v2.3.0 canonical chain end-to-end on both Linear and Jira paths and surfaced three classes of issue:

1. **Toolkit-managed file drift** — `/setup`'s bootstrap commit covers CLAUDE.md and the three settings files but not the spec files written by `/spec-write`. The canonical chain ends with `??` `specs/frs/<id>.md`, `specs/technical-spec.md`, `specs/testing-spec.md`, `specs/plan/M<N>.md` in `git status`. Production users hit this on their first `/implement` after `/setup`. (STE-179)
2. **Single-FR vs milestone divergence** — `/implement <FR-id>` and `/implement M<N>` produce divergent end states by design (STE-83), but downstream messaging (smoke driver, `/spec-write`'s closing summary) disagrees on which form to recommend, and `/gate-check` probe #14 fires misleadingly when the single-FR form completes the milestone's last unchecked task. (STE-180, STE-181)
3. **Cosmetic prose drift** — `/spec-write`'s filename-policy-override row reads as if the user proposed a name when none was proposed; `/setup`'s bootstrap commit subject mentions the project's `bun init` version instead of the toolkit version. Both are cosmetic but cheap to fix. (STE-182, STE-183)

After M51 ships:

- `/spec-write` produces a `chore(specs): write FR <id>` commit on every successful run; canonical chain ends with no `??` toolkit files.
- Probe #22 widens to `toolkit-bootstrap-committed` and covers the spec-files scope.
- Probe #14 carves out the fully-checked-Done state and emits a "milestone ready to close" advisory.
- `/implement` SKILL.md gains an "Invocation forms" section documenting the `<FR-id>` vs `M<N>` divergence; `/spec-write` and the smoke driver agree on which form to recommend.
- `/spec-write`'s filename-policy-override row renders the right prose for both override-and-non-override paths.
- `/setup`'s bootstrap commit subject is clean; toolkit version lives in a `Toolkit:` footer.
