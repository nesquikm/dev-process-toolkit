# M77: /tdd spec-review audit step {#M77}

Release target: v2.24.0
Codename: TBD

## Summary

Single-FR milestone. STE-296 adds a fourth forked subagent (`tdd-spec-reviewer`) to the /tdd orchestrator that runs end-of-FR after REFACTOR, independently traces every AC to its implementation + test, blocks the pipeline only on `missing_acs` (binary contract violations), attempts a single bounded auto-retry through test-writer + implementer scoped to the missing ACs, and halts non-zero with a new `spec-gap` failure mode if the second audit still finds missing ACs. Advisory findings (`partial_acs`, `drift_count`, `cross_cutting_drift`) ride along in the report without halting. Closes the second-line-of-defense gap STE-225 deliberately left open.

## In Scope

- [x] STE-296 — Spec-review audit step in /tdd pipeline (block on Missing AC, bounded auto-retry)
  - verify: 10 ACs covering new subagent file, new child skill file, new parser + tests, orchestrator state-machine extension, retry budget, halt path with new `spec-gap` mode, gate-check probe, capability-key emission, and orchestrator integration test fixtures (paths A–F). `bun test plugins/dev-process-toolkit/tests/tdd-spec-review-*` PASSES.

## Out of Scope (deferred)

- Folding `/spec-review`'s standalone user-invocable surface into /tdd — out of scope; both remain.
- Extending the audit to /implement's Phase 3 Pass 1 (currently inline spec-compliance) — out of scope; /implement's own self-review pattern handles its scope.
- Making `drift_count` or `cross_cutting_drift` blocking — explicitly rejected in the brainstorm; ships advisory.
- Migration helper for existing /tdd usage — none needed per `project_no_users_yet`.

None. STE-225's multi-agent orchestrator is in production; STE-230's read-only fork pattern is in production. Extension only.
