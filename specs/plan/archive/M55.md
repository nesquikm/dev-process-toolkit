## Why this milestone exists

The 2026-05-04 smoke runs surfaced two recurring pattern failures that no prior milestone closed:

1. **`/spec-write` `-p` asymmetry** (B-F1): the commit gate already auto-applies under Auto mode / `-p`, but the upstream FR-draft acceptance gate has no symmetric default-apply path. Headless `claude -p /spec-write …` invocations stop mid-flow.
2. **Cross-cutting spec-drift propagation** (B-F3 / smoke #6 F4 / smoke #8 F5): when `/implement` removes a file, it updates the plan + traceability matrix but not `technical-spec.md` / `testing-spec.md`. 4× recurring.
3. **`/simplify` silent dirty tree** (smoke #6 F6 / #7 F7 / #8 F2 / #10 F1): 4× recurring; `/simplify` modifies tracked files but doesn't tell the operator the tree is dirty.

Plus three single-finding fixes that would otherwise stay as durable noise:

4. **`/gate-check` probe #26 false GATE FAILED** on Linear projects with zero milestones (smoke #9 F1) — `/spec-write` already declares the capability gap; probe #26 doesn't read it.
5. **`/gate-check` probe #28 negative-assertion false-positive** (smoke #10 F4) — verify lines like `returns "No such file or directory"` are deliberate negative assertions and shouldn't surface as stale-ref advisories.
6. **`## smoke-test runs` reference table cleanup** (user-requested during brainstorm) — the table accretes per-run noise without earning its keep; per-run findings files are the persistent audit trail. Inline the F-DR rationale at each reference site so the SKILL.md body stands alone.

After M55 ships:

- Headless `claude -p /spec-write …` invocations complete end-to-end without manual prompt-body pre-authorization for the draft gate; the smoke driver retires its B-F1 workaround.
- `/gate-check` no longer emits false GATE FAILED on Linear projects with zero milestones when `/spec-write` has declared the capability gap; every Linear smoke run produces clean PASSED-or-PASSED-WITH-NOTES outcomes.
- `/implement` propagates file-removal to `technical-spec.md` / `testing-spec.md` directory-tree blocks; the dev-process-toolkit's own accumulated drift (`.placeholder.test.ts` references at `technical-spec.md:13` + `testing-spec.md:18`) is also cleared as a fix-it-now sweep.
- `/simplify` emits a loud `⚠ tree dirty:` advisory + capability row when leaving the tree dirty; operators no longer silently inherit a dirty working tree.
- `/gate-check` probe #28 recognizes the four canonical negative-assertion shapes and skips the path-existence check on those lines; the M2 plan's stable false-positive disappears.
- `.claude/skills/smoke-test/SKILL.md` reads top-to-bottom without lookup; the F-DR codes and runs table are gone; future readers don't need an external reference table.
