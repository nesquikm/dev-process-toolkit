## M93: Project-authored verification skills {#M93}

**Goal:** Encourage & enable toolkit users to author project-local, post-`/implement` "check/drive" skills (modeled on the sibling `glacy-app-fe` skills `glacy-drive` / `glacy-progress-e2e`) and wire them into the flow. `/implement` verifies with a deterministic gate (compile/lint/tests) but never answers "does it actually run / look right in the app?" — projects need a bespoke check the toolkit can't ship generically, so the toolkit gives it a **place to plug in** (`## Verification` convention + a Phase 4b″ hook) plus a **blueprint** (a stack-aware generator + docs). Approved hybrid "Approach 1 UX + Approach 2 guts" (2026-07-01 `/brainstorm`): seamless `/implement`-native UX, but the failure-routing is native (proposes shipped `/brainstorm` / `/spec-write` / inline-fix) because `/conformance-loop` is maintainer-only and absent from a user's project. Realizes M91's deferred **Deliverable B** (visual conformance check).

**Prerequisites:** None new. Builds on `/implement`'s Phase 4 structure, `/setup`'s stack detector + CLAUDE.md generation, the shipped `/visual-check` (HG95TV), the `.claude/skills/<name>` + `disable-model-invocation: true` convention (STE-224), and the `/gate-check` `closing_summary_capability_keys` probe harness.
