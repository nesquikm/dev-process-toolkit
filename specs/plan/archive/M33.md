# M33 — Trace {#M33}

**Release target:** v1.35.0. Codename "Trace" — provisional; finalize at `/ship-milestone`.
**Status:** drafted 2026-04-27 from `/smoke-test` 2026-04-27 run #4 findings (`/tmp/dpt-smoke-findings-2026-04-27.md`) F1–F5.
**Origin:** smoke-test against v1.34.0 "Tripwires" reproduced five gaps across three categories. The highest-severity (F2) breaks the single-FR happy path: `/implement` archives an FR but doesn't rewrite the plan-file traceability link, and `/gate-check` probe #23 fails on the commit `/implement` just made.

## Why this milestone exists

The M32 self-check work closed three discretionary-check failure modes inside `/spec-write` and `/setup`. Smoke-test run #4 surfaced a different family of gaps: cleanup work that `/implement`'s Phase 4 archival doesn't reach.

M33 closes the gap on three surfaces:

1. **Archival hygiene (F2 + F3)** — `/implement` Phase 4 archives the FR but leaves plan-file prose stale. The link-rewrite logic exists (`adapters/_shared/src/spec_archive/rewrite_links.ts`); `/spec-archive` invokes it; `/implement` Phase 4 doesn't. Plan-file `verify:` lines that reference deleted files (e.g., `tests/.placeholder.test.ts`) survive in the same way.
2. **Workflow UX consistency (F1)** — `/spec-write` is the only skill in the chain that emits no closing summary on success. Operators and any driver lose the per-skill console-status contract that the rest of the chain honors.
3. **Template policy reconciliation (F4 + F5)** — the gate is permissive about test layout (`*.test.ts` outside `node_modules` always passes); the templates are strict (`tests/` mirror prescribed). `requirements.md` advertises "cross-cutting only" but ships an FR-1 placeholder. Both rot the docs without any probe catching them.

Each fix shifts work from the discretionary path (LLM remembers, or operator fixes by hand) to in-band enforcement (helper + post-write check + gate probe).
