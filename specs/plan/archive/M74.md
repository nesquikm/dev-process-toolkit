# M74: Hooks bundling redo — M71/M72/M73 install-side falsified {#M74}

Release target: v2.22.2
Codename: TBD

## Summary

Forward-only fix reversing the M71/M72/M73 install-side hook mechanism. M71 (STE-285) shipped `/setup --hooks` writing to user `.claude/settings.json`, M72 (STE-286) added smoke fixture group 8 + the `--hooks=<value>` preselect flag, M73 (STE-288) "fixed" the path shape to a literal `${CLAUDE_PLUGIN_ROOT}` token. None of it fired empirically — the Claude Code harness only expands `${CLAUDE_PLUGIN_ROOT}` in plugin-bundled `<plugin>/hooks/hooks.json`, not in user `.claude/settings.json`. M74 ships the plugin-bundled `plugins/dev-process-toolkit/hooks/hooks.json` (harness auto-discovers + fires across every project where the plugin is enabled) and rips out the install-side mechanism entirely.

Discovery 2026-05-14: `/dev-process-toolkit:setup --hooks=all` run against v2.22.1 in `~/workspace/quack` produced the harness refusal `Hook command references ${CLAUDE_PLUGIN_ROOT} but the hook is not associated with a plugin. This variable is only available in hooks defined in a plugin's hooks/hooks.json file, not in [user settings.json]`. Empirical research via the `claude-code-guide` agent + `code.claude.com/docs/en/plugins-reference.md` confirmed the contract.
