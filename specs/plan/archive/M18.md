# M18 — Filename convention (tracker-ID-keyed) {#M18}

**Release target:** v1.21.0 "Filename Convention" (pragmatic) or v2.0.0 (SemVer-strict). Not time-bounded. Decision deferred to Phase C.
**Status:** draft (not yet kicked off)
**Origin:** brainstormed 2026-04-23 during M17 spec-write; re-brainstormed 2026-04-23 post-M17-ship to lock the remaining open questions (archive handling → option α, mode: none filename → option B short-ULID, granularity → 2 FRs). STE-62 added 2026-04-23 after discovering `active_ticket:` is a documented-but-unwired dead-surface key, same shape as M17's STE-55..STE-58 sweep.

## Why this milestone exists

Filename ergonomics. A bare `specs/frs/<fr_26-char-ULID>.md` path is collision-proof and stable, but humans don't read ULIDs. In tracker-mode projects — the primary use case — the ticket ID is the natural reference in prose, ACs, headings, commit messages, and Linear UI. The filename should match.

M13 (FR-47/48, file-per-FR layout) and M16 (FR Identity Stability) had good reasons for ULID as file identity — collision-proof under concurrent creation, mode-neutral, survives tracker renames. M18 accepts the ergonomic tradeoff: the collision-proof ID stays in frontmatter (`id: fr_<ULID>`), but the filename switches to the human-facing identifier. In tracker mode: `STE-53.md`. In `mode: none`: `<short-ULID>.md` matching M16's AC-prefix convention (e.g., `VDTAF4.md`).

## Deferred decisions

1. **Codename**: "Filename Convention" is the working title. Something more poetic welcome at spec-write or PR time.
2. **Version bump**: v1.21.0 vs v2.0.0. Decision at Phase C.
3. **Mode-transition rename atomicity** — exact sequencing inside `/setup`'s `mode: none → tracker` flow (rename file + create ticket + update frontmatter as one commit vs. staged). Sketched in STE-60 AC-STE-60.6; may surface edge cases during implementation.
4. **Rewrite-script commit vs. discard** — STE-61's mechanical rewrite uses a one-shot script; whether to commit `scripts/rewrite-fr-filenames.ts` (record of the transform) or discard after use. Preference to commit for auditability; decide during implementation.

## Not in M18 scope (explicit)

- **CHANGELOG.md rewrite** — historical entries reference old convention; intentionally preserved as immutable record
- **git history rewrite** — `git log --follow` tracks renames; no need to rewrite history
- **Short-ULID collision handling beyond M16's `scanShortUlidCollision`** — reuse as-is
- **Slugs in filenames** — bare identifier only; titles rename but filenames shouldn't
- **Gate-check perf optimization for large archive sets** — noted in M17 risk scan; future work, not M18
