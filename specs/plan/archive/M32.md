# M32 — Convention Enforcement {#M32}

**Release target:** v1.34.0. Codename TBD (decide at `/ship-milestone`).
**Status:** drafted 2026-04-27 from /smoke-test 2026-04-27 run #3 findings (`/tmp/dpt-smoke-findings-2026-04-27.md`) F4/F5/F6.
**Origin:** smoke-test against v1.31.0 "Polyglot" reproduced three discretionary-check failure modes that survived M29's STE-108/110/114 fixes. Each because the load-bearing rule lives in SKILL.md prose alone — and the LLM-as-runtime can drift past prose. Same shape as M31's project-binding / project-milestone / milestone-collision failures.

## Why this milestone exists

The toolkit's gate-check probes (#13, #15, #19) catch malformed `/spec-write` and `/setup` output at gate time, exactly as designed. But the producing skills (`/spec-write`, `/setup`) emit the malformed output in the first place because their internal contracts are LLM-discretionary: SKILL.md prose telling the LLM what to do, with no system-enforced post-condition or helper-backed canonical-shape generator.

M32 closes that gap on three surfaces:

1. **F4** — `/spec-write` leaks `id:` ULID frontmatter in tracker mode despite STE-110's prose flip.
2. **F5** — `/spec-write` lands ACs as `AC-<milestone>.<N>` instead of `AC-<tracker-id>.<N>` because the requirements template hardcodes the wrong shape.
3. **F6** — `/setup` produces CLAUDE.md without the `## /setup audit` section because per-step `appendAuditEntry` calls are LLM-discretionary.

Each fix shifts the rule from "prose-only" to "prose + helper + post-write self-check". The probe at gate time remains as the universal safety net.

## Implementation order

**Phase A — all three FRs in parallel (independent surfaces).**

- **STE-121** introduces `adapters/_shared/src/fr_frontmatter.ts:buildFRFrontmatter`, mandates the helper in /spec-write SKILL.md step 0b, and adds an in-band post-write probe-13 self-check. One-time migration script for any active tracker-mode FRs that already carry `id:`.
- **STE-122** edits `templates/spec-templates/requirements.md.template` to use the `AC-<tracker-id>.<N>` placeholder (mirroring STE-66's filename rule), introduces `adapters/_shared/src/guessed_tracker_id_scan.ts` for in-band post-write scanning, and updates /spec-write SKILL.md to mandate the substitution + scan.
- **STE-123** adds /setup step 8a "audit-section post-condition", introduces `adapters/_shared/src/setup/synthesize_audit.ts` for deterministic synthesis from the in-scope resolved-defaults table, and exposes `hasDefaultApplicableOutcomes` from probe-19's module for cross-module reuse.

No inter-FR sequencing. STE-121 and STE-122 both edit `skills/spec-write/SKILL.md` at different sections (step 0b vs. step 1+); lockstep merge inside the milestone is straightforward.
