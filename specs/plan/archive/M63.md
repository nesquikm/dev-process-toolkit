**Release target:** v2.13.0 (minor — additive; one new subagent, one new forked skill, two cross-cutting integrations into existing skills, one allowlist append, one new gate-probe. Bundles with M61 + M62 in the next release window.)

## Why this milestone exists

Prior FR design rationale lives in archived `## Technical Design` sections and dies on archive — `specs/frs/archive/` is excluded from `/gate-check`, `/docs`, `ac-lint`, and `/spec-review`. New `/brainstorm` sessions re-litigate tradeoffs already settled; new `/spec-write` runs miss reusable AC patterns from cousin features. Industry SDD tooling (GitHub Spec Kit, OpenSpec) addresses this either via a project-level `constitution` doc or by lifting deltas into living specs at archive time.

M63 takes a third path: an on-demand topic-aware retrieval subagent that runs in its own context window and returns a fixed-shape ≤25-line block to the parent skill. Archive stays a passive audit trail; no new artifact type; no schema churn.

After M63 ships:

- `/brainstorm` invokes the new `spec-research` forked skill at the end of Step 1 (problem clarified, approaches not yet proposed). Step 2's approach proposals reference real precedents.
- `/spec-write` invokes the same skill in § 0b between "Build canonical frontmatter" and "AC prefix". The technical-design interview asks targeted questions like "FR-X chose approach A here — follow or diverge?" instead of starting from zero.
- `--no-tech` short-circuits both invocations (no consumer for the seed; vacuous).
- `/gate-check`'s new `spec_research_result_shape` probe (#41) catches drift in the subagent's output contract.

1. Recency-window injection — rejected: recency ≠ relevance, archive grows unbounded.
2. Decision lift on archive (ADR pattern) — rejected: requires new artifact + manual judgment per archive.
3. Topic-aware subagent retrieval — selected.

- Original `## Cautions` section dropped from the output contract; replaced with the "may be stale" banner. Rationale: gate-probing contradiction detection is shape-not-content; trust the parent LLM.
