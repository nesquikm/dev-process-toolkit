# M34 — Workflow Diagram {#M34}

**Release target:** v1.36.0 (provisional). Codename TBD; finalize at `/ship-milestone`.
**Status:** drafted 2026-04-27 from `/brainstorm` decision (Approach 2 — Phased subgraphs).
**Origin:** user-noticed gap — the README's "What You Get" commands table forces new visitors to reconstruct the lifecycle from a flat list of 14 commands. A diagram makes the four-phase pipeline (Setup → Plan → Build → Ship) visible at a glance.

## Why this milestone exists

The README is the install funnel's first page. Today it shows install + setup invocation, then jumps straight into a 14-row commands table that gives no sense of order or cadence. First-time visitors infer "lots of commands, unclear how they fit together"; mid-onboarding adopters who already ran `/setup` have to scan the same table to figure out what's next.

A mermaid `flowchart TD` with four labeled subgraphs (Setup, Plan, Build, Ship) collapses the 14 commands into a four-phase mental model that both audiences can read in seconds. User-invoked skills only — internal skill-to-skill calls (e.g., `/implement` invoking `/tdd` + `/gate-check` + `/docs --quick` + `/pr`) get a short prose paragraph below the diagram, keeping the visual surface small.

Single-FR milestone. The /brainstorm session explicitly rejected scope expansion to a regression test asserting skill-name parity with `skills/*/SKILL.md` — the simplicity constraint dominates. If diagram drift becomes a recurring problem, a future FR adds the probe.

## Risk scan

| Category | Risk | Severity | Mitigation |
|----------|------|----------|------------|
| GitHub mermaid compatibility | Mermaid syntax not rendering on github.com (some `flowchart` features don't ship in GitHub's renderer) | low | Stick to vanilla `flowchart TD` + `subgraph` syntax both confirmed working on GitHub since 2022. No automated probe — explicitly rejected by /brainstorm "as simple as possible". Re-evaluate if drift recurs. |

Phase labels are the bridge — first-timers read top-down for the lifecycle; adopters scan to their phase. PR review is the second checkpoint. |

No high-severity risks. Two medium (drift, audience-fit) — both human-review checkpoints, no automated mitigation per the simplicity constraint. Two low.
