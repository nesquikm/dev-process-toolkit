## M91: Capture & store design reference images {#M91}

**Goal:** Let toolkit users persist design reference images (Figma exports, mockups, screenshots) into a version-controlled `specs/design/` tree and reference them from FR specs. Approach 2 from the 2026-06-30 brainstorm (persistent store, specs-only archive; duck-council-confirmed): images **never move**, references are repo-root-relative, so a reference is byte-identical and valid whether its FR sits in `specs/frs/` or `specs/frs/archive/` — zero archival rewrite machinery. This is **Deliverable A**; the `/implement` visual conformance check (Deliverable B — Dart MCP for Flutter incl. Flutter web, Chrome MCP for pure web, advisory, Flutter first) is explicitly deferred to a later milestone.

**Prerequisites:** None new. Builds on the existing FR-file layout (`Provider.filenameFor`), the archival machinery (`rewrite_links.ts`), and the `/gate-check` probe harness.
