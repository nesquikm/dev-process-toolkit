# M57 — Conformance Loop (/conformance-loop skill) {#M57}

**Release target:** v2.10.0 (minor — additive, project-local skill; zero plugin-API changes; honors `/smoke-test`'s existing rules in default mode).
**Codename:** TBD at `/ship-milestone` time.

**Status:** drafted 2026-05-05 from a brainstorm + spec-write session that surfaced the deliberate-override conflict with `/smoke-test`'s "Capture, don't fix" + "One run per release cycle" rules. The new skill is the formal sanction for that override under explicit safety rails.

## Why this milestone exists

The toolkit's smoke-test workflow today is two-terminal manual:

1. Operator opens terminal A → `/smoke-test --tracker linear`.
2. Operator opens terminal B → `/smoke-test --tracker jira --jira-project DST`.
3. Both runs surface findings into per-tracker files.
4. Operator reads both, manually opens FRs via `/spec-write`, runs `/implement` per FR.

Steps 1–3 take ~10 min wall-clock per tracker. Step 4 is unbounded: post-M55 runs found 6 FRs to file; post-M56 runs find 3+. The manual loop dominates triage time.

M57 closes this with a single new project-local skill `/conformance-loop` that:

- Replaces the two-terminal pattern with one parent-driven invocation that spawns both `/smoke-test` runs as parallel `claude -p` subprocess sessions.
- Aggregates the two per-tracker findings files into one deduplicated report (cross-tracker regressions surface once, not twice).
- In default `capture-only` mode, exits there — the operator runs `/spec-write` + `/implement` manually as before.
- In opt-in `--auto-fix` mode, sequentially walks the deduplicated high-severity findings list and dispatches `/spec-write` (allocate FR) → `/implement` (build + commit) per finding, then re-iterates until `green` | `max-iterations` | `no-progress`.

Capture-only is the default to preserve `/smoke-test`'s "Capture, don't fix" rule unchanged for raw invocations; `--auto-fix` is the formally-sanctioned deviation.

After M57 ships:

- Two-terminal manual smoke workflow continues to work unchanged (`/smoke-test` is not modified).
- `/conformance-loop` (no flags) replaces the two-terminal pattern with one invocation that yields a single deduplicated findings report.
- `/conformance-loop --auto-fix --max-iterations 3` runs the full automated cycle until convergence (`green` | `max-iterations` | `no-progress`).
