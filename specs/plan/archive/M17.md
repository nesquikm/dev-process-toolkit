# M17 — Fixes and Cleanup {#M17}

**Release target:** v1.20.0 "Fixes and Cleanup" (may ship as v2.0.0-rc.1 if the SemVer-strict interpretation is adopted — deleting shipped behavior triggers the major bump. Decision deferred to release time.)
**Status:** draft (not yet kicked off)
**Origin:** 2026-04-23 brainstorm session. Context: the plugin is approaching wider consumer use and dogfooding surfaced two bugs (bare plugin-internal paths don't resolve in consumer installs; `/implement` Phase-4 sometimes leaves Linear tickets stuck `In Progress`). The milestone combines those two fixes with a four-item deletion sweep (v1→v2 migrator, `specs/.dpt-layout` + probes, `specs/INDEX.md` + `regenerateIndex`, `### Sync log` + helper) that removes shipped-but-unused surface before v2, plus a seventh FR installing a root spec-file hygiene gate motivated by the drift report from M12-M16 archival.

## Why this milestone exists

Two bugs and four deletions fell out of preparing the plugin for wider use.

**Bugs are real consumer-install blockers.** FR-A: five skills invoke bundled helpers via bare `adapters/_shared/...` paths that don't resolve outside this repo — silent degradation in every consumer install since v1.17.0. FR-B: `/implement` Phase 4 inconsistently drops `releaseLock`, leaving Linear tickets stuck `In Progress` (instruction gap + mechanism gap + no backstop).

**Deletions are pre-release right-sizing.** No v1 projects exist outside this repo, so the v1→v2 migrator and `.dpt-layout` probe are dead weight (FR-C1, FR-C2). `INDEX.md` is write-only with zero readers (FR-C3). The `### Sync log` is a write-only audit trail duplicating `git log` (FR-C4). Collectively these remove ~400 lines of code + ~1000 lines of docs that consumer projects would otherwise inherit.

**Spec hygiene FR (FR-D)** is motivated by `/tmp/dpt-drift-2026-04-23.md`: M12-M16 archival surfaced that the three root spec files (`requirements.md`, `technical-spec.md`, `testing-spec.md`) had accumulated milestone-specific framing that rotted after milestones archived. FR-D's gate makes the "root specs are shape-only, current-only" invariant enforceable rather than aspirational.

Each deletion exits through `/gate-check` green + its own ripgrep gate before proceeding.

**Phase D — Hygiene (STE-59):**

Runs AFTER STE-55..58 so the cleanup reflects post-deletion state (no migrator references, no INDEX.md mention, no Sync log narrative).

**Phase E — Release (no FR):**

- Bump versions in `plugin.json`, `marketplace.json`, `CHANGELOG.md`, `README.md` per release checklist
- Ship v1.20.0 "Fixes and Cleanup" — or v2.0.0-rc.1 if SemVer-strict. Decide at release time.
