# M22 — v1.22.0 release-prep audit sweep {#M22}

**Release target:** v1.22.0 (M22 lands on `feat/m12-tracker-integration` before the tag cuts; v1.22.0 ships M19 + M22 together).

## Why this milestone exists

Before the current PR merges and v1.22.0 tags, every user-facing claim must match code reality. The audit found three classes of drift:

1. **Mechanical cleanups** — uncommitted hygiene, stale references, CHANGELOG test-count accuracy, keyword sync across manifests.
2. **Incomplete shipped ACs** — AC-STE-67.2 (rewrite 5 Linear descriptions) was deferred because the M19 `/implement` session had no Linear MCP. M22's session does (verified 2026-04-24).
3. **Contract parity** — `adapters/jira.md` missing the `claimLock-skipped` and `silent no-op` trap subsections that `adapters/linear.md` has; both traps fire via shared `adapters/_shared/src/tracker_provider.ts`, so the behavior is adapter-agnostic but the docs aren't.

Per "no external users, no migration burden": cleanups delete, not preserve. No tracker ticket per finding — the 22 audit findings collapse into 6 coherent FRs (ratios: H items → 8 findings across 3 FRs; M items → 9 findings across 4 FRs; L items → 5 findings folded into the mechanicals FR). Dropped under the same principle: L6 (directory naming label, not a bug yet), L7 (no migration to guard).

- **Ships as first commit.** `fix(m22): swap .mcp.json off V1 SSE endpoint (STE-77)`.

**Phase B — Mechanicals, AC-STE-67.2 close, template convention (parallel):**

Three independent FRs, each a one-shot doc/config edit. STE-79 rewrite executes 5 `save_issue` calls. STE-78 mechanicals touch 10 files but each edit is surgical (keyword add, changelog count fix, broken-link removal, etc.). STE-80 template convention is 2 files.

- **Ships as one or two commits.** Likely `chore(m22): release-prep mechanicals + AC-STE-67.2 close + template convention (STE-78, STE-79, STE-80)`.

**Phase C — Adapter contract parity:**

Copies `claimLock-skipped trap` and `Silent no-op trap` subsections from `adapters/linear.md` into `adapters/jira.md` (s/Linear/Jira/ where tracker-specific). Expands `_template.md` with "MCP tool names" and "Adapter-specific traps" sections. Deletes `read_status` dead capability from `linear.md`.

- **Ships as its own commit.** `docs(m22): adapter contract parity — jira trap sections + template expansion (STE-81)`.

**Phase D — Gate-check probe integration tests (largest code work):**

Biggest code work in M22. Adds 6 `tests/gate-check-<slug>.test.ts` files for probes 1, 2, 3, 4, 7, 11. Each test builds a minimal project-tree fixture that triggers the probe's grep/scan logic (positive + negative). Adds a convention note to `skills/gate-check/SKILL.md`: every new probe ships with a corresponding `gate-check-<slug>.test.ts` file.

- **Ships as its own commit.** `test(m22): gate-check probe integration tests + probe-test convention (STE-82)`.

**Phase E — Release cut:**

Augment CHANGELOG v1.22.0 entry with `### Fixed (release-prep)` listing the 22 audit finding IDs + the 6 M22 FR tracker IDs (STE-77..STE-82). Decide whether to revise v1.22.0 codename from "Branch Convention" to cover M19+M22 scope. Tag v1.22.0. Merge PR.
