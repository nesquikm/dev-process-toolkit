# M46 — Smoke-test parallel-safe artifact isolation {#M46}

**Release target:** v1.46.0 (minor — skill-prose changes only; no plugin-facing API change). Codename: TBD (finalize at `/ship-milestone`).
**Status:** drafted 2026-04-30.

## Why this milestone exists

Two-terminal `/smoke-test` parallelism (Linear + Jira simultaneously) is the simplest path to halving release-candidate validation wall-clock from ~20 min to ~10 min. The brainstorm session (2026-04-30) considered three approaches: (1) per-tracker artifact isolation, operator runs two terminals; (2) one-driver fan-out in Phase 2; (3) full driver-side parallelism. Approach 1 was selected — minimum viable surface area, clean failure isolation per tracker, no console-multiplexing or merged-findings logic to maintain.

A sibling FR rolls in mid-flight: STE-160 closes a `/spec-write` skill-prose gap surfaced during STE-159's own spec-write run (LLM operator extrapolated `/implement`'s "claim at FR start" rule and pre-transitioned the new ticket to `In Progress`; user caught it and asked for a plugin-level fix). STE-160 lands an explicit "No claim on create" invariant in `/spec-write` so the regression doesn't recur on every tracker-mode spec-write.

After M46 ships:

- Two `/smoke-test` invocations with different `--tracker` values can run concurrently in two terminals without filesystem collision or artifact-overwrite races.
- The bare `../dpt-test-project` basename is retired; both canonical-good basenames (`dpt-test-project-linear`, `dpt-test-project-jira`) appear in pre-flight #6's allow-list.
- All `/tmp/dpt-smoke-*` artifacts carry a tracker suffix.
- The operator-driven parallelism contract (no combined-mode flag, no merged findings, expected win below 2× due to shared rate-limit budget) is documented in the skill prose.
- `/spec-write` carries an explicit "No claim on create" invariant — freshly-created tracker tickets land in the tracker's default state (Linear: Backlog unassigned; Jira: project start state), with the claim deferred to `/implement` Phase 1.
