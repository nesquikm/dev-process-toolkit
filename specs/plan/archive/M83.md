# M83: README Commands-table Args column {#M83}

Release target: v2.28.1 (provisional, patch — documentation-only README edit)
Codename: TBD (decide at /ship-milestone)

## Summary

Document the argument shapes of the 16 user-callable skills in the README Commands table by adding a third column (`Args`) populated verbatim from each skill's `argument-hint:` YAML frontmatter, and strip the `/dev-process-toolkit:` namespace prefix from the Commands column to match the bare-name convention already used in the Agents table directly below it (STE-130 precedent).

Documentation-only workaround for upstream Claude Code issue [#43401](https://github.com/anthropics/claude-code/issues/43401) — `argument-hint:` does not render in autocomplete for plugin-namespaced skills, in contrast to project-local skills (e.g., `/conformance-loop`) and the older `commands/*.md` format (e.g., ralph-loop), both of which render their hints inline. The README Commands table is the primary discovery surface until upstream closes the gap. Skill YAML stays untouched for forward-compat: when upstream lands the rendering fix, autocomplete will start working without a second migration.
