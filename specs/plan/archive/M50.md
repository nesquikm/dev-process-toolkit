# M50 — Socratic: /setup prompt-sequencing tightening + shared patterns rule {#M50}

**Release target:** v2.3.0 (minor — additive prose; new `Pattern 26` section in `docs/patterns.md`; no Release-Files schema change, no adapter contract change). Codename: **"Socratic"**.
**Status:** drafted 2026-05-01 from user feedback during a /spec-write session ("we use socratic discussion in /brainstorm — /setup asks a lot of questions at once"). Single-FR milestone: STE-177.

## Why this milestone exists

`/brainstorm` and `/spec-write` enforce one-question-at-a-time prompting (Socratic discipline). `/setup` does not — step 7d at `plugins/dev-process-toolkit/skills/setup/SKILL.md:195` instructs **"Three yes/no prompts in this exact order, after 7c"**, ambiguous wording the LLM reads as "ask the trio in one turn." Adjacent risk: steps 7b → 7c → 7d → 7e are four separate prompt sites within ~30 lines of skill prose; without an explicit rule binding them together, an LLM tends to collapse them into a single turn at the handoff between sections.

A user running `/setup` interactively for the first time on 2026-05-01 hit this directly: walls of yes/no prompts plus tracker-mode + branch-template + release-files questions in one turn, instead of the patient back-and-forth `/brainstorm` and `/spec-write` already deliver.

The fix is two-part:

1. **Extract the Socratic-prompting rule** into `plugins/dev-process-toolkit/docs/patterns.md` as `Pattern 26: Socratic Prompting {#pattern-socratic-prompting}`. The rule lives in one canonical place; future skills inherit by anchor reference. `/brainstorm`'s rationalization-prevention table is the existing source of truth — Pattern 26 is extracted from it (the table is preserved in `/brainstorm`; the canonical statement + table is added to `patterns.md`).
2. **Update `/setup` SKILL.md** to honor the rule — rewrite step 7d's lede explicitly ("ask one at a time, wait for each answer before asking the next") and add a top-level rule under `## Rules` naming steps 7b/7c/7d/7e and referencing Pattern 26. Add cross-reference back-pointers from `/spec-write` and `/brainstorm`.

After M50 ships:

- `/setup` step 7d's lede explicitly directs one-at-a-time prompting; ambiguous "Three yes/no prompts in this exact order" wording is gone.
- `docs/patterns.md § Pattern 26` is the single authoritative statement of the Socratic-prompting rule.
- `/setup`, `/spec-write`, `/brainstorm` all reference Pattern 26 by anchor; the rule has one source of truth and three reference points.
- `/implement`, `/ship-milestone`, `/gate-check`, `/docs` audit-confirmed as single-question by construction; no SKILL edits to those four (audit recorded in STE-177's `## Notes`).
