# M30 — Stale doc references {#M30}

**Release target:** v1.32.0. Codename TBD (decide at `/ship-milestone`).
**Status:** drafted 2026-04-27 from a /spec-write conversation flagging that several plugin-side docs encode shipped-milestone state in their filenames and bodies.
**Origin:** `plugins/dev-process-toolkit/docs/v2-layout-reference.md` is named after a v1→v2 layout migration that completed long ago (its own opening line declares "v2 is the only supported layout"). `plugins/dev-process-toolkit/docs/fr-39-sync.md` is named after FR-39 but its body opens with "STE-17 — Bidirectional AC Sync" — already drifted. Both files, plus several others in `docs/`, contain prose ("STE-26..STE-25", "Pre-M18 archives... until STE-61's one-time rewrite lands") that names shipped milestones in a way that is meaningless to a new reader.

## Why this milestone exists

The plugin's `docs/` tree is read by *future installers* who never lived through M18 or STE-39. Filenames and prose that encode internal milestone history advertise context the reader doesn't have. Per memory note `project_no_users_yet.md`, there are no users to keep stale links working for — the cleanup can be aggressive.

## Implementation order

**Phase A — STE-115 (mechanical).**

1. `git mv plugins/dev-process-toolkit/docs/v2-layout-reference.md plugins/dev-process-toolkit/docs/layout-reference.md`
2. `git mv plugins/dev-process-toolkit/docs/fr-39-sync.md plugins/dev-process-toolkit/docs/ac-sync.md`
3. Rewrite inbound references across active surfaces (per Q2 boundary): plugin `docs/*.md`, plugin `skills/**/SKILL.md`, plugin `agents/**/*.md`, root `README.md`, root `specs/{requirements,technical-spec,testing-spec}.md`. Skip: `**/archive/**`, `CHANGELOG.md`.
4. Verify with grep gate (AC-STE-115.4).
5. **Double-check pass** (AC-STE-115.6): re-run discovery against the *new* filenames to confirm every rewritten reference resolves to a real file; broader markdown-link integrity check across the in-scope surfaces catches collateral breakage.

**Phase B — STE-116 (editorial).**

6. Pass through each active doc and rewrite prose that names shipped state ("v2 is the only supported layout", "STE-26..STE-25", "Pre-M18 archives... until STE-61's one-time rewrite lands") into current-state framing. Preserve AC citations that explain *current* behavior (e.g., "(AC-STE-60.1)").
7. Reviewer reads each diff for whether the prose reads natural to a fresh installer.

**Phase C — Release:** `/ship-milestone M30` → v1.32.0.
