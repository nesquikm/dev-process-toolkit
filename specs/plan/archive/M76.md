# M76: Hook refusal exit code — make it actually block {#M76}

Release target: v2.22.4
Codename: TBD

## Summary

Single-FR fix-class milestone. STE-291 closes the residual exit-code-semantics gap left by STE-290 (M75 "Wired"). The 3 Refusing hooks fire and emit NFR-10 stderr, but call `process.exit(found ? 0 : 1)` — Claude Code treats `exit 1` as advisory (stderr to operator only, command proceeds). The actual blocking contract is `exit 2` (stderr to model, command blocked). One-byte-per-hook flip; 6 assertion tightenings; one doc paragraph pinning the Claude Code exit-code contract.
