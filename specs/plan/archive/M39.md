# M39 — Audit cleanup tier 2 {#M39}

**Release target:** v1.39.0 (provisional minor — schema deletions and skill-overflow extraction are observable behavior changes for contributors). Codename TBD; finalize at `/ship-milestone`.
**Status:** drafted 2026-04-28 alongside M38 from the same audit reviews. M39 ships AFTER M38 — these FRs were classified as "next milestone (optional)" by the comprehensive review's ship-readiness verdict.

## Why this milestone exists

The 2026-04-27 audit produced 35 verified findings. M38 cleared the four mandatory release blockers. M39 absorbs the remaining audit-cleanup work that's worth doing but was not blocking ship:

- **F1 + F3 + F3a** (STE-139) — NFR-1 enforcement triple. `setup/SKILL.md` (428) + `implement/SKILL.md` (371) violate the ≤300-line cap; `tests/docs-skill-shape.test.ts` enforces it for `docs/SKILL.md` only; 23 of 29 gate-check probe tests lack a repo-self-check, so live-repo drift like F0 (closed in M38 STE-136) was invisible to `bun test`. Fix: extract overflow + per-skill probe + 23 backfilled self-checks.
- **F7** (STE-140) — `fr.schema.json` and `plan.schema.json` describe a pre-STE-76 / pre-archived-status world. **121 of 153** archived FRs would fail `fr.schema.json`; **37 of 37** plans would fail `plan.schema.json`. Zero `ajv` consumers in production code; `docs/layout-reference.md` falsely claims live FRs/plans validate. Fix: delete (subtract) under no-users.
- **F9** (STE-141) — pre-M18 dead test infrastructure: `v2-minimal` fixture + `runSchemaMProbe` + `verify-regression.ts` script-mode + `.sh` wrappers + `capture-regression.ts`. Self-referential pre-M18 island that always passes against itself; zero `bun test` consumers. Resolver fixtures (initially flagged) are NOT dead — `resolve.test.ts:436-486` consumes all four; explicitly out of scope. Fix: delete.
- **F10** (STE-142) — commitlint variant inherits `header-max-length: 100` from `@commitlint/config-conventional` defaults; shell hook enforces 72 per STE-133.1. Fix: rule override + parity test.
- **F8-Tier-2 + Tier-3** (STE-143) — adapter docs (~15 refs, hand-edit) + skill prose + docs (~700 refs, scripted sweep). Bundled because they share the cleanup goal; can be split into two atomic commits if review pressure benefits.
- **G14** (STE-144) — `mode-none-regression` fixture references in `testing-spec.md` point at a non-existent fixture. Rename to `mode-none-baseline` (the real fixture).
- **G15** (STE-145) — `spec-review/SKILL.md` is the only skill carrying `allowed-tools:` frontmatter. Document the convention in `docs/skill-anatomy.md`.

The seven FRs run mostly independently, with one ordering constraint: STE-143 (the namespace sweep) should NOT run while STE-139 (overflow extraction) is mid-flight, because both touch skill SKILL.md files. Sequence Phase A (STE-139) before Phase E (STE-143).
