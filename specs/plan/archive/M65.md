# M65: Auto-Mode Refusal Protocol {#M65}

Release target: v2.16.0
Codename: TBD

## Summary

Close the autonomous-mode contract on the **refusal side**. STE-226 (M-prior) gave the toolkit a byte-checkable marker for default-apply; the dual was missing — when no marker is present and no live human can answer, every `requires-input:` step must refuse with NFR-10 canonical shape rather than silently model-impute. Adds a single canonical protocol doc, a shared `requireOrRefuse` helper, an `imputed` audit-trail column, and a `/gate-check` probe that enforces citation + sentinel coverage across every skill in scope (incl. `/smoke-test` Phase 0).

Triggered by the v2.13.0 incident gist (https://gist.github.com/nesquikm/d6d7c8303b7b923ab0dfc4aa902ec5ec) — `/setup` step 7b model-imputed `tracker_mode=none` despite the skill text literally forbidding it.
