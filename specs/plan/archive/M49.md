# M49 — Smoke #6/#7 Follow-up: Archive Hardening + Smoke-Driver Hygiene {#M49}

**Release target:** v2.2.0 (minor — additive prose + small behavior fixes to `/implement`, `/spec-review`, `/setup`, `/simplify`, `/smoke-test`; no Release-Files schema change). Codename: TBD (finalize at `/ship-milestone`).
**Status:** drafted 2026-04-30 from smoke runs #6 (Jira, `/tmp/dpt-smoke-findings-2026-04-30-jira.md`) and #7 (Linear, `/tmp/dpt-smoke-findings-2026-04-30-linear.md`). FRs allocated 2026-04-30: STE-171…STE-176.

## Why this milestone exists

Smoke #6 (Jira, run #6 in the smoke-test history) and smoke #7 (Linear, run #7) both passed end-to-end against v2.1.0 (29/29 conformance probes green; full canonical chain `/setup → /spec-write → /implement → /gate-check → /spec-review → /simplify` cleared on each tracker), but each surfaced a small batch of low-to-medium-severity follow-ups that wouldn't have been caught without the live runs. M49 absorbs both findings sets in one milestone:

1. **`/implement` Stage D archive flow** has three small holes the smoke caught:
   - the **traceability matrix row** in `specs/requirements.md` § 6 is not appended on AC ship even though the inline comment promises "rows are added by /spec-write or /implement once an AC ships" (F4 Jira),
   - `git mv` falls back to plain `mv` when the FR file is **untracked**, losing rename history in `git log --follow` (F2 Jira),
   - the `cleanup_plan_verify_lines.ts` helper exists and is mentioned in Stage D prose but did **not** fire on the Jira run for the M1 placeholder-removal verify line — `/gate-check` probe #28 still warned (F3 Jira). Either the helper was not invoked or its `deletedFiles[]` argument missed `src/.placeholder.test.ts`; either way the failure mode is reproducible.

2. **`/spec-review` is correctly catching cross-cutting drift** but only at audit time — the operator gets a `PASS WITH NOTES` verdict with four drifts and no nudge to re-run anything. M49 adds a one-line "live-spec refresh suggested" hint on `drift_count >= 2` (threshold tuned to suppress single-line cosmetic drifts) so the next `/implement` invocation isn't surprised by accumulated drift (F4 Jira forward-looking).

3. **Two small SKILL.md prose gaps** caught by smoke #7:
   - `/setup` step 6b silently degrades when the model-layer block fires on `.git/hooks/commit-msg` writes; the audit log captures the skip but `SKILL.md` never tells the operator the install is "best-effort under non-interactive runs" or names the manual `cp` fallback (F6 Linear).
   - `/simplify` skips the gate re-run on no-op runs (correct behavior — re-running a clean gate against an unchanged tree is wasted tokens) but `SKILL.md` never documents the conditional, so a reader has to reverse-engineer it from the prompt (F6 Jira / F7 Linear, same shape across both runs).

4. **`/smoke-test` driver-side fixes** (project-local skill, `.claude/skills/smoke-test/SKILL.md`, not the plugin) — both runs hit the same stale-scratch friction:
   - prompt files at `/tmp/dpt-smoke-prompt-*.txt` were not rotated between runs (smoke #6 F1 — root cause of phantom "Linear bits" args contradiction; smoke #7 F2/F4 reproduced),
   - smoke #7 F1 — pre-flight #5 probes Linear team by `query=` (name-prefix match) instead of `key`, so `STE` (the team key) is rejected even though it's the canonical operator entry point,
   - smoke #7 F5 — pre-flight #2 refuses on existing `../dpt-test-project-<tracker>` with no `--reset` escape hatch.

5. **Two `/smoke-test` coverage gaps** that need driver-side extensions:
   - smoke #6 F5 — orphaned `dpt-smoke`-labeled non-`Done` issues accumulate in the Jira Space across runs; an optional pre-flight #9 (Jira-only) would catch the cluster before Phase 1 starts.
   - smoke #6 left **AC-STE-154.9 AC 6 (Jira comment path) uncovered** — the canonical chain doesn't exercise `mcp__atlassian__addCommentToJiraIssue`, so a regression there would slip past the smoke. A stand-alone Phase 2 probe (chosen over the natural side-effect-of-/implement variant — validates the MCP tool independent of /implement's narration policy) closes the gap.

After M49 ships:

- `/implement` Stage D appends the traceability row, `git add`s untracked FR files before `git mv`, and reliably invokes `cleanupPlanVerifyLines` (or the helper itself is patched to recognize "removed" verify-line semantics).
- `/spec-review` surfaces a one-line refresh hint when drift count >= 2.
- `/setup` and `/simplify` SKILL.md prose document their respective conditional behaviors.
- `/smoke-test` clears `/tmp/dpt-smoke-prompt-*` and `/tmp/dpt-smoke-<tracker>-*.log` at Phase 0.5, probes Linear team by key, supports `--reset`, ships an optional Jira ghost-cluster pre-flight, and exercises the Jira comment path via a stand-alone probe.
- AC-STE-154.9 AC 6 is no longer the only uncovered AC in the smoke-validated set.

Deferred from M49 (de-scoped, not blocking): smoke #7 F3 — `--prune-old-findings <days>` flag for `/tmp/dpt-smoke-findings-*.md` housekeeping. Low-priority informational; revisit when a follow-up batch is queued.

## Implementation order

**Phase A — STE-171 first.** The /implement Stage D hardening touches the canonical archive flow that every milestone-archival run after M49 will exercise. Its three sub-fixes are all in `plugins/dev-process-toolkit/skills/implement/SKILL.md` Stage D plus `plugins/dev-process-toolkit/adapters/_shared/src/spec_archive/`, so they share file-edit surface and collide on diff if split.

**Phase B — STE-172, STE-173, STE-174 in any order.** `/spec-review` drift-hint, `/setup` commit-msg docs, `/simplify` noop docs are independent and small (1–2 lines of SKILL.md prose each, plus the spec-review needs a small numeric drift-count threshold). Run them in parallel after Phase A; sequence them on the implementation side for diff readability.

**Phase C — STE-175 then STE-176.** `.claude/skills/smoke-test/SKILL.md` changes have no plugin dependency, but landing them after Phase A/B gives the next smoke run cleanly-instrumented coverage of the just-hardened archive flow + the new prose. Splits across two FRs because driver hygiene (scratch-reset + pre-flight #5 + --reset) and coverage extension (pre-flight #9 + AC 6 comment path) are functionally independent. Sequence STE-175 before STE-176 to avoid merge conflicts on the same SKILL.md file.
