# M78: Dependency docs catalog and retrieval {#M78}

Release target: v2.25.0
Codename: Vendored

## Summary

Single-FR milestone. STE-301 adds a dependency-docs catalog so consumer projects (e.g. a mobile app with several private packages) can fold sibling-package docs into design context during `/brainstorm` and `/spec-write`. A git-tracked manifest at `specs/deps.yaml` lists sibling packages by relative path (`../<name>`) + optional origin URL; a new `/deps` management skill provides `add | edit | delete | list | sync` subcommands (Socratic, AskUserQuestion-driven, offers `git clone` for missing entries); a new `deps-researcher` Haiku subagent (Read/Grep/Glob only, paired with a `deps-research` forked skill on the NON_COMMIT_PRODUCING_SKILLS allowlist) emits a closed-schema `deps-research-result` fenced block capped at 25 lines. Injection seeds in `/brainstorm` Step 1.5b and `/spec-write` § 0b step 2.5b mirror the STE-230 spec-research injection points. Architectural twin of STE-230 retargeted from the FR archive to `specs/deps.yaml`.

## Out of Scope (deferred)

- Non-toolkit doc formats (third-party libs, raw READMEs) — `kind: "toolkit-docs"` only in M78.
- Auto-pull / staleness detection on sibling checkouts — `/deps sync` is clone-only.
- Per-dep auth handling — relies on operator's existing `git` / `gh` credentials.
- Caching / memoization across fork invocations — fresh fork every time (STE-230 precedent).
- Bootstrap via `/setup` — manifest is operator-created on first `/deps add` in M78; auto-scaffold deferred.
- Migration helper for existing consumers — none needed per `project_no_users_yet`.
