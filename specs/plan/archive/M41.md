## Why this milestone exists

/smoke-test run #4 (2026-04-28) surfaced one high-severity workflow break:

- **F-S4-1** (STE-151) — `/gate-check` probe #14 ("Ticket-state drift — active side") fires a spurious **GATE FAILED** after `/dev-process-toolkit:implement <FR-id>` lands a single FR cleanly. Phase 4 Close releases the lock (Linear ticket → **Done**) but the FR file stays `status: active` per the milestone-bulk-archive design at `skills/implement/SKILL.md:216`. Probe #14 then sees `active FR + Done ticket` and fails the assertion `status == in_progress AND assignee == currentUser`. Workaround today is `/spec-archive <FR-id>`, which contradicts /spec-archive's documented escape-hatch framing.

The fix relaxes probe #14 to exempt the single-FR-clean state — pure prose + predicate edit; no migration, no schema change, no tracker-side behaviour change.
