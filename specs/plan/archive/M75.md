# M75: Hook contract fix — session.sh stdin-payload reader {#M75}

Release target: v2.22.3
Codename: TBD

## Summary

Single-FR fix-class milestone. STE-290 ports the broken `$CLAUDE_SESSION_FILE`-reading session.sh (STE-285, v2.21.0 "Honored", shipped 2026-05-13) to a Bun TS library + 4 per-hook TS modules that read `transcript_path` from hook stdin JSON per the empirically-verified 2026-05-14 Claude Code harness contract.

Empirical probe (2026-05-14, via a one-off `/tmp/dpt-hook-probe` minimal plugin) confirmed: harness passes session info as stdin JSON carrying `{session_id, transcript_path, cwd, permission_mode, hook_event_name, tool_name, tool_input, tool_use_id, prompt}`. The `transcript_path` field is the active session log path and correctly tracks `CLAUDE_CONFIG_DIR`. `$CLAUDE_SESSION_FILE` is never set by the harness — STE-285's read pattern hit its "fail-open when unset" branch on every dispatch, leaving the byte-checkable enforcement layer inert since v2.21.0.
