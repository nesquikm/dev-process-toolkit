# M53 — Smoke #10 Follow-ups: Probe Sensitivity + Driver Retry {#M53}

**Release target:** v2.6.0 (minor — one probe-behavior branch in `/gate-check` probe #26 (backwards-compatible: only false-positive case downgrades), one new driver sub-flow in `.claude/skills/smoke-test/SKILL.md` (project-local prose), one docs section in `skills/simplify/SKILL.md`. No probe-count delta, no plugin-facing API change). Codename: TBD at `/ship-milestone` time.
**Status:** drafted 2026-05-04 from the 2026-05-04 Linear + Jira smoke runs against v2.5.0 (`/tmp/dpt-smoke-findings-2026-05-04-{linear,jira}.md`). 3 FRs: 2 medium-severity sensitivity / hygiene fixes (gate-check probe #26 false-positive on documented capability gap; smoke driver retry-with-rollback on stream-idle) + 1 low-severity 4-time-re-confirmed docs gap (`/simplify` no-op gate-skip).

## Why this milestone exists

The 2026-05-04 smoke runs validated the v2.5.0 canonical chain (`/setup → /spec-write → /implement → /gate-check → /spec-review → /simplify`) end-to-end on both Linear and Jira paths. Both runs reported `PASSED WITH 4 NOTES`. The remaining net-new findings group into three families:

1. **Gate signal calibration (medium).** Linear-mode smoke runs gate-fail by construction on `/gate-check` probe #26 (`tracker-project-milestone-attached`) because the smoke driver creates a fresh Linear project per run with zero milestones. `/spec-write` already declares the capability gap in the FR's `## Notes` section via the canonical `milestone_attach_unavailable` capability key, but probe #26 does not consult the declaration. Result: every Linear smoke output reads `PASSED WITH ≥1 GATE FAILURE` despite the chain being otherwise clean — degrades the audit-trail signal. (STE-194)
2. **Driver retry on Anthropic-API transient (medium).** The Jira smoke `/setup` child spawn exited with `API Error: Stream idle timeout - partial response received` on the first attempt; the driver recovered manually using a deterministic rollback recipe (`git clean -fdq -e .claude -e .mcp.json && git checkout -- .`). Building the retry-with-rollback in turns a smoke-blocker into a quiet retry. (STE-195)
3. **Docs gap closure (low, 4th re-confirmation).** `/simplify` declares "Per skill rules, gate re-run is skipped" but the rationale lives only inside the skill body — `skills/simplify/SKILL.md` doesn't surface the carve-out. Four consecutive smoke runs have flagged this as durable; landing the one-section docs pass closes the loop. (STE-196)

After M53 ships:

- Probe #26 emits ADVISORY (not GATE FAILED) when the FR's `## Notes` carries the `milestone_attach_unavailable` capability-gap declaration; smoke audit-trail signal returns to `PASSED WITH 0 GATE FAILURES` for the Linear-tracker smoke fixture.
- `/smoke-test` Phase 2 spawns auto-recover from one stream-idle timeout via deterministic rollback + retry; double-timeout aborts the run with a canonical NFR-10 refusal naming both attempts and the recipe.
- `/simplify`'s no-op gate-skip behavior is documented in SKILL.md; the four-run-old "Per skill rules" reference now points at a real section.
