# M43 — Jira adapter Phase H conformance {#M43}

**Release target:** v1.43.0 (minor — adapter spec corrections + extended-value `jira_ac_field` sentinel; no new canonical key, no schema break for existing `mode: linear` users). Codename: TBD (finalize at `/ship-milestone`).
**Status:** drafted 2026-04-29 from `/smoke-test` run #5 findings against v1.42.0 (Atlassian Rovo MCP newly authenticated against `stellartechlab.atlassian.net`).

## Why this milestone exists

`/smoke-test` run #5 (2026-04-29) was the first Tier 5 manual conformance pass against a live Atlassian MCP — the prerequisite that v1.0–v1.42 lacked. The provisional `adapters/jira.md` was authored from public Rovo docs alone; smoke #5 surfaced 7 discrepancies between the documented spec and the live tool surface:

- **F1** ([STE-154](../frs/STE-154.md)) — `transition_status` matches by `to.name`; live MCP needs `to.statusCategory.key` as fallback for portable workflows. Default Kanban templates work zero-config only with this fix.
- **F2** — `contentFormat: "markdown"` parameter exists on the live MCP; the spec only mentions ADF / plain-text. Documentation gap.
- **F3** — Team-managed Kanban templates ship **no AC custom field** — `discover_field.ts` always returns `{ ok: false }` there. The spec assumes the field is discoverable or the user creates one. No documented description-body fallback. M43 extends the existing `jira_ac_field:` Schema L key to accept `description` as a sentinel value (no new canonical key).
- **F4** — `upsert_ticket_metadata` hard-codes `issuetype="Story"`. Kanban projects don't have Story (Task/Epic/Subtask only). Adapter as written rejects creation in the most common Jira project template.
- **F5** — `/setup` for `mode: jira` doesn't document that the user must create the Jira Space (Project) manually first; the MCP exposes no project-creation tool.
- **F6** — `deleteJiraIssue` is not exposed by the MCP. Adapter cannot delete; archival must use status transitions. Documentation gap.
- **F7** (high) — Spec MCP tool names are snake_case (`get_issue`, `update_issue`, `transition_issue`, `get_transitions`, `create_issue`); live names are camelCase + `Jira` infix (`getJiraIssue`, `editJiraIssue`, etc.). The adapter as written **cannot dispatch a single call correctly**.

The fix consolidates all 7 findings into one FR (STE-154); single-FR-clean structure means M43 ships v1.43.0 as a doc-and-config milestone with one minor implementation extension on `/setup`'s side.
