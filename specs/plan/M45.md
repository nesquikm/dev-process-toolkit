# M45 — Smoke #6+#7 follow-ups {#M45}

**Release target:** v1.45.0 (minor — skill-prose changes + new test fixtures; no plugin-facing API change). Codename: TBD (finalize at `/ship-milestone`).
**Status:** drafted 2026-04-30 as the post-ship follow-up to M44 / v1.44.0 (smoke runs #6 + #7).

## Why this milestone exists

M44 / v1.44.0 ("Tripwire") shipped end-to-end on 2026-04-29 after the first automated `/smoke-test --tracker jira` run (smoke #6) plus the Linear regression run (smoke #7). Both runs PASSED — full canonical chain end-to-end, retroactive validation for M43's AC-STE-154.9 steps 2–6 closed — but they surfaced a small set of skill-prose issues that don't justify amending the (already-archived) M44 milestone. Per M44.md's risk table:

> "Any new findings get an `M44-followup` label in Linear and route to a dedicated post-ship FR rather than amending M44 mid-flight."

M45 is that dedicated post-ship FR set. Two FRs:

- **STE-157 (medium)** closes smoke #6's only real-defect finding (F6): `/spec-write`'s Jira-side push retry left an orphan duplicate work item (DST-3) when the create call returned a Gateway-Timeout response while the server-side write succeeded as DST-2. The single-shot JQL idempotency probe missed the eventual-consistency lag. Hardening the retry path with backoff (1+2+4s) prevents the orphan-stub accumulation pattern from repeating on every Gateway-Timeout-affected run.
- **STE-158 (low / bundle)** rolls up four smoke-side cosmetic/operator-UX issues into one FR so M45 stays single-pass-reviewable: `/spec-write` `-p` stdout completeness (Linear F1), `/implement` `archived_at` precision (Jira F1), `/gate-check` post-archive probe-count messaging (Jira F2), and `/spec-write` filename-policy-override messaging (Jira F5).

After M45 ships:

- The orphan-duplicate failure mode is closed (STE-157), so future smoke runs against Atlassian-side intermittent timeouts no longer accumulate stub issues in DST.
- Headless `/spec-write` invocations emit a parsable summary (STE-158 AC-STE-158.1), unblocking automation use cases beyond the smoke driver itself.
- Three smaller polish items (STE-158.2, .3, .4) clear the operator-UX backlog from this round of dogfooding.
- Smoke #8 re-run (`/smoke-test --tracker jira --jira-project DST`) is the live-verification gate before /ship-milestone M45 fires.
