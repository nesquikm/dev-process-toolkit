# M86 — Jira Project-Milestone Support {#M86}

Release target: v2.31.0 (provisional, minor — new adapter capability + adapter-aware probe/helper)
Codename: TBD (decide at /ship-milestone)

## Summary

Bring Jira to parity with Linear's *used* project-milestone surface (name-grouping + verify-after-write), entirely through the atlassian MCP with no raw REST. The MCP exposes no version/milestone create-or-list tool, so the milestone M-number is mirrored onto each Jira issue as a create-on-write hyphen label `milestone-M<N>` via `editJiraIssue`, verified by `getJiraIssue` read-back. Flips the Jira adapter's long-standing `project_milestone: false` (STE-38/STE-154) to `true` and adds the Jira branches that STE-118/STE-198 deliberately left Linear-only.

Approved via /brainstorm 2026-05-26: labels are the only MCP-writable create-on-write grouping surface; milestone-number allocation stays local + git-arbitrated (labels are a downstream mirror, never an allocator — the tracker offers no atomic milestone-number primitive, unlike issue-IDs).
