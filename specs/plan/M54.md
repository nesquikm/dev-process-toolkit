## Why this milestone exists

The 2026-05-04 Dart-lib smoke run produced 16 findings spanning `/setup`, `/spec-write`, `/spec-archive`, `/implement`, `/ship-milestone`, and `templates/`. The 2026-05-04 Flutter-app smoke run added 12 more findings around `/setup` host integration (harness self-modification denial, `.mcp.json` shape rejection), Linear MCP round-trip preservation, and Phase 4 archival ordering. Brainstorm verification overturned 1 sub-agent verdict, confirmed 0 false-positives across both smokes, and grouped the surviving findings into 9 FRs by skill / file co-location:

1. **Plan-file output hygiene** (STE-197) — three compounding bugs in how `/setup` writes plan files (multi-milestone bundling, no frontmatter, leftover placeholders) cascade into `/spec-archive`/`/implement`/`/ship-milestone` failures downstream.
2. **Milestone-attach capability split** (STE-198) — `/spec-write` skips milestone attachment when the Linear project has zero milestones, even though the adapter could create one. Capability key conflates two distinct adapter states.
3. **Tracker availability + Auto Mode requires-input** (STE-199, **revised**) — both `/setup` (first-run, MCP not registered) and `/implement` (mid-session, MCP unloadable) have no graceful fallback. New `--code-only` flag + `<deferred>` binding mode + pre-flight probes. Revision adds F1/F4: Auto Mode does NOT relax `requires-input` semantics — step 7b refuses on missing pre-bake instead of default-applying.
4. **Scaffolding-milestone closure** (STE-200) — `/spec-archive` refuses on zero-FR milestones; `/implement` Phase 1/5 assume work to do; `/ship-milestone` would bump versions for placeholders. Add plan-only archival + slim closure path.
5. **`/ship-milestone` task-bullet pre-flight** (STE-201) — pre-flight walks FR rows but not `**Tasks:**` checkboxes, so a plan with unchecked task bullets passes pre-flight cleanly.
6. **Doc & workflow polish** (STE-202, **revised**) — four mechanical fixes bundled: `/simplify` in the Feature workflow, singular `specs/plan.md` prose drift, M-resolver kind, `branch_template` trunk-OK allowlist, `docs_default_applied` capability row. AC.4 (permissions trim) **superseded by STE-209** after Flutter-app brainstorm proved the original plan insufficient.
7. **Setup-time canonical-shape hygiene** (STE-209, **new** — Flutter-app) — three host/tooling rejections at `/setup` time: harness self-modification denial on `.claude/settings.json` (F3b), `.mcp.json` schema rejection (F6), flutter doctor probe wrong invocation (F2). Replaces STE-202 AC.4's empirically-disproven plan.
8. **Phase 4 archival hardening** (STE-210, **new** — Flutter-app) — `/implement → /ship-milestone` chain breaks because Phase 4 archives the plan before `/ship-milestone` reads it (F10), and frontmatter-flip ordering before `git mv` stages stale content (F11).
9. **Linear AC-token round-trip preservation** (STE-211, **new** — Flutter-app) — Linear MCP auto-link mangles `AC-STE-NNN.N` tokens into `<issue id>` XML wrappers (F7); toolkit-side backtick-wrap on emit + strip on import preserves byte-identical round-trip.

After M54 ships:

- A fresh `/setup` against any project produces plan files that pass `/gate-check` cleanly (frontmatter present, single milestone per file, no leftover placeholders).
- Linear projects with zero milestones get auto-created milestones during `/spec-write`, not silently dropped attachments.
- `/setup` no longer crashes on a project where the operator hasn't yet authenticated the configured MCP; `/implement` can run in `--code-only` mode when the MCP is mid-session unavailable.
- Auto Mode no longer rationalizes past `requires-input` prompts — step 7b refuses with a clear remedy if no `--tracker=<mode>` answer is supplied.
- Bootstrap milestones (M1 in fresh-project repos) close cleanly without manual `git mv` workarounds.
- `/ship-milestone` refuses to release a milestone whose plan still has unbacked task bullets (no false-positive ship).
- Operators following the canonical Feature workflow run `/simplify` before `/spec-review`, catching premature splits and template-name leftovers that previously required dedicated cleanup milestones.
- A fresh `/setup` writes a `.claude/settings.json` and `.mcp.json` that pass Claude Code's own validation (no harness self-modification denial; no `/doctor` rejection).
- The full chain `/implement M<N>` → `/ship-milestone M<N>` succeeds end-to-end without operator intervention; archived plan files carry correct `status: archived` + `archived_at:` frontmatter.
- Linear FR descriptions round-trip byte-identically via `importFromTracker`; idempotency probes no longer flag spurious "remote modified" diffs from the Linear MCP's auto-linker.
