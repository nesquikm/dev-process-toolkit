## M104: .dpt layout consolidation {#M104}

**Goal:** collapse the toolkit's two root-level folders — `.dpt-locks/` (tracked lock coordination
state) and `.dev-process/` (git-ignored token ledger) — into one `.dpt/` tree governed by a
toolkit-authored, committed `.dpt/.gitignore`. The folders have **opposite VCS semantics**, so "one
folder, gitignored" cannot be the shape: locks must stay tracked (cross-branch detection reads them
via `git ls-tree` against remote tips), while ledger and research scratch must stay ignored. The
real split is **tracked vs ignored**, and it cuts across both current folders — research scratch
lives in the tracked `.dpt-locks/` today, sited there for adjacency, not durability (STE-230:117).
Layout: `.dpt/locks/` tracked, `.dpt/ledger/` + `.dpt/scratch/` ignored, `.dpt/.gitignore`
committed and shipping the rules itself, so `/setup`'s ignore duty collapses from "append to the
consumer's root `.gitignore` with dedup logic" to "write one file we own". Approved 2026-07-15
`/brainstorm`.

**Two live defects the relayout exposes and fixes:**

1. **`EISDIR` namespace collision** — `.dpt-locks/<id>` is a *file* (locks) while
   `.dpt-locks/<ulid>/spec-research-result.txt` makes `<ulid>` a *directory*. `findStaleLocks`
   (`local_provider.ts:216-217`) `readFileSync`s every entry with no `isDirectory` guard and no
   try/catch. Mechanism reproduced 2026-07-15; latent (mode-none only) rather than live.
2. **Hard-coded pathspec** — `findRemoteBranchWithLock` (`local_provider.ts:160`) hard-codes
   `.dpt-locks/${id}` while write (`:107`) and release (`:167`) derive from `this.locksDir`.
   Harmless only by coincidence today; after the move it would scan a folder that no longer exists
   and silently answer "unclaimed" — the exact double-claim failure this milestone prevents.

**Ignore polarity is a deliberate choice, verified empirically** (2026-07-15, `git check-ignore`
per path): the naive `.dpt/` + `!.dpt/locks/` form **silently unversions locks** (git cannot
re-include a path whose parent directory is excluded). Tracked-by-default fails loud and harmless
(scratch leaks into a commit); ignore-and-negate fails silent and correctness-breaking. M104 takes
the loud-failure polarity.

**Prerequisites:** v2.45.0 shipped (main @ PR #38). Load-bearing existing machinery: STE-28
(tracked-lock semantics, `claimLock`/`releaseLock`/stale scan), STE-344 (`ledgerPath()` + the
AC-STE-344.5 root append being retired), STE-230 / STE-301 (research-result persistence + probes
#41/#64 and their vacuity contract), STE-303 (idempotent toolkit-owned-file write), STE-133

installs ⇒ forward-only; no migration rails.
