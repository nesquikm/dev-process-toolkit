# M61 — Universal pre-commit branch gate (trunk-OK narrowed to ci) {#M61}

**Release target:** v2.11.0 (minor — additive contract change, behavior break is intentional and unmitigated per `project_no_users_yet`).
**Codename:** TBD at `/ship-milestone` time.

**Status:** drafted 2026-05-05 from `/brainstorm` session 2026-05-05 (Approach 1 selected: lift `/implement`'s branch-proposal helper into a shared pre-commit gate, narrow trunk-OK to `ci`, per-skill `branchNameFor` builders, hardcoded `["main", "master"]` protected list, collision-suffix probe).

## Why this milestone exists

Branch protection on `main` (the toolkit's repo, and typical user repos) rejects pushes of commits made directly on the protected branch. Today:

- `/spec-write` produces `chore(specs): write FR <id>` and `docs(specs): edit cross-cutting specs` commits, both of which land on `main` because `/implement` step 0.b″ documents `chore` and `docs` as trunk-OK (STE-202 AC-STE-202.5). The local commit succeeds; the subsequent push is rejected by branch protection.
- `/setup`, `/spec-archive`, and `/ship-milestone` have **no branch logic at all**. They commit on whatever branch the operator happens to be on; if that's `main`, same push-rejection wall.
- `/implement` already has branch logic at step 0.b″ but its trunk-OK list `[chore, docs, ci]` is the wrong shape for protected-trunk repos.

The recent commits on this repo (`ad8ae84 chore(specs): write FR STE-227`, `d56910c chore(specs): write FR STE-226`, `ca18140 chore(specs): write FR STE-225`) all sit directly on `main`. They're locally fine but cannot be pushed.

After M61 ships:

- Every commit-producing skill (`/setup`, `/spec-write`, `/spec-archive`, `/ship-milestone`, `/implement`) calls a shared `requireCommittableBranch(opts)` gate before staging.
- The gate refuses to commit on `main` / `master` unless the Conventional Commits type is `ci` (the narrow trunk-OK list).
- When the gate fires, the skill proposes a branch name (per-skill `branchNameFor(...)` builder), prompts `[Y] create / [e] edit / [n] abort`, and runs `git checkout -b` on `Y`.
- Concurrent runs of literal-named skills coexist via collision-suffix probe (`-2`, `-3`, …).
- The `feedback_branch_isolation` memory rule (stay on current branch for `/implement`) composes cleanly — off-trunk runs are a silent no-op.
