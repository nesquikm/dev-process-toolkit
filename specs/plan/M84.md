## Summary

Three user-hostile-drift fixes surfaced by the 2026-05-20 repo review (`/tmp/dpt-review-findings-2026-05-20.md`, 41 verified survivors / 30 problem areas). M84 closes the three HIGH-severity items most likely to trip a new operator within their first day of the toolkit:

- **STE-315 (A1)** — Add `/gate-check` probe #57 `public_surface_count_drift` and backfill the seven stale-count surfaces (README.md L3 / L10 / L93 / L126 / L150-151 + CLAUDE.md L15-16 + docs/skill-anatomy.md L54 + skills/gate-check/SKILL.md L214 + specs/testing-spec.md L58). One probe + one backfill commit closes the largest documented drift class in the toolkit and prevents the same drift from re-introducing.
- **STE-316 (A4)** — Rewrite five tracker-mode docs (`ac-sync.md`, `implement-tracker-mode.md`, `spec-review-tracker-mode.md`, `spec-write-tracker-mode.md`, `layout-reference.md`) plus `setup-tracker-mode.md` to teach the canonical v2 file-per-FR layout. The current text still teaches the v1 `### FR-N` blocks in `specs/requirements.md`, which probe #29 (`requirements-md-no-placeholder`) hard-rejects on the next gate-check. User-hostile failure mode — a new operator follows the docs and lands code that the gate rejects.
- **STE-317 (A8)** — Replace obsolete `Bash(<cmd> *)` glob shapes in the four `examples/<stack>/*.md` permission blocks with explicit-subcommand allowlists matching `templates/permissions.json`. Same F3b harness self-modification denial STE-209 (M54) closed at the canonical surface; example files were missed in M54 scope. User-hostile failure mode — a new operator copy-pastes the example and the harness denies the first permission grant.
