# M21 — De-ULID tracker mode {#M21}

**Release target:** TBD (after v1.23.0 / M20). Codename TBD.
**Status:** seed placeholder (single FR captured; awaiting full `/spec-write`).
**Origin:** identified 2026-04-23 during M20 spec-write. User observed that `id: fr_<ULID>` frontmatter in tracker-mode FRs is ceremonial — the filename and AC prefix are the tracker ID, and STE-67 (M19) made ULIDs invisible in user-facing prose. The ULID only mattered as a hedge against tracker-workspace renames, which `findFRByTrackerRef` already handles.

## Why this milestone exists

Tracker-mode ULIDs are ceremony without corresponding value:
- Filename is the tracker ID (post-M18 STE-60).
- AC prefix is the tracker ID (post-M16 FR Identity Stability).
- User-facing prose has no ULIDs (post-M19 STE-67).
- Tracker-workspace renames are handled by `findFRByTrackerRef` via the `tracker:` map.

Removing the ceremony simplifies the mental model: "the Linear ticket is the identity in tracker mode". Mode-none keeps short-ULIDs unchanged (it has no tracker-side identity to defer to).

## Dependencies

**Order: M22 → M20 → M21.**

- **M22 must ship before M21 kicks off.** M22's STE-80 (template `<tracker-id>` convention in `requirements.md.template` + `plan.md.template`) installs seeded examples that STE-76's Schema Q revision will touch — STE-76 makes `id:` mode-conditional (required in mode-none, absent in tracker-mode) and the template examples must mirror the revised invariant. Cleanest if M22's seed lands first, then STE-76 adjusts from a known baseline.
- **M22's STE-82 probe authoring contract** also affects STE-76 AC-STE-76.6 (gate-check assertion: `id:` absent from tracker-mode active FRs). Per the contract, that probe needs `tests/gate-check-<slug>.test.ts`; STE-76's full `/spec-write` pass must include this AC explicitly.
- **M20 shipped first (v1.23.0).** Its docs generation settled a stable FR set before M21's archive migration rewrites 91+ files — avoided cross-milestone churn. No hard blocker; scheduling preference.

## Out of scope

- Mode-none ULID changes — mode-none keeps short-ULID filename + AC prefix convention.
- Git-history rewrite of ULID references in old commits / CHANGELOG entries — historical records remain intact.
- Changes to the `tracker:` frontmatter map — that stays as the cross-ref mechanism.
