# M44 — /smoke-test Jira branch {#M44}

**Release target:** v1.44.0 (minor — driver-side `.claude/skills/smoke-test/SKILL.md` extension; no plugin-facing change). Codename: TBD (finalize at `/ship-milestone`).
**Status:** drafted 2026-04-29 as a follow-up to M43. Sequencing: M43 ships v1.43.0 first; M44 builds on the corrected adapter.

## Why this milestone exists

M43 fixes the Jira adapter spec, but `/smoke-test` (the dogfood driver this repo uses for Tier 5 conformance) still hard-codes `mode: linear`. After M43 ships, Jira regressions could re-creep in without an automated tripwire. M44 productizes the manual smoke pattern from M43's AC-STE-154.9 into a `--tracker jira` branch on the existing skill so:

- Both adapter modes (linear, jira) get the same automated conformance pass.
- Future Jira-side regressions are caught by re-running `/smoke-test` rather than by ad-hoc manual passes.
- The shape of the canonical chain (`/setup → /spec-write → /implement → /gate-check → /spec-review → /simplify`) is identical in both modes; only Phase 1 (project setup) and Phase 5 (teardown) branch.

The asymmetry vs Linear is real but bounded: Atlassian MCP exposes no `createJiraProject` / `deleteJiraIssue`. Jira runs therefore reuse a pre-existing throwaway Space (`DST` for the stellartechlab site) and clean up by transitioning labeled work items to `Done` rather than archiving a project. Documented as a limitation, not a bug.

## Risks

| Category | Risk | Severity | Mitigation |
|----------|------|----------|------------|
| External dependency | Atlassian Rovo MCP `tools/list` shape may shift between OAuth refreshes (e.g., `transitionJiraIssue` rename) | low | Provenance line in `adapters/jira.md` (M43 AC-STE-154.8) documents verification date. Smoke runs detect drift the next time they run; the skill's NFR-10-shape refusals on missing tools surface failures cleanly. |
| OAuth-token leakage | Child `claude-st -p` session needs Atlassian credentials; risk of writing token to `.mcp.json` | low | Driver writes only the `mcpServers:` envelope (URL, transport type) — no auth material. OAuth state lives in `~/.claude-st/` and is inherited by the child process at startup. The token never lands on disk under the test project. |
| DST Space pollution | Aborted runs leave orphaned `dpt-smoke`-labeled work items in DST | low | Phase 5 cleanup is idempotent — the next run's teardown picks up orphans (JQL filter is by label + creation window; expanding the window costs nothing). | Sequencing | M44 depends on M43; running M44 against pre-M43 main breaks the Jira chain (wrong tool names) | medium | Document M43-as-prerequisite at the top of STE-155. M44 plan ships only after M43's v1.43.0 cuts. CHANGELOG entry for v1.44.0 explicitly notes the dependency. |
| Back-compat | Existing operators muscle-memoried to `/smoke-test` (no flags) get the Linear path automatically; no new behavior | low | Default `--tracker linear` preserves byte-for-byte parity with pre-M44 runs. The first AC enforces this with a regression run as part of the gate. |
