# M59 — Auto-mode/`-p` carve-out: deterministic pre-authorization marker {#M59}

**Release target:** v2.11.0 (minor — additive at the user-facing surface; replaces the prose-only Auto-mode detection contract inside `/spec-write`. No users → rip-and-replace per memory `project_no_users_yet`.)
**Codename:** TBD at `/ship-milestone` time.

**Status:** drafted 2026-05-05 from `/conformance-loop` iter-1's F1 + M1 findings (STE-220's third attempt; the first two prose-only fixes both falsified end-to-end across four smoke runs).

## Why this milestone exists

`/spec-write`'s Auto-mode/`-p` carve-out has shipped twice as prose-only changes — STE-213 (M55) and STE-220 (M56) — and has been falsified twice by end-to-end smoke runs (2026-05-04 v2.8.0 Linear F2-1 + Jira C-F1; 2026-05-05 v2.10.0 `/conformance-loop` iter-1 F1 + M1). The architectural lesson: instructional SKILL.md prose is not a reliable substitute for an explicit byte-checkable contract when the LLM is processing the same prose to decide whether to apply it. Two prose-only attempts have failed; the smoke-test driver workaround that empirically succeeded in iter-1 ("inject explicit pre-authorization marker into prompt body") is the deterministic mechanism this milestone formalizes.

After M59 ships:

- `/spec-write` under `claude -p` with a canonical marker line in the prompt body proceeds through both gates without further prompting and emits the documented audit rows.
- `/spec-write` under `claude -p` without the marker gates interactively (no auto-apply). The legacy `Auto Mode Active` system-reminder detection path is removed — marker is the single mechanism.
- `/smoke-test` and `/conformance-loop` (project-local) inject the marker line into every documented `claude -p` spawn that targets a skill carrying a gate.
- `/gate-check` carries a new probe `auto_approve_marker_in_canonical_spawns` that hard-fails when a SKILL.md `claude -p` spawn block lacks the marker.
- `/smoke-test` Phase 2.X group 1 (STE-220) extends to two sub-fixtures (1a marker-present → audit rows; 1b marker-absent → stdout halts at gate).
