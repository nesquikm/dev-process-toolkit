## Technical Design

Three independent prose edits + one new gate-check probe.

Prose edits (mechanical replacements per AC.1+AC.2+AC.3): each surface gets a targeted Edit. AC-STE-318.3 picks branch (a) — drop the false probe claim. The replacement reads: "Output shape is operator-judgment, not runtime-enforced — the architectural twin asymmetry with `spec-researcher.md` is intentional (deps-research output shape carries no byte-checkable contract because dependency-package signatures vary across stack ecosystems)."

Probe module at `adapters/_shared/src/cross_skill_contract_drift.ts`: exports `runCrossSkillContractDriftProbe(projectRoot: string): ProbeResult` matching the canonical signature of probes #55–#57. Walks file glob `plugins/dev-process-toolkit/{skills,docs,agents,README.md}` and scans each for the forbidden substring list:

```typescript
const FORBIDDEN_SUBSTRINGS: readonly string[] = [
  "three forked subagents",
  "three forked TDD subagents",
  "three forked-subagent stages",
  "forks three subagents",
  "RED → GREEN → REFACTOR for one FR via three",
  "RED → GREEN → VERIFY",
  "3-tier ticket-binding",
  "3-tier resolver",
  "deps_research_result_shape",
];
```

Any active-surface match is a violation — NFR-10 canonical shape (`<file>:<line>:<column> — <reason>` with `Refusing: / Remedy: / Context:` sub-lines).

Architectural twin: probe shape mirrors STE-263 / STE-270 / STE-262 alternate-trigger scan probes (#47, #48). Literal substring match (no regex) so future SKILL.md copy edits don't accidentally regress detection.

## Testing
