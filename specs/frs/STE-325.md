## Requirement

Patch `.claude/skills/conformance-loop/SKILL.md`'s Phase A spawn snippets so the slash command and its arguments are wrapped in a single quoted positional prompt, matching STE-185's canonical child-spawn discipline and the form already used at Phase B `/implement` (line 298). The current bare-arg shape — `claude -p /smoke-test --tracker linear ...` — is rejected by the `claude` CLI as `error: unknown option '--tracker'` and has been ship-broken since STE-224 (M57) authored `/conformance-loop` in commit `a33d935` (2026-05-05). The `conformance-loop-dry-run.test.ts` integration test mocks the subprocess and cannot catch this class of regression, so the bug went undetected for ~16 days until a live `/conformance-loop --jira-project DST` run hit it on 2026-05-21.

Scope is deliberately narrow per operator selection: spec fix only, no new test, no new gate-check probe. Out of scope: real-CLI smoke probe for spawn-shape regressions (rejected as scope creep), /gate-check static probe for bare-arg form (deferred).

## Technical Design

Two-line edit to `.claude/skills/conformance-loop/SKILL.md`.

**Line 151 (Phase A — Linear)** — before:

```bash
CLAUDE_CONFIG_DIR=~/.claude-st claude -p /smoke-test --tracker linear --linear-team "${LINEAR_TEAM:-STE}" \
    --plugin-dir "${PLUGIN_DIR}" \
    > "${LOG_LINEAR}" 2>&1 <<'PROMPT_EOF'
```

After:

```bash
CLAUDE_CONFIG_DIR=~/.claude-st claude -p "/smoke-test --tracker linear --linear-team ${LINEAR_TEAM:-STE}" \
    --plugin-dir "${PLUGIN_DIR}" \
    > "${LOG_LINEAR}" 2>&1 <<'PROMPT_EOF'
```

**Line 160 (Phase A — Jira)** — same shape (`claude -p "/smoke-test --tracker jira --jira-project ${JIRA_PROJECT}"`).

Reference: STE-185 (M52) canonical form is `claude -p <command> [args] <<'PROMPT_EOF' … PROMPT_EOF` where `<command>` is the quoted single positional carrying slash-cmd + skill-args, and `[args]` carries `claude` CLI flags only. STE-224 mis-substituted `<command>` = `/smoke-test` (unquoted), spilling skill-args into `[args]`. Phase B `/implement` at line 298 already uses the correct form and serves as the in-file reference. No helpers, adapters, tests, or non-SKILL.md files are touched.
