# STE-314: README Commands table — add Args column and strip namespace prefix {#STE-314}

## Requirement

Document each user-callable skill's argument shape in the README Commands table by adding a third column (`Args`), populated verbatim from each skill's `argument-hint:` YAML frontmatter. Strip the `/dev-process-toolkit:` namespace prefix from the first column to match the bare-name convention already used in the Agents table directly below (STE-130 precedent).

Rationale: Claude Code does not yet render `argument-hint` in autocomplete for plugin-namespaced skills (upstream issue [#43401](https://github.com/anthropics/claude-code/issues/43401), closed 2026-04-07 as duplicate but the feature gap remains as of v2.27.0). Users typing `/dev-process-toolkit:implement ` see no inline hint — in contrast to ralph-loop (`commands/*.md` format) and project-level skills like `/conformance-loop`, both of which render their hints. The README Commands table becomes the primary arg-discovery surface until upstream closes the gap.

Forward-compat: all SKILL.md `argument-hint:` frontmatter is left untouched. When upstream lands the rendering fix, autocomplete will start working automatically — no second migration needed.

## Acceptance Criteria

- AC-STE-314.1: `README.md` Commands table has exactly three columns (`Command`, `Purpose`, `Args`) and one row per user-callable skill (16 rows). The `Args` cell for each row is byte-identical to the `argument-hint:` value in that skill's `plugins/dev-process-toolkit/skills/<name>/SKILL.md` frontmatter, with the outer YAML quotes stripped.
- AC-STE-314.2: The `Command`-column entries are bare-prefixed `/setup`, `/brainstorm`, `/spec-write`, `/implement`, `/tdd`, `/gate-check`, `/debug`, `/spec-review`, `/spec-archive`, `/visual-check`, `/docs`, `/pr`, `/simplify`, `/report-issue`, `/ship-milestone`, `/deps` — no `dev-process-toolkit:` substring appears in the column (matches the Agents-table convention from STE-130).
- AC-STE-314.3: A one-line note immediately preceding the Commands table explains that commands are typed with the `dev-process-toolkit:` plugin-namespace prefix in actual use, omitted from the table for readability. The note references upstream issue #43401 by URL.
- AC-STE-314.4: No file under `plugins/dev-process-toolkit/skills/`, `plugins/dev-process-toolkit/agents/`, or `plugins/dev-process-toolkit/.claude-plugin/` is modified by this FR — the diff touches `README.md` exclusively.

## Technical Design

Single-file edit to `README.md` lines 103–124 (the `### Commands` heading + table + child-skills footnote). Source data: each user-callable skill's `argument-hint:` field, read from `plugins/dev-process-toolkit/skills/<name>/SKILL.md` frontmatter. Preserve the source quoting characters (angle brackets, square brackets, pipes) so the column matches the source string byte-for-byte after stripping the outer YAML quotes.

Sibling-package retrieval: vacuous (`specs/deps.yaml` absent).

## Testing

Manual verification — render `README.md` and confirm: 3 columns × 16 user-callable rows; each `Args` cell byte-identical to the source skill's `argument-hint:`; no `dev-process-toolkit:` substring in the `Command` column; pre-table note explains the dropped prefix and links upstream issue #43401.

No automated test coverage — documentation-only change with no executable surface. A future `/gate-check` probe could enforce row-to-frontmatter sync (precedent: AC-STE-91.4 cross-consistency, AC-STE-130.1 structural-element assertion). Deferred per minimal-scope directive.
