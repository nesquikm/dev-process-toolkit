## Requirement

The smoke driver must never end its session (or report results) while grandchildren are in flight or truncated. Both legs of the 2026-07-02 conformance run truncated silently (findings F2 medium + F3 high): the `/implement` grandchild was SIGTERM'd at the harness's 10-minute foreground Bash ceiling (completed RED→GREEN→REFACTOR, never reached AUDIT/commit), and both `claude -p` /smoke-test children fired grandchild spawns in the background then exited RC 0 with "waiting for its completion notification" — a `-p` session cannot resume on background-task notifications, so Phases 2.X/3/4/5/8/9 never ran on either leg.

## Acceptance Criteria

- AC-STE-355.1: Every canonical-chain grandchild spawn uses a detached-spawn + bounded-poll-until-exit wrapper (spawn with PID capture, then repeated short poll calls until exit) — no single foreground Bash call caps a grandchild at the harness 10-minute ceiling, and the driver is forbidden from ending its session while any spawned grandchild is alive; a meta-test pins the wrapper shape.
- AC-STE-355.2: End-of-run chain-integrity assertion: before Phase 3 capture, the driver asserts every expected per-skill capture exists, is non-empty, and carries a stream-json `result` event (reusing `smoke_child_capture` / `stream_json_events`); any miss is a high-severity finding naming the truncated child, and the run is barred from a green summary; the helper is unit-tested (0-byte / result-absent / healthy fixtures).
- AC-STE-355.3: `/conformance-loop` Phase A mirror: a child leg whose expected grandchild log set is incomplete (or whose final message matches the fire-and-exit shape) is treated as a failed leg regardless of RC 0; a meta-test pins the prose.

## Technical Design

**Spawn wrapper (`.claude/skills/smoke-test/SKILL.md` § Phase 2).** Replace the foreground per-skill spawn form with a detached spawn + poll loop: `claude -p … > log 2>&1 & echo $! > /tmp/dpt-smoke-<tracker>-<skill>.pid` in one Bash call, then repeated bounded poll calls (`kill -0 $(cat pidfile)` + `sleep 30`, each call well under the 10-minute ceiling) until the PID exits. The STE-195 stream-idle retry wrapper composes on top (detection runs after exit, unchanged). The driver prose gains a hard rule: ending the session with a live pidfile is forbidden — the poll loop is the only sanctioned wait.

**Chain-integrity assertion.** New helper `assertChainIntegrity(expectedLogPaths)` in `adapters/_shared/src/smoke_child_capture.ts` (or sibling): for each expected per-skill log, assert exists + non-empty + `parseStreamJsonEvents` finds a `type: "result"` event. Returns findings in the `ChildSpawnFinding` shape with diagnostic `STE-355 regression: chain truncated — <child> (result event absent)`. `/smoke-test` runs it as a new step between Phase 2.X and Phase 3; any finding forces the run summary to FAIL.

**Conformance-loop mirror (`.claude/skills/conformance-loop/SKILL.md` § Phase A).** After both children return, Phase A verifies each leg's expected grandchild log set (`/tmp/dpt-smoke-<tracker>-{setup,spec-write,implement,gate-check,spec-review,simplify}.log`) is complete and result-bearing before aggregation; an incomplete leg aborts the iteration with the same fail-fast path as a non-zero RC.

**Phase A spawn mechanics (backfilled during implementation — underspecified in the original draft).** The mirror alone is insufficient: Phase A's own reference snippet spawned both `/smoke-test` children and foreground-`wait`ed them inside one Bash call, so once STE-355's poll wrapper makes children genuinely wait for their grandchildren (~10+ min per leg), the parent's single-call `wait` hits the same 600s harness ceiling F2 identified. Phase A therefore uses the same detached-spawn + pidfile + bounded-poll discipline as the smoke driver's Phase 2: both children spawn detached in one Bash call with PID capture to `/tmp/dpt-conformance-loop-<date>-iter-<N>-{linear,jira}.pid`, then repeated short poll calls (`kill -0` + `sleep`, each call well under the ceiling) until both exit; ending the session with a live pidfile is forbidden. A meta-test pins the Phase A wrapper shape (pidfile literals + poll loop + no foreground `wait`-in-spawn-call).

## Testing

- Meta-test: Phase 2 spawn fences carry the pidfile + poll-loop shape; no bare foreground `claude -p` canonical-chain spawn remains; the no-live-pidfile session rule is present (AC-STE-355.1).
- Meta-test: conformance-loop Phase A carries the leg-completeness check (AC-STE-355.3).
- Meta-test: conformance-loop Phase A spawns its two `/smoke-test` children detached with pidfile capture + bounded poll loop — no foreground `wait` in the spawn call (backfilled Phase A spawn mechanics, same 600s-ceiling rationale as AC-STE-355.1).
- Runtime (smoke-validated on the next run): full canonical chain completes on both legs; a deliberately killed grandchild surfaces as a truncation finding, not a green run.
