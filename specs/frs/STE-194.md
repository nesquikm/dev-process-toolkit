# STE-194: gate-check probe #26 milestone-attach respects FR ## Notes capability-gap declaration {#STE-194}

## Requirement

Probe #26 (`tracker-project-milestone-attached`) currently fires GATE FAILED whenever a tracker-mode FR has no `projectMilestone` binding. In Linear-mode smoke runs, the test project starts with zero milestones by construction (the smoke driver does not create Linear project milestones), so probe #26 false-positive-fails on every Linear smoke run despite the chain being otherwise clean. `/spec-write` already declares the capability gap in the FR's `## Notes` section via the canonical `milestone_attach_unavailable` capability key (per its capability-key map in `skills/spec-write/SKILL.md` § Step 7). Probe #26 should consult that declaration and downgrade GATE FAILED → ADVISORY when present.

Out of scope:

- Auto-creating Linear project milestones on the smoke driver's behalf (the alternative fix surfaced in the smoke findings). Out-of-band: the milestone-creation policy is a smoke-test-driver concern, not a gate-check concern. `/spec-write`'s declaration of the gap in `## Notes` is the authoritative signal — probe #26 just needs to read it.
- Generalizing the pattern to other capability keys. This FR addresses only `milestone_attach_unavailable` (the one capability gap that probe #26 currently checks). Other probe/capability pairs land separately if needed.
- Jira-side equivalent. Jira adapters declare `project_milestone: false` in Schema M frontmatter, so probe #26 is vacuous there by construction.

## Acceptance Criteria

- AC-STE-194.1: Probe #26 reads the FR's `## Notes` section and detects the literal `milestone_attach_unavailable` token (case-sensitive, word-bounded — substring match inside a longer identifier does NOT count). {#AC-STE-194.1}
- AC-STE-194.2: When the token is present in `## Notes` AND the FR has no `projectMilestone` binding on the tracker side, probe #26 emits ADVISORY (not GATE FAILED) with the canonical prose `milestone-attach skipped — capability gap declared in FR Notes (milestone_attach_unavailable)`. {#AC-STE-194.2}
- AC-STE-194.3: When the token is absent from `## Notes` AND the FR has no `projectMilestone` binding, probe #26 still emits GATE FAILED (no regression on the genuinely-unbacked case — the gate must continue to fire for FRs that should have been attached). {#AC-STE-194.3}
- AC-STE-194.4: Test fixture: a Linear-mode FR with `milestone_attach_unavailable` in `## Notes` and no `projectMilestone` → probe #26 → ADVISORY with the canonical prose. {#AC-STE-194.4}
- AC-STE-194.5: Test fixture: same FR shape without the token → probe #26 → GATE FAILED (regression guard against accidentally always-downgrading). {#AC-STE-194.5}

### Token detection rule

- Word-bounded grep: `\bmilestone_attach_unavailable\b` (or equivalent) inside the FR's `## Notes` body. Matching anywhere else in the file does not count — the token must be in `## Notes` to count as a declaration.
- Case-sensitive. The capability-key map in `/spec-write` emits the lowercase form; matching is exact.
