## Requirement

Kotlin is a second-class stack today: it exists only as a single row in
`docs/adaptation-guide.md` (`Java/Kotlin ŌåÆ ./gradlew compileJava | spotlessCheck
| test | build`), with **no** `examples/<stack>/` config, **no** `/setup`
auto-detection, **no** `templates/permissions.json` key, and **no**
scaffold/gate coverage. Flutter/Dart, Python, and TypeScript/Node are
first-class ŌĆö each carries an `examples/<stack>/` dir (`gate-commands.md` +
`release.yml`), a `permissions.json` `stacks.<stack>` key, `/setup` detection,
and scaffold guidance.

This FR promotes Kotlin to first-class parity, targeting **generic Gradle/JVM**
Kotlin projects ŌĆö the broadest flavor (Android, Kotlin Multiplatform, and
server-side Ktor/Spring are explicitly out of scope; raise separate FRs if a
real project needs them). The lint gate standardizes on **detekt**
(`./gradlew detekt`), the de-facto Kotlin static analyzer, rather than the
adaptation-guide's current `spotlessCheck`. Gate set:
`./gradlew compileKotlin` ŌåÆ `./gradlew detekt` ŌåÆ `./gradlew test` ŌåÆ
`./gradlew build`.

## Technical Design

**New example dir.** `examples/kotlin/gate-commands.md` modeled on
`examples/flutter-dart/gate-commands.md` (sections: Gate Check Commands, TDD
Patterns, Key Conventions, Settings Example). `examples/kotlin/release.yml`
modeled on `examples/typescript-node/release.yml`, swapping the JS
`package.json` version field for a `gradle.properties` `version=` regex:

```yaml
files:
  - path: gradle.properties
    kind: regex
    pattern: '^version=(?<version>\d+\.\d+\.\d+)'
    replace: 'version={version}'
  - path: CHANGELOG.md
    kind: changelog
  - path: README.md
    kind: regex
    pattern: 'Latest: \*\*v(?<version>\d+\.\d+\.\d+) ŌĆö '
    replace: 'Latest: **v{version} ŌĆö '
    optional: true
```

**Permissions.** One `stacks.kotlin` row in `templates/permissions.json` with
the five explicit subcommands. `merge_settings.ts` `canonicalAllowList` is
data-driven from this file, so adding the key alone wires it ŌĆö no code change,
verified by a `merge_settings` test (AC-STE-336.6).

**`/setup` edits** at the four cited anchors (detection probe :53, doctor
prefix :65, scaffold bullet :95 / verify branch :103, permissions read :137),
plus a `docs/setup-reference.md ┬¦ Kotlin detekt scaffold-verify branch` note
mirroring the existing Bun branch.

**Gate probe.** `kotlin_detekt_configured.ts` follows the STE-82 flat-path +
STE-113 stack-conditional pattern: detect Gradle Kotlin (lockfile-equivalent =
`build.gradle.kts` / `settings.gradle.kts`), read whether the gate-commands
declare `detekt`, then grep `build.gradle.kts` for `io.gitlab.arturbosch.detekt`
or a `detekt {` block. Vacuous-pass otherwise.

  keeps `spotlessCheck` so Java tooling is unaffected.
