## Requirement

`/conformance-loop` iteration 1 (2026-05-07, capture file `/tmp/dpt-conformance-loop-2026-05-07-iter-1.md`) surfaced a **systemic gap** flagged across three high-severity findings (F1+F3+F4) and one low-severity finding (F9), plus a small skill doc-drift (F8 doc-drift half). The behavioral contracts of STE-226 (marker default-apply), STE-228 (universal branch gate), and STE-230 (spec-research subagent) all fire correctly at runtime — markers trigger auto-apply, branches get created when applicable, spec-research forks return blocks. **The byte-checkable capability-key tokens those contracts specify (`spec_write_draft_default_applied`, `spec_write_commit_default_applied`, `branch_gate_default_applied`, `spec_research_invoked` / `_no_matches` / `_shape_violation`) are absent from runtime stdout** — the static plain-language map at `/spec-write` § 7 documents them, but the LLM running the skill emits narrative prose, not the literal tokens. Downstream `/gate-check` probes that grep for these tokens cannot verify the contract. STE-220 lesson re-applies: prose-only enforcement drifts; byte-checkable structural verification is required. Plus two micro-cleanups: F9 — `/gate-check` probe `cross-cutting-spec-stale-file-refs` (current probe #37) skip-reason text fires "require Linear MCP" even when the MCP is loaded (diagnostic drift); F8 doc-drift — `/smoke-test` SKILL.md says probe `#26`, runtime emits `#37`.

## Acceptance Criteria

- AC-STE-238.1: `/spec-write` closing-summary block MUST emit literal capability-key tokens for every behavior that fired during the run, rendered verbatim from the static plain-language map at `/spec-write` § 7. Specifically: when the FR-draft gate (§ 0b step 4) marker auto-applies, the closing summary MUST contain the literal token `spec_write_draft_default_applied` (not paraphrased); same for commit gate (§ 7a) → `spec_write_commit_default_applied`. SKILL.md prose at § 4 and § 7a is strengthened with an explicit "MUST emit literal token, not paraphrase" directive at each emission site. F1 closure.

- AC-STE-238.2: `/spec-write` closing-summary block MUST emit `branch_gate_*` literal tokens for every branch-gate outcome path. Marker-driven auto-apply with branch creation → `branch_gate_default_applied`. Gate short-circuits because operator was already off-trunk → NEW token `branch_gate_skipped_already_non_main` (added to the static map under AC.6). Gate creates branch interactively → existing `branch_gate_created`. Edited proposal accepted → existing `branch_gate_edited`. Declined → existing `branch_gate_declined`. F3 closure.

- AC-STE-238.3: `/spec-write` closing-summary block MUST emit exactly one of `spec_research_invoked` / `spec_research_no_matches` / `spec_research_shape_violation` literal tokens whenever the spec-research forked subagent fires (per § 0b spec-research seed and § 1.5 of `/brainstorm`). SKILL.md prose at the seed-invocation site is strengthened to require literal token emission, not narrative paraphrase. F4 closure.

- AC-STE-238.4: New `/gate-check` probe `closing_summary_capability_keys`. Globs `plugins/dev-process-toolkit/skills/{spec-write,gate-check,smoke-test}/SKILL.md` and `.claude/skills/{spec-write,gate-check,smoke-test}/SKILL.md`. For every capability-key listed in the static plain-language map at `/spec-write` § 7, asserts the skill body containing the corresponding emission site carries a literal "MUST emit `<key>`" directive (regex match on the documented directive shape). Hard-fails the gate; surfaces `closing_summary_capability_keys_violation` rows naming the missing key + skill. Modeled on STE-232:AC.5 / STE-228 probe shape. Catches future drift.

- AC-STE-238.5: New `/smoke-test` Phase 9 — `Phase 9 — Capability-Row Emission Verification`. Runs three lenient-assertion fixtures (per STE-231:AC.3 shape — "at least one expected key for the scenario MUST appear in stdout"):

- AC-STE-238.7: `/smoke-test` SKILL.md prose doc-drift cleanup — replace literal `probe #26` references with `probe #37` where they apply to the `cross-cutting-spec-stale-file-refs` probe (the runtime emits #37; the prose says #26). Text-only edit; no behavior change. F8 doc-drift half closure.

- AC-STE-238.8: `/gate-check` probe `cross-cutting-spec-stale-file-refs` (current probe #37) skip-reason text correction. Diagnostic-text accuracy fix only; no behavior change. Unit test on the rendering helper. F9 closure.

## Technical Design

`/conformance-loop` iteration 1 caught the STE-220 recurrence pattern again: SKILL.md prose specifies byte-checkable rows; the runtime emits narrative prose; downstream probes can't verify. STE-237 (M66, sibling FR) addresses the structural-enforcement-of-Socratic-loops side; this FR addresses the structural-enforcement-of-capability-row-emission side. Same milestone theme: convert prose-described contracts into byte-checkable structured outputs.

| Layer | Mechanism | What it catches |
|---|---|---|
| Source-level | SKILL.md prose strengthened with "MUST emit literal `<key>` token" directives at every documented emission site | LLM paraphrasing the row instead of emitting the literal key |
| Static map | Single source of truth at `/spec-write` § 7; AC.6 adds the missing `branch_gate_skipped_already_non_main` row | Drift between skill bodies; new keys spawned ad-hoc |
| Source probe | `/gate-check` `closing_summary_capability_keys` (AC.4) | Skill body that omits emission directive for a documented key |
| Behavioral fixture | `/smoke-test` Phase 9 (AC.5) lenient assertion | LLM regression: directive present in source but ignored at runtime |
| Doc cleanups | AC.7 + AC.8 | Documented for completeness; not load-bearing on the systemic fix |

Maps to STE-232 / STE-237 / STE-228 / STE-230 by analogy:

| Layer | Per-step gate (STE-232) | Per-conversation loop (STE-237) | Capability-row emit (this FR) |
|---|---|---|---|
| Token / structural primitive | `<dpt:auto-approve>v1</dpt:auto-approve>` marker | `AskUserQuestion` tool call | static plain-language map @ § 7 |
| Refusal / verification site | `requireOrRefuse(spec, key, sentinel)` | first-turn contract + `assertFirstTurnShape` | "MUST emit `<key>`" prose directive |
| Source probe | `requires_input_sentinel_coverage` | `socratic_loop_uses_ask_user_question` | `closing_summary_capability_keys` (AC.4) |
| Audit column / signal | `imputed: true|false` | `loop_entered: true|false` | literal token in closing summary |
| Behavioral fixture | (none) | `/smoke-test` Phase 8 | `/smoke-test` Phase 9 (AC.5) |
