# STE-295: Smoke fixture group — M70 post-iter-1 cohort follow-ups {#STE-295}

## Requirement

Bundle five surviving smoke fixtures from `/conformance-loop` iter-1 (2026-05-14) into one fixture-group FR under the STE-231 / STE-286 precedent. Each AC carries the canonical diagnostic shape `STE-<sut> runtime regression: <fixture>` (per STE-231 AC-5) and bundles a small SUT change with a byte-checkable smoke fixture. Smoke ACs may ship `[~]` deferred per `feedback_smoke_post_ship_retroactive` — the next `/conformance-loop` iteration validates retroactively.

This FR pairs with STE-294 (separately, same milestone) which closes the F1 STE-226 contract drift. Together they cover the high + medium-severity findings from iter-1; F8 / F9 / F10 are deferred separately.

3. `.claude/skills/smoke-test/SKILL.md` — Phase 9 master-merge step (AC-STE-295.3 sub-fix) + Phase 3 findings template severity-format normalization (AC-STE-295.5).
4. `plugins/dev-process-toolkit/skills/gate-check/SKILL.md` — probe #37 fence-only scope documentation line (AC-STE-295.4).
5. **NEW** `plugins/dev-process-toolkit/tests/pre-commit-tdd-orchestrator.test.ts` — staged-path classifier cases (AC-STE-295.1).
6. **NEW** `plugins/dev-process-toolkit/tests/linear-ac-token-round-trip.test.ts` — push → fetch byte-equality (AC-STE-295.2).

**Precedents:** STE-290 (hook wiring + emitNFR10 contract), STE-211 (write-side strip; this extends to read), STE-228 (branch gate + capability row format), STE-231 / STE-286 (fixture-group FR convention + diagnostic shape).

If `/implement` finds them too tangled, F9 can split off as its own AC during TDD without altering the FR scope — the merge step is the precondition either way.
