# STE-386: Gate-check probe #67 `fr_summary_altitude`: validate FR Summary altitude when present, vacuous when absent {#STE-386}

## Summary

A new project-health probe checks that any FR summary stays readable: short, plain prose, free of
code identifiers and spec jargon. Files without a summary pass untouched, so older FRs and
hand-written specs are unaffected. When a summary drifts into implementation detail, the gate
reports the exact location and which rule it broke.

## Requirement

STE-385 defines the `## Summary` altitude rule as prose; this FR makes it deterministic. Per the
operator decision at brainstorm ("altitude probe, presence optional"), a new `/gate-check` probe
validates the altitude of any `## Summary` section found in an **active** FR — and does nothing
when the section is absent, so pre-M105 FRs, tracker-imported FRs, and hand-authored FRs never
turn red. The four checks mirror STE-385's rule verbatim: (1) ≤ 6 non-empty lines; (2) zero
backtick characters; (3) no AC-ID tokens; (4) no path-like tokens (a whitespace-delimited token
containing both a slash and a dot-extension — chosen over a bare-slash rule to keep "and/or",
"read/write", and version strings out of the violation set). Severity is **error**: the altitude
rule is binary by construction, and a summary that names file paths or AC IDs is exactly the drift
the section exists to prevent.

The probe count moves 66 → 67, which touches the pinned README surface: probe #57
(`public_surface_count_drift`) independently computes the max numbered probe from the gate-check
skill and compares it to two README lines, and the count-drift meta-test pins both lines by
regex — all three surfaces move together or the gate fails.

## Acceptance Criteria

- AC-STE-386.1: **Scanner.** A new shared module in `adapters/_shared/src/` exports the probe backend: for every active FR (`specs/frs/*.md`, archive excluded) whose body contains a `## Summary` section, it enforces the four altitude rules and returns violations naming `file:line` + rule id from the closed set {`line_cap`, `backtick`, `ac_id`, `path_token`}. {#AC-STE-386.1}
- AC-STE-386.2: **Vacuity.** An FR without `## Summary` produces no violation and no note; an absent or FR-less `specs/frs/` passes with no note — a run that never authored a summary stays byte-identically green. {#AC-STE-386.2}
- AC-STE-386.3: **Probe wiring.** The scanner is wired as `/gate-check` probe #67 (`fr_summary_altitude`, severity error) in the numbered probe list, following the existing probe-entry shape. {#AC-STE-386.3}
- AC-STE-386.4: **Count pins.** Both pinned README lines move 66 → 67; the count-drift meta-test regexes are updated in the same commit; probe #57 reports zero drift. {#AC-STE-386.4}
- AC-STE-386.5: **Meta-tests + dogfood.** Fixture coverage exists for each of the four rules (one violating fixture each), for a clean summary, and for both vacuous paths; a dogfood assertion runs the scanner over this repo's own active FRs — the M105 FR summaries must clear their own probe. {#AC-STE-386.5}
- AC-STE-386.6: **Full gate green.** `bun test` from `plugins/dev-process-toolkit` passes with calibration snapshots bumped as needed (gate-check skill has line headroom; no STE-token ceiling movement expected). {#AC-STE-386.6}

## Technical Design

**Scanner shape** mirrors the design-references scanner: locate the section by heading regex
(`/^##\s+Summary\s*$/`), bound it at the next `^##` heading or EOF, evaluate the four rules over
the section body only. Pure function over file text + path list; no I/O beyond reading the FR
files handed to it (walk shape follows the existing active-FR walks).

**Rule notes.** `line_cap` counts non-empty lines (> 6 fails; the 3-line floor is authoring
guidance in STE-385, not probe-enforced — a 1-line summary is terse, not drift). `backtick` fails
on any backtick character (also subsumes code fences). `ac_id` matches AC-ID tokens of the
AC-prefix shape regardless of tracker flavor. `path_token` requires slash + dot-extension in one
token to avoid natural-prose false positives.

**Probe #67 wiring** follows the current gate-check probe conventions; `public_surface_count_drift`
picks the new max automatically once the numbered list gains entry 67, so only the two README
lines and the meta-test regexes need explicit edits.

## Notes

- **Active FRs only.** Archived FRs are frozen history; re-flagging them would force edits to
  shipped files. The archive walk is explicitly out of scope.
- **Severity error, not warning** — deliberate: the four rules are constructed to have near-zero
  false-positive surface, and a red gate is the only signal that reliably stops altitude drift.
- **Single rule source.** If the rules ever change, STE-385's prose and this scanner move in the
  same commit — the convention test pins the prose, the fixtures pin the scanner.
- Origin: GitHub issue #39 → `/brainstorm` 2026-07-16; probe precedent: the section-content-aware
  probe #26; vacuity precedent: the research-result shape probes' pass-with-no-note contract.
