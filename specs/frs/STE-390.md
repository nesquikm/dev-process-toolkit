# STE-390: Docs-drift sweep — retire stale branch-automation phrasing, clear tree-leaf advisories {#STE-390}

## Summary

Shipped docs still describe branch automation the way it worked before the last
three milestones changed it: a template comment lists retired trunk-OK types, two
docs say the branch type is inferred by the model, and the technical spec calls
the seeded default scope-aware. This sweep aligns those leftovers with the current
contract and clears the five standing tree-leaf advisories in the technical spec —
three stale references fixed, two illustrative examples reworded. Prose-only.

## Requirement

Retire the three stale branch-automation phrasings from shipped docs
(`templates/CLAUDE.md.template` trunk-OK comment; `docs/setup-tracker-mode.md` +
`docs/patterns.md` "LLM-inferred `{type}`" phrasing; `specs/technical-spec.md`
"scope-aware default" note) and clear all five probe-#37 advisories in
`specs/technical-spec.md` (three genuinely stale tree leaves fixed to resolvable
paths, two illustrative example fences reworded to resolvable/neutral tokens). Pin
every correction with one doc-conformance meta-test so retired phrasings cannot
return.

## Acceptance Criteria

- AC-STE-390.1: **Template trunk-OK comment.** `templates/CLAUDE.md.template`'s `branch_template:` comment names the trunk-OK allowlist as `ci` only — the retired `chore`/`docs` phrasing is gone from that comment block; meta-test pins presence of the corrected line + absence of the retired phrasing. {#AC-STE-390.1}
- AC-STE-390.2: **Deterministic `{type}` phrasing.** `docs/setup-tracker-mode.md` + `docs/patterns.md` placeholder docs describe `{type}` as derived deterministically via `branchTypeFor` from `changelog_category` (unknown-clamp mention kept); no "LLM-inferred" phrasing attached to `{type}` in either file — `{slug}` stays LLM-inferred and is not pinned; meta-test asserts both directions. {#AC-STE-390.2}
- AC-STE-390.3: **Schema L note.** `specs/technical-spec.md`'s `branch_template:` bullet says `/setup` seeds the single canonical default `{type}/m{N}-{slug}` in every mode; the "scope-aware default" phrasing is gone. {#AC-STE-390.3}
- AC-STE-390.4: **Stale tree leaves resolve.** The three genuinely stale leaves in `specs/technical-spec.md` (`agents/code-reviewer.md`, `docs/layout-reference.md`, `docs/reference/api/task_tracking_config.md`) are rewritten so every flagged token resolves on disk or no longer parses as a path token. {#AC-STE-390.4}
- AC-STE-390.5: **Illustrative examples clean + dogfood.** The two illustrative example fences (traceability-map example, docs-fragment frontmatter example) are reworded to resolvable/neutral tokens; a dogfood meta-test asserts `runCrossCuttingSpecStaleFileRefsProbe(repoRoot)` returns zero violations on this repo. {#AC-STE-390.5}
- AC-STE-390.6: **Full gate green.** `bun test` from `plugins/dev-process-toolkit` GREEN; no `skills/` edits (STE-token ceiling untouched); SKILL line caps untouched. {#AC-STE-390.6}

## Technical Design

- Prose-only sweep — no scanner/adapter code changes (scanner exemption explicitly
  rejected at spec time; probe #37's fence-only scope stays as pinned by the
  earlier scope decision).
- One new doc-conformance meta-test carries all pins (retired-phrase absence +
  corrected-phrase presence per site) plus the probe-#37 dogfood assertion — same
  shape as the probe-#67 dogfood.
- Precedent shapes: the deterministic-`{type}` docs sweep AC from the
  branchTypeFor FR; the enumerate-every-site docs-sweep AC pattern; byte-checkable
  text-only drift fixes.
- Note: the dogfood assertion hardens probe #37 from advisory to de-facto error
  for this repo — accepted; future tree-leaf rot fails the gate instead of
  accumulating.

## Testing

  assertion over the live repo; deterministic, no network; full gate from
  `plugins/dev-process-toolkit`.
