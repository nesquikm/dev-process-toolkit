# STE-363: Archival-time milestone-label assertion {#STE-363}

## Requirement

`/gate-check` probe #26 (`tracker-project-milestone-attached`) is the standing check that an FR-backed ticket carries its milestone binding ‚Äî but it is **"vacuous when the FR is archived."** So the invariant is enforced only while an FR is `status: active`; the moment `/implement` Phase 4 (or `/spec-archive`) flips the FR to archived, a missing binding stops being checked and is hidden forever. That is exactly how **GB-11** slipped: whatever the attach failure was, it was never re-flagged after archival.

This FR adds a **durable net at the archival boundary** ‚Äî the last moment an FR is "ours." Before `/spec-archive` (single-FR and milestone-group) or `/implement` Phase-4-close flips an FR-backed ticket's FR to archived, it asserts the bound ticket carries its milestone binding, using the same adapter-aware surface probe #26 uses. On a missing binding it attempts one attach; if the binding is still absent on re-verify, it **refuses the archive** with an NFR-10 canonical shape, so a label gap can never be sealed behind an archived FR. Scope is **FR-backed only** ‚Äî tickets with no FR (GB-12-class bugs) are never fetched; bugs are not milestone-scoped.

## Acceptance Criteria

- AC-STE-363.1: **Assertion at the archival boundary.** `/spec-archive` (single-FR and milestone-group paths) and `/implement` Phase-4-close, for each FR being archived in tracker mode with `project_milestone: true`, fetch the bound ticket and assert its milestone binding is present via the adapter-aware surface: `milestoneBinding: "object"` (Linear) ‚Üí `projectMilestone.name` byte-equals the plan-file H1 canonical name (`planFileHeadingToMilestoneName`); `"label"` (Jira) ‚Üí `labels` contains `milestone-<M-token>` (the leading `M\d+` of that canonical name). {#AC-STE-363.1}
- AC-STE-363.2: **Attempt-then-refuse.** On a missing binding, attempt `attachProjectMilestone` once and re-verify; if the binding is now present, proceed with archival; if still absent, **refuse the archive** with an NFR-10 canonical shape naming the ticket, the expected label/milestone name, and the remedy (`run /spec-archive --backfill-milestone-labels, or attach the milestone manually, then re-run`). On refusal the FR is **not** flipped to archived and **no `git mv`** fires for that FR (the milestone-group path skips only the offending FR and reports it; other FRs proceed). {#AC-STE-363.2}
- AC-STE-363.3: **FR-backed scope only.** The assertion iterates exactly the FR files being archived and their `tracker:` bindings; it never enumerates the tracker board and never fetches a ticket that has no FR. Bugs and ad-hoc tickets (GB-12-class) are out of scope by construction. {#AC-STE-363.3}
- AC-STE-363.4: **Vacuity.** Vacuous ‚Äî no fetch, no assertion, byte-identical archival ‚Äî on `mode: none`, adapters with `project_milestone: false`, and any FR whose `milestone:` plan file is missing (that diagnostic belongs to `/gate-check` probe #27, `frontmatter-milestone-not-archived`; this assertion does not double-report it). {#AC-STE-363.4}
- AC-STE-363.5: **Capability tokens.** The closing summary emits exactly one literal token per archived FR for the assertion outcome ‚Äî `milestone_label_asserted_at_archive` (binding present, or missing-then-attached) or `milestone_label_archive_refused` (still missing ‚Üí refusal) ‚Äî added to the `skills/spec-write/SKILL.md` ¬ß 7 static capability map (rendered prose) with `MUST emit` directives in `/spec-archive` + `/implement`, so `/gate-check`'s `closing_summary_capability_keys` probe enforces it. {#AC-STE-363.5}

## Technical Design

**Scope: a shared assertion helper + wiring into the two archival surfaces.**

**Files touched:**
- `adapters/_shared/src/` (new, e.g. `assert_milestone_binding_at_archive.ts`) ‚Äî `assertMilestoneBindingAtArchive(provider, project, frFile, deps)`: read the FR's `milestone:` + `tracker:` frontmatter; if vacuous-case, return `{ outcome: "vacuous" }`; else derive the canonical name via `planFileHeadingToMilestoneName`, fetch the ticket, assert the adapter-aware binding; on miss call `attachProjectMilestone` once + re-verify; return `{ outcome: "asserted" | "refused", token, detail }`. Reuses the STE-362 attach helper (with its retry).
- `skills/spec-archive/SKILL.md` ‚Äî call the helper before each FR's `git mv` frontmatter flip; on `refused`, skip that FR's archival with the NFR-10 message; in the milestone-group Diff Preview, list the assertion outcome per FR. Add the two `MUST emit` token directives.
- `skills/implement/SKILL.md` ‚Äî Phase-4-close: call the helper for the FR(s) being archived before the archival sweep; refusal blocks that FR's archival with the NFR-10 message.
- `skills/spec-write/SKILL.md` ¬ß 7 ‚Äî two new capability keys + rendered prose in the static map.

**Adapter-aware surface (reuse, don't reinvent).** The binding check mirrors probe #26's `milestoneBinding`-selected assertion exactly (object ‚Üí `projectMilestone.name` byte-equality; label ‚Üí `labels ‚àã milestone-<M-token>`). Sharing the predicate keeps the archival assertion and the active-side gate byte-consistent, so a ticket that passes probe #26 while active also passes the archival assertion (no spurious refusals at the boundary).

**Why the archival boundary (not an extended gate-check probe).** Extending probe #26 to all archived FRs would re-fetch every historical Done ticket on every `/gate-check`, forever ‚Äî cost grows with every shipped FR. The archival boundary checks each FR exactly once, at the last moment it is active-and-ours, which is precisely where GB-11 escaped. (2026-07-03 design decision.)

## Notes

- **Origin.** 2026-07-03 `/brainstorm` ‚Üí `/spec-write`, Option 2 + the "Archival-time assertion (+ one-shot backfill)" placement (user-selected).
- **The hole this closes.** Probe #26 is documented "Vacuous when ‚Ä¶ the FR is archived" (`skills/gate-check/SKILL.md`). The archived FR still declares its milestone in frontmatter, so the assertion is well-defined at archival time ‚Äî this FR uses that last well-defined moment.
- **Interplay with the capability-gap downgrade.** Probe #26 has an advisory downgrade (STE-194/STE-214) when the FR `## Notes` carries a `milestone_attach_*` token. The archival assertion is a **hard** boundary (attempt-then-refuse) and does not honor that downgrade ‚Äî the downgrade excuses a *known adapter limit* on the active side, but at archival the binding must actually land or the archive is refused. A `project_milestone: false` adapter is handled by AC-STE-363.4 vacuity, not by the token downgrade.
- **Sibling FRs.** STE-362 hardens the write side so the assertion rarely has to attach; STE-364 backfills tickets already archived before this FR shipped (which the archival boundary cannot retroactively catch).
