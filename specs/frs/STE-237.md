## Requirement

STE-232 (M65) closed the autonomous-mode contract on the **per-step data-gate** side: when `/setup` reaches step 7b, `requireOrRefuse(...)` refuses with NFR-10 unless a real answer is supplied. The Skills In Scope table covers single-gate sites (step 7b, draft/commit, Phase 4 step 15, release approval). What it does **not** cover is the **whole Socratic clarification loop being skipped before any gate is reached.** Magpie incident (gist `https://gist.github.com/nesquikm/2904e50c7213b6aa392b998d4137f609`, 2026-05-07, plugin v2.16.0): `/setup` invoked under the harness-injected autonomous-mode reminder *"work without stopping for clarifying questions"* + verbose pre-baked-args prose + **no** `<dpt:auto-approve>v1</dpt:auto-approve>` marker. Model emitted `Write` calls (plugin.json, .mcp.json, 7 `SKILL.md` files, README, CLAUDE.md) before any clarifying question. Step 7b never fired; STE-232's `imputed:` audit row was never written. The recurrence class is **"whole Socratic loop skipped"**, distinct from STE-232's per-gate coverage. STE-220 cautionary precedent applies: prose-only Pattern-26 discipline failed at runtime; the fix must be byte-checkable, not prose-only.

## Acceptance Criteria

- AC-STE-237.1: `docs/auto-mode-protocol.md` gains a new top-level section `## Socratic Loop Contract` immediately after `## Refusal Mechanism`. Sections in order: (a) the rule — every clarifying Q in a Pattern-26-tagged skill body MUST be emitted as an `AskUserQuestion` tool call (closed-form OR open-ended; "Other" free-form fallback covers the open-ended case), regardless of the autonomous-mode reminder, the auto-approve marker, or pre-baked `<command-args>` prose; (b) the first-turn contract — `Write` / `Edit` / `NotebookEdit` tool calls are forbidden before the first `AskUserQuestion` tool_use OR `RequiresInputRefusedError` raise in the model's response stream (read-only tools — `Read`, `Grep`, `Glob`, `Bash`-read-only — are allowed for orientation); (c) the in-scope skill list (initial: `/setup`, `/brainstorm`, `/spec-write`, `/report-issue`; forward-extension hook below); (d) cross-references to STE-226 (default-apply), STE-232 (refusal), STE-220 (prose-only failure precedent).

- AC-STE-237.2: Each in-scope skill body — `plugins/dev-process-toolkit/skills/{setup,brainstorm,spec-write,report-issue}/SKILL.md` — is rewritten so every clarifying Q is structured as an `AskUserQuestion` tool call. Open-ended Qs use 2–3 templated options + the always-on "Other" free-form fallback (no new harness primitive required). Each updated skill body cites `docs/auto-mode-protocol.md § Socratic Loop Contract` by relative path. Pattern 26 prose stays as the conceptual anchor; `AskUserQuestion` is the structural enforcement.

- AC-STE-237.3: New `/gate-check` probe `socratic_loop_uses_ask_user_question`. Globs `plugins/dev-process-toolkit/skills/*/SKILL.md` and `.claude/skills/*/SKILL.md` (matching STE-226:AC.5 / STE-232:AC.5 globbing). For every skill body that (a) cites `Pattern 26` OR (b) carries a `socratic: true` Schema-K tag (forward-extension hook), asserts: (i) every conversational Q in the body is wrapped as an `AskUserQuestion` invocation block, (ii) the body contains a relative-path citation of `docs/auto-mode-protocol.md`. Either missing ⇒ separate violation row. Hard-fails the gate. Surfaces `socratic_loop_uses_ask_user_question_violation` capability rows.

- AC-STE-237.4: `/smoke-test` gains `Phase 8 — Socratic Loop Entry`. For each of `{setup, brainstorm, spec-write, report-issue}`, spawns a `claude -p` child with a heredoc body that (i) carries the harness autonomous-mode reminder verbatim ("The user has asked you to work without stopping for clarifying questions…"), (ii) supplies a verbose pre-baked-args prompt that *appears* to answer all questions, (iii) does **not** carry the `<dpt:auto-approve>v1</dpt:auto-approve>` marker. Captures the child's response stream (tool_use entries) and asserts: no `Write` / `Edit` / `NotebookEdit` tool_use entry appears before the first `AskUserQuestion` tool_use OR `RequiresInputRefusedError` raise. Violation surfaces `socratic_first_turn_contract_violation` capability row in the smoke summary; hard-fails the smoke run.

- AC-STE-237.5: Pure-I/O helper `assertFirstTurnShape(transcript)` at `adapters/_shared/src/socratic_first_turn.ts`. Takes a parsed transcript (array of `{ type: 'text' | 'tool_use', name?, … }` entries), returns `{ outcome: 'ok-asked' | 'ok-refused' | 'violation', firstScaffoldEntry?, askIndex? }`. Throws `SocraticFirstTurnViolationError` (NFR-10 canonical shape) on violation, naming the offending tool name + zero-based index in the response stream. Unit-tested via `socratic_first_turn.test.ts` covering the six-case matrix: ask-first, refuse-first, scaffold-before-ask (violation: Write), scaffold-before-ask (violation: Edit), read-only-then-ask (ok), text-then-ask (ok). The helper is the single arbiter consumed by the Phase 8 fixture and any future runtime detector.

- AC-STE-237.6: `## /setup audit` row format gains a `loop_entered: true | false` column (additive, parsed tolerantly per STE-232's `imputed:` precedent). Set to `true` when `/setup` Steps 1–6 emitted at least one `AskUserQuestion` clarifier (model entered the Socratic loop); `false` otherwise. Helper `appendAuditRow(...)` (`adapters/_shared/src/setup/audit_log.ts`) gains the `loopEntered: boolean` parameter. Pairs with STE-232's `imputed:` for two-axis loop visibility — `loop_entered:false` flags the magpie regression class structurally; `imputed:true` flags the per-gate regression class. Round-trip parser test for both columns present / absent / mixed.

- AC-STE-237.7: `docs/patterns.md § Pattern 26: Socratic Prompting` gains a new sub-section `### Structural enforcement (STE-237)` pinning: "Pattern 26 is enforced structurally via `AskUserQuestion` (B-side) and behaviorally via `/smoke-test` Phase 8 (C-side); prose discipline alone is insufficient (STE-220 lesson)." Cross-links `docs/auto-mode-protocol.md § Socratic Loop Contract` and the magpie incident gist as the originating signal.

## Technical Design

Brainstorm-approved B+C hybrid (gist `https://gist.github.com/nesquikm/2904e50c7213b6aa392b998d4137f609` + 2026-05-07 brainstorm session). The user explicitly elevated the design to "universal `AskUserQuestion` mandate" (every clarifying Q, closed or open-ended, becomes an `AskUserQuestion` call) plus "pragmatic first-turn contract" (read-only orientation allowed, scaffolding before first Q forbidden).

| Layer | Mechanism | What it catches |
|---|---|---|
| Source-level (B) | Skill body emits `AskUserQuestion` for every clarifying Q | Bare-prose Qs that the model could synthesize answers for |
| Structural (B) | Harness blocks loop progression until `AskUserQuestion` returns | Model literally cannot fabricate an answer |
| Behavioral (C) | `/smoke-test` Phase 8 + `assertFirstTurnShape` | Model regression: scaffolding emitted before first Q |
| Source probe (C) | `/gate-check` `socratic_loop_uses_ask_user_question` | Skill drift: bare prose Qs reintroduced; missing protocol citation |
| Audit (C) | `/setup` `loop_entered:` column | Operator-visible structural signal that the loop fired |

Forward-extension hook: any new skill that carries Pattern 26 prose OR a `socratic: true` Schema-K tag is automatically picked up by the AC.3 probe — no manual list maintenance. The probe re-checks on every gate-check run; new skills that ship without `AskUserQuestion` wiring hard-fail immediately.

Maps to STE-226 + STE-232 by analogy:

| Layer | Per-step gate (STE-226 + STE-232) | Per-conversation loop (this FR) |
|---|---|---|
| Default-apply token | `<dpt:auto-approve>v1</dpt:auto-approve>` marker | (no analog — Socratic loops have no "safe default") |
| Refusal site | `requireOrRefuse(spec, key, sentinel)` | first-turn contract + `assertFirstTurnShape(transcript)` |
| Source probe | `requires_input_sentinel_coverage` | `socratic_loop_uses_ask_user_question` |
| Audit column | `imputed: true|false` | `loop_entered: true|false` |
| Behavioral fixture | (none — gate fires deterministically) | `/smoke-test` Phase 8 |

Out of scope:

- `/implement` Phase-1 stack-detection prompt — it's an approval gate, not a Socratic loop. If a similar regression class surfaces there, separate FR.
- A new free-form `RequestUserText` harness primitive — `AskUserQuestion`'s always-on "Other" fallback covers all in-scope cases.
- Backwards-compat shim for legacy bare-prose Qs in skill bodies — per `project_no_users_yet`, skills are rewritten in AC.2 with no migration path.

## Testing

Unit:

- `socratic_first_turn.test.ts` — six-case matrix per AC.5 (ask-first, refuse-first, scaffold-Write-before-ask, scaffold-Edit-before-ask, read-only-then-ask, text-then-ask). Covers the helper's outcome enumeration + violation shape.
- `audit_log.test.ts` extension — round-trip with `loop_entered:` column present / absent; combined with `imputed:` permutations; tolerant parse of legacy rows lacking either column.

Integration:

- `gate-check.test.ts` — fixture skill markdown variants: (a) Pattern-26 cited + `AskUserQuestion` blocks present + protocol citation present (pass); (b) Pattern-26 cited + bare prose Qs (fail AC.3.i); (c) Pattern-26 cited + `AskUserQuestion` present + protocol citation absent (fail AC.3.ii); (d) `socratic: true` tag + bare prose Qs (fail — forward-extension hook); (e) no Pattern-26 / no tag (vacuous, no probe firing). Probe must hard-fail (b) / (c) / (d), pass (a) / (e).

Smoke:

- `/smoke-test` Phase 8 itself is the regression coverage — four child-spawn fixtures (one per in-scope skill), each with the simulated trigger heredoc described in AC.4. Fixture artifacts capture under `tests/fixtures/socratic-first-turn/` for replay. Hard-fail bubbles up to the smoke summary; pass adds a `socratic_first_turn_contract_ok` capability row per skill.

Behavioral spot-check (manual, one-time, post-implementation):

- Re-run the magpie scenario: invoke `/setup` against an empty repo with the prompt body from the gist's `report.md` (verbose pre-baked-args prose) under a session carrying the autonomous-mode reminder. Expected first action: `AskUserQuestion`. Capture the transcript and link from this FR's notes as the regression-coverage anchor.
