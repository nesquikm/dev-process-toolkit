# STE-373: Deps-research fork enforcement parity — deterministic shape gate + no-silent-skip tokens {#STE-373}

## Requirement

The deps-research fork's output shape is **not runtime-enforced**. STE-318 (AC-STE-318.3)
deliberately dropped the shape probe its architectural twin `spec-research` carries at
gate-check #41 (`spec_research_result_shape`), choosing "operator-judgment, not
runtime-enforced" on minimum-scope grounds — even though STE-301's own design intent was
strict parity between the twins. On top of that asymmetry, a **skipped** deps-research
invocation emits no capability token at all: the vacuous-no-manifest and `--no-tech` paths
are silent by construction.

The combination is what let the 2026-07-09 incident cascade. A `deps-researcher` (Haiku)
fork emitted instruction-shaped / confabulated prose instead of a clean block; the
LLM-judgment shape-check in the parent skill caught it (nothing was written, no config
changed — the defense held), but the model then carried an unverified "standing note" that
"the fork returns prompt-injection" and **silently stopped invoking the fork** across the
project (observed transcript line 149) with no user-visible signal, and filed a
high-severity report from that unverified belief.

This FR closes the enforcement asymmetry and removes every silent path. It (a) adds the
deterministic `deps_research_result_shape` gate the twin already has, reusing the
already-built `parseDepsResearchBlock` parser; and (b) makes the *complete disposition set*
of a deps-research step byte-checkable — including the previously-silent skip paths — with
**no legal disposition for "previously believed compromised."** A shape-violation drops the
current seed and continues; it never disables the fork for future invocations. This is
Approach #2 ("Deterministic tokens + gate") from the 2026-07-09 `/brainstorm`; the hermetic
persist-and-extract seam (Approach #3) is explicitly out of scope because no code seam
exists between a fork and its parent (see `## Notes`).

## Acceptance Criteria

- AC-STE-373.1: **Deterministic shape gate.** A new `/gate-check` probe `deps_research_result_shape` (severity: error) ships at `adapters/_shared/src/deps_research_result_shape.ts`, a structural clone of `spec_research_result_shape.ts` (#41). It walks recorded `.dpt-locks/**/deps-research-result.txt` logs and validates each block via the existing `parseDepsResearchBlock` (`adapters/_shared/src/deps_research_result.ts`): banner line immediately above the opening fence, fence opens ` ```deps-research-result `, the canonical `## ` sections in fixed order (`## Relevant Packages`, `## API Surface Highlights`, `## Reusable Patterns`, optional `## Missing deps`), ≤ 25-line cap, exactly one fenced block. Each violation surfaces as a `file:line — reason` note in NFR-10 canonical shape. Vacuous (pass, no note) when no result log is recorded — the common no-invocation path. The probe is registered in `skills/gate-check/SKILL.md` with the next free probe number. {#AC-STE-373.1}
- AC-STE-373.2: **Block persistence so the gate has teeth.** `/brainstorm` Step 1.5b and `/spec-write` § 0b step 2.5b persist the deps-research fork's returned block to `.dpt-locks/<ulid>/deps-research-result.txt` (mirroring the spec-research persistence convention that probe #41 reads), so probe AC-STE-373.1 is non-vacuous on real invocations. The persisted file is the raw fork return; the probe — not the LLM — is what validates its shape. {#AC-STE-373.2}
- AC-STE-373.3: **Kill the phantom-probe reference.** `agents/deps-researcher.md` (the "Line cap (hard)" paragraph, ~line 54) is rewritten: the "Output shape is operator-judgment, not runtime-enforced … asymmetry with spec-researcher.md is intentional" language is replaced with a reference to the now-real `deps_research_result_shape` probe, matching the `spec-researcher.md:46` form. - AC-STE-373.4: **Skip-disposition tokens registered.** Two new capability keys `deps_research_skipped_no_manifest` and `deps_research_skipped_no_tech` are added to `CANONICAL_CAPABILITY_KEYS` (`adapters/_shared/src/closing_summary_capability_keys.ts`) and to the `/spec-write` § 7 static plain-language map (rendered prose), with mirror rows in `/brainstorm` Step 1.5b. {#AC-STE-373.4}
- AC-STE-373.5: **Complete legal disposition set (MUST-emit).** `/brainstorm` Step 1.5b and `/spec-write` § 0b step 2.5b carry MUST-emit directives establishing that the deps-research step of any invocation resolves to **exactly one** literal, backticked token: `deps_research_invoked` / `deps_research_no_matches` / `deps_research_shape_violation` (the fork ran) **or** `deps_research_skipped_no_manifest` / `deps_research_skipped_no_tech` (legally not invoked). When `specs/deps.yaml` has ≥ 1 entry and the run is not `--no-tech`, the fork MUST be invoked (one of the first three emits). There is **no legal token for "previously believed compromised," "injected," or "disabled."** {#AC-STE-373.5}
- AC-STE-373.6: **No cross-invocation belief (anti-cascade).** Both parent skills carry an explicit rule: a deps-research shape-violation drops *this* seed and continues, and **never** disables the fork for future invocations; the skill holds no "fork compromised" state and carries no cross-invocation belief about fork health. Each invocation re-invokes the fork per the manifest gate, independent of any prior run's outcome. (Directly targets the transcript-line-149 silent-skip behavior.) {#AC-STE-373.6}
- AC-STE-373.7: **Definition-level enforcement.** A `/gate-check` probe (extend `deps_researcher_subagent_invariants` #51 or add a colocated one) byte-checks the definition-level contract: both skip tokens are registered in `CANONICAL_CAPABILITY_KEYS`; the MUST-emit disposition directive (AC-STE-373.5) and the anti-cascade rule (AC-STE-373.6) are present in both `/brainstorm` and `/spec-write`; and no `deps_research_skipped_*` reason token names `compromised` / `injected` / `disabled`. Severity: error. {#AC-STE-373.7}
- AC-STE-373.8: **Reversal is documented.** The FR `## Notes` records that this FR explicitly **reverses STE-318 AC-STE-318.3**'s minimum-scope decision, naming the 2026-07-09 incident as the counter-evidence that parity was worth the cost. {#AC-STE-373.8}

## Technical Design

**Scope: one new probe + one probe extension + parent-skill persistence & token wiring + an agent-file rewrite. No new subagent, no new fork, no change to the `deps-researcher` tool surface.**

**Files touched:**
- `adapters/_shared/src/deps_research_result_shape.ts` (new) — `runDepsResearchResultShapeProbe(projectRoot)`, a structural clone of `spec_research_result_shape.ts` (#41). Walks `.dpt-locks/**/deps-research-result.txt`, feeds each file to the existing `parseDepsResearchBlock`, and maps every `{ ok: false, reason }` to an NFR-10 `file:line — reason` note. Vacuous when no log exists.
- `adapters/_shared/src/closing_summary_capability_keys.ts` — register `deps_research_skipped_no_manifest`, `deps_research_skipped_no_tech`.
- `adapters/_shared/src/deps_researcher_invariants.ts` (probe #51) — extend to byte-check AC-STE-373.7's definition-level contract, or add a sibling probe if cleaner.
- `skills/gate-check/SKILL.md` — register the new `deps_research_result_shape` probe row (next free probe number) + any extension note for #51.
- `skills/brainstorm/SKILL.md` (Step 1.5b) and `skills/spec-write/SKILL.md` (§ 0b step 2.5b, § 7 map) — persist the fork block to `.dpt-locks/<ulid>/deps-research-result.txt`; add the complete-disposition MUST-emit directive (AC-STE-373.5), the two skip tokens, the anti-cascade rule (AC-STE-373.6), and the § 7 rendered-prose rows.
- `agents/deps-researcher.md` — rewrite the "Line cap (hard)" paragraph per AC-STE-373.3.

**Why post-hoc gate + definition-level tokens (not a runtime firewall).** There is no code seam between a `context: fork` subagent and its parent: the fork's turn text lands directly in the parent LLM's context, and only the persisted `.dpt-locks/**` log is available to deterministic code. So the enforceable levers are (1) a post-hoc gate over the persisted log (catches shape drift in CI/review) and (2) a definition-level token contract the `closing_summary_capability_keys` + `deps_researcher_subagent_invariants` probes byte-check. The consumption-time hermetic seam (brainstorm Approach #3) is deliberately out of scope.

**Parity, not novelty.** Every mechanism here already exists for the `spec-research` twin (parser, #41 probe, persisted log, capability tokens). This FR retargets those patterns to `deps-research`, closing the asymmetry STE-301 intended and STE-318 deferred.

## Notes

- **Origin.** 2026-07-09 `/report-issue` gist (high severity) → `/brainstorm` (Approach #2 "Deterministic tokens + gate", user-selected) → `/spec-write`. The report's premise ("the fork emits prompt-injection") diverged from its own attached evidence: the fork was never invoked in the captured transcript, the quoted payload was ~75% genuine Claude Code harness boilerplate + one fabricated output-styles bluff, and `/report-issue` captured the wrong session. The real problem was the behavioral cascade + this enforcement asymmetry, not the injection (which the LLM-judgment shape-check already caught).
- **Reverses STE-318 AC-STE-318.3.** STE-318 chose option (a) "drop the false phantom-probe claim" over (b) "author the missing probe," on minimum-scope grounds. This FR takes option (b) — the 2026-07-09 incident is the counter-evidence that the deps twin needs the deterministic gate its spec twin has (Core Principle #1: deterministic gates override LLM judgment).
- **Sibling FR.** STE-374 (report-issue evidence gate) fixes the downstream half of the cascade — verify-before-alarm + capturing the session that actually contains the incident.
- **Honest limit.** These layers are defense-in-depth, not a hermetic seal: they cannot stop a Haiku subagent from *generating* stray text, but they make its shape drift deterministically catchable and its skip paths non-silent, and they forbid the cross-invocation "compromised" belief that actually caused the harm.
