# STE-383: `/setup` writes a self-contained `.dpt/.gitignore`; retire the root `.dev-process/` append {#STE-383}

## Requirement

`/setup` currently owns an entry in the **consumer's** root `.gitignore` (AC-STE-344.5: append `.dev-process/`, create the file if absent, no-op if present — existence-guarded and idempotent). That couples the toolkit's correctness to a file the consumer edits freely, and it cannot express the layout `.dpt/` needs: locks tracked, ledger + scratch ignored.

Ship the ignore rules **inside the folder the toolkit owns** — a committed `.dpt/.gitignore` containing `ledger/` and `scratch/`. Verified empirically 2026-07-15: the nested file yields the correct three-way result (`locks/` tracked, `ledger/` + `scratch/` ignored) with the consumer's root `.gitignore` **completely empty**. `/setup`'s ignore duty collapses from "append to someone else's file with dedup logic" to "write one file we own": idempotent by construction, impossible to partially apply, survives a consumer rewriting their root `.gitignore`, and self-documenting — reading `.dpt/.gitignore` tells you the tracked/ignored split.

**The polarity is deliberate.** The alternative — blanket-ignore `.dpt/` and re-include locks — is a trap. Git cannot re-include a path whose parent directory is excluded, so `.dpt/` + `!.dpt/locks/` **silently unversions every lock** (verified: `git check-ignore` reports `.dpt/locks/01HLOCK` as IGNORED). Only the `.dpt/*` glob form works, and it sits one careless "tidy-up" away from the broken form. The two failure modes are not symmetric:

| Polarity | Forgotten rule ⇒ | Character |
|---|---|---|
| Tracked-by-default (this FR) | scratch leaks into a commit | loud, visible, harmless |
| Ignore-and-negate | a coordination signal stops being tracked | silent, invisible, correctness-breaking |

This FR takes the loud-failure polarity. Approved via `/brainstorm` 2026-07-15 (Approach 1).

## Acceptance Criteria

- AC-STE-383.1: **Idempotent toolkit-owned write.** A new helper composes and writes `<root>/.dpt/.gitignore` with the canonical body (`ledger/` + `scratch/`), creating `.dpt/` when absent. Idempotent by byte-compare on the STE-303 pattern: baseline identical to proposal ⇒ no write, no prompt; safe to re-run. {#AC-STE-383.1}
- AC-STE-383.2: **`/setup` invokes it.** `/setup` calls the helper on every run (existence-guarded, no-op on match), and the write is recorded in the `## /setup audit` surface per the STE-133 best-effort-and-log precedent — a failure here never fails `/setup`. {#AC-STE-383.2}
- AC-STE-383.3: **Root append retired.** `/setup`'s `.dev-process/` root-`.gitignore` append (AC-STE-344.5; `skills/setup/SKILL.md:158` prose + its implementation) is removed. `/setup` no longer writes to the consumer's root `.gitignore` for toolkit state at all. {#AC-STE-383.3}
- AC-STE-383.4: **Toolkit's own root `.gitignore`.** The `.dev-process/` line is removed from this repo's root `.gitignore`, and **no `.dpt/` line is added** — the nested `.dpt/.gitignore` governs. A root entry would defeat it, since git never descends into an excluded directory. {#AC-STE-383.4}
- AC-STE-383.5: **`check-ignore` assertions.** A test builds a scratch repo carrying the shipped `.dpt/.gitignore`, an empty root `.gitignore`, and one file in each subtree, then asserts via `git check-ignore`: `.dpt/locks/<id>` **tracked**, `.dpt/ledger/token-ledger.jsonl` **ignored**, `.dpt/scratch/<ulid>/spec-research-result.txt` **ignored**, `.dpt/.gitignore` itself **tracked**. A negative case asserts the naive `.dpt/` + `!.dpt/locks/` form ignores locks — pinning why the polarity is what it is. {#AC-STE-383.5}
- AC-STE-383.6: **Docs + accepted hazard.** `docs/layout-reference.md` and `docs/hooks-reference.md` describe the `.dpt/` tree and the tracked/ignored split. **Scope widened 2026-07-15** (see § Notes, "Unowned doc drift"): the sweep also re-points every remaining retired-literal line under `docs/` — `sdd-methodology.md` (`.dpt-locks/<ulid>` files + merge-time conflict prose), `tracker-adapters.md` (`LocalProvider` entry), `implement-reference.md` (Phase 4 Release + post-release verification), and `workflow-overview.md` (artifact table row) — so that zero `.dpt-locks` / `.dev-process` literals survive under `docs/` outside STE-384's history allow-list. The one hole is documented explicitly: a consumer with a blanket `.dpt/` rule in their **root** `.gitignore` still wins, since git never descends into an excluded directory — accepted risk (the operator declined a `/setup` conflict warning at brainstorm), so it is recorded rather than defended. {#AC-STE-383.6}
- AC-STE-383.7: **Full gate green.** `bun test` from `plugins/dev-process-toolkit` passes; no probe regresses. {#AC-STE-383.7}

## Technical Design

**Canonical `.dpt/.gitignore` body** — deliberately relative, not rooted: nested `.gitignore` patterns resolve against their own directory, so the file is position-independent.

```gitignore
ledger/
scratch/
```

**Empirical basis** (2026-07-15, `git check-ignore` per path, root `.gitignore` empty):

| Strategy | `locks/` | `ledger/` | `scratch/` | Verdict |
|---|---|---|---|---|
| Nested `.dpt/.gitignore` (this FR) | tracked ✅ | ignored ✅ | ignored ✅ | works with an empty root ignore |
| Root ignore-list (`.dpt/ledger/`, `.dpt/scratch/`) | tracked ✅ | ignored ✅ | ignored ✅ | works; needs the consumer's file |
| Root `.dpt/*` + `!.dpt/locks/` | tracked ✅ | ignored ✅ | ignored ✅ | works; one edit from the trap |
| Root `.dpt/` + `!.dpt/locks/` | **IGNORED ❌** | ignored | ignored | **silently kills locking** |

**Precedents.** STE-303 (`/setup` idempotent toolkit-owned-file write — baseline == proposal ⇒ no prompt / no write; delta ⇒ operator approves; `cancel` keeps baseline). STE-133 (`.git/hooks/commit-msg` `cp` from `templates/`, best-effort, logged to `## /setup audit` rather than failing under `bypassPermissions`). `.mcp.json` § 7 merge-don't-overwrite.

**The AC.5 `check-ignore` matrix runs against real git, not a simulation** — `Bun.spawnSync(["git", "check-ignore"])` inside a `mkdtempSync` repo whose `.dpt/.gitignore` is written by the real `writeDptGitignore` helper (not a hardcoded string), with a genuinely **0-byte** root `.gitignore`, which is the FR's whole empirical claim. Verified non-vacuous by mutation: deleting the nested file collapses the split (ledger + scratch flip to tracked).

**The negative case reproduces the real trap**, and is paired with a control that isolates the variable. `check-ignore -v` on the naive `.dpt/` + `!.dpt/locks/` form reports `.gitignore:1:.dpt/` as the firing rule — the blanket exclusion wins and the negation never gets a chance. The paired `.dpt/*` + `!.dpt/locks/` control shows locks **tracked**, proving the negation *syntax* is valid and that the trap is caused by the one-character `.dpt/` vs `.dpt/*` difference, not a malformed pattern. Per the design intent, this case documents the design rather than merely checking it — it is kept even though it asserts a form we do not ship.

**Deliberately NOT asserted:** `existsSync(".dev-process") === false`. Per § Notes ("Installed-plugin transition window"), the version-pinned hook may transiently re-create the folder until the plugin cache refreshes, so that assertion would go red on a tree this FR calls correct. The **durable** half is asserted instead: no root rule references it, and `git ls-files -- .dev-process` is empty.

**Calibration at FR exit:** `skills/setup/SKILL.md` at exactly **354/354** lines (NFR-1, measured `split("\n").length`); skills STE-token at **245/245**. Both at zero headroom, both verified with their snapshot files untouched (`git diff --stat` empty) — neither pin was bumped to accommodate this FR. The replacement `/setup` step cites its precedents **by mechanism, not by STE-N token**, spending zero token budget.
