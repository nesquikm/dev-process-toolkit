# STE-383: `/setup` writes a self-contained `.dpt/.gitignore`; retire the root `.dev-process/` append {#STE-383}

## Requirement

`/setup` currently owns an entry in the **consumer's** root `.gitignore` (AC-STE-344.5: append `.dev-process/`, create the file if absent, no-op if present â€” existence-guarded and idempotent). That couples the toolkit's correctness to a file the consumer edits freely, and it cannot express the layout `.dpt/` needs: locks tracked, ledger + scratch ignored.

Ship the ignore rules **inside the folder the toolkit owns** â€” a committed `.dpt/.gitignore` containing `ledger/` and `scratch/`. Verified empirically 2026-07-15: the nested file yields the correct three-way result (`locks/` tracked, `ledger/` + `scratch/` ignored) with the consumer's root `.gitignore` **completely empty**. `/setup`'s ignore duty collapses from "append to someone else's file with dedup logic" to "write one file we own": idempotent by construction, impossible to partially apply, survives a consumer rewriting their root `.gitignore`, and self-documenting â€” reading `.dpt/.gitignore` tells you the tracked/ignored split.

**The polarity is deliberate.** The alternative â€” blanket-ignore `.dpt/` and re-include locks â€” is a trap. Git cannot re-include a path whose parent directory is excluded, so `.dpt/` + `!.dpt/locks/` **silently unversions every lock** (verified: `git check-ignore` reports `.dpt/locks/01HLOCK` as IGNORED). Only the `.dpt/*` glob form works, and it sits one careless "tidy-up" away from the broken form. The two failure modes are not symmetric:

| Polarity | Forgotten rule â‡’ | Character |
|---|---|---|
| Tracked-by-default (this FR) | scratch leaks into a commit | loud, visible, harmless |
| Ignore-and-negate | a coordination signal stops being tracked | silent, invisible, correctness-breaking |

This FR takes the loud-failure polarity. Approved via `/brainstorm` 2026-07-15 (Approach 1).

## Acceptance Criteria

- AC-STE-383.1: **Idempotent toolkit-owned write.** A new helper composes and writes `<root>/.dpt/.gitignore` with the canonical body (`ledger/` + `scratch/`), creating `.dpt/` when absent. Idempotent by byte-compare on the STE-303 pattern: baseline identical to proposal â‡’ no write, no prompt; safe to re-run. {#AC-STE-383.1}
- AC-STE-383.2: **`/setup` invokes it.** `/setup` calls the helper on every run (existence-guarded, no-op on match), and the write is recorded in the `## /setup audit` surface per the STE-133 best-effort-and-log precedent â€” a failure here never fails `/setup`. {#AC-STE-383.2}
- AC-STE-383.3: **Root append retired.** `/setup`'s `.dev-process/` root-`.gitignore` append (AC-STE-344.5; `skills/setup/SKILL.md:158` prose + its implementation) is removed. `/setup` no longer writes to the consumer's root `.gitignore` for toolkit state at all. {#AC-STE-383.3}
- AC-STE-383.4: **Toolkit's own root `.gitignore`.** The `.dev-process/` line is removed from this repo's root `.gitignore`, and **no `.dpt/` line is added** â€” the nested `.dpt/.gitignore` governs. A root entry would defeat it, since git never descends into an excluded directory. {#AC-STE-383.4}
- AC-STE-383.5: **`check-ignore` assertions.** A test builds a scratch repo carrying the shipped `.dpt/.gitignore`, an empty root `.gitignore`, and one file in each subtree, then asserts via `git check-ignore`: `.dpt/locks/<id>` **tracked**, `.dpt/ledger/token-ledger.jsonl` **ignored**, `.dpt/scratch/<ulid>/spec-research-result.txt` **ignored**, `.dpt/.gitignore` itself **tracked**. A negative case asserts the naive `.dpt/` + `!.dpt/locks/` form ignores locks â€” pinning why the polarity is what it is. {#AC-STE-383.5}
- AC-STE-383.6: **Docs + accepted hazard.** `docs/layout-reference.md` and `docs/hooks-reference.md` describe the `.dpt/` tree and the tracked/ignored split. The one hole is documented explicitly: a consumer with a blanket `.dpt/` rule in their **root** `.gitignore` still wins, since git never descends into an excluded directory â€” accepted risk (the operator declined a `/setup` conflict warning at brainstorm), so it is recorded rather than defended. {#AC-STE-383.6}
- AC-STE-383.7: **Full gate green.** `bun test` from `plugins/dev-process-toolkit` passes; no probe regresses. {#AC-STE-383.7}

## Technical Design

**Canonical `.dpt/.gitignore` body** â€” deliberately relative, not rooted: nested `.gitignore` patterns resolve against their own directory, so the file is position-independent.

```gitignore
ledger/
scratch/
```

**Empirical basis** (2026-07-15, `git check-ignore` per path, root `.gitignore` empty):

| Strategy | `locks/` | `ledger/` | `scratch/` | Verdict |
|---|---|---|---|---|
| Nested `.dpt/.gitignore` (this FR) | tracked âś… | ignored âś… | ignored âś… | works with an empty root ignore |
| Root ignore-list (`.dpt/ledger/`, `.dpt/scratch/`) | tracked âś… | ignored âś… | ignored âś… | works; needs the consumer's file |
| Root `.dpt/*` + `!.dpt/locks/` | tracked âś… | ignored âś… | ignored âś… | works; one edit from the trap |
| Root `.dpt/` + `!.dpt/locks/` | **IGNORED âťŚ** | ignored | ignored | **silently kills locking** |

**Precedents.** STE-303 (`/setup` idempotent toolkit-owned-file write â€” baseline == proposal â‡’ no prompt / no write; delta â‡’ operator approves; `cancel` keeps baseline). STE-133 (`.git/hooks/commit-msg` `cp` from `templates/`, best-effort, logged to `## /setup audit` rather than failing under `bypassPermissions`). `.mcp.json` Â§ 7 merge-don't-overwrite.
