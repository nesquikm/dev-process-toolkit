## Requirement

The iter-2 Linear driver self-reported "Phase 0.5 — PASS (scratch cleared)" while the morning run's per-skill logs survived on disk (finding F2, medium) — a stale result-bearing log can false-pass the Phase 2.Y / Phase A chain-completeness checks, which have no freshness gating today. Companion doc note (finding F6, low): the child model layer denies ALL `.claude/settings.json` writes including append-only Edit, so no child-side merge path exists — the Phase 1 step 6 pre-creation must carry the FULL final allow-list.

## Acceptance Criteria

- AC-STE-358.1: The Phase 0.5 wipe is verified on disk: a post-`rm` `ls` assertion (the wiped globs yield zero survivors) replaces self-report; the wipe is widened to all per-run scratch artifacts (`.log`, `.pid`, `.rc`, aux); a meta-test pins the verification fence + widened globs.
- AC-STE-358.2: `assertChainIntegrity` gains freshness gating (a run-start parameter): a capture whose mtime predates run-start is a `capture stale (pre-run)` high-severity finding, never healthy; unit tests cover the stale-result-bearing fixture ⇒ finding and fresh ⇒ healthy cases; Phase 2.Y and conformance-loop Phase A prose pass the run-start timestamp; meta-tests pin both prose sites.
- AC-STE-358.3: Phase 1 step 6 prose documents that the child model layer denies all settings.json writes including append-only Edit — no child-side merge path exists; pre-creation must carry the FULL final allow-list; a meta-test pins the sentence.

## Technical Design

**Verified wipe (`.claude/skills/smoke-test/SKILL.md` § Phase 0.5).** Widen the rm fence to every per-run scratch class keyed on the resolved tracker — `/tmp/dpt-smoke-prompt-*.txt /tmp/dpt-smoke-<tracker>-*.log /tmp/dpt-smoke-<tracker>-*.pid /tmp/dpt-smoke-<tracker>-*.rc /tmp/dpt-smoke-<tracker>-*.start /tmp/dpt-smoke-<tracker>-*.attempt* /tmp/dpt-smoke-mcp-config-<tracker>.json` — then verify on disk in the same fence: `ls` over the wiped globs must yield zero survivors, else refuse (NFR-10) naming the survivors. The findings-file and approval-record globs stay excluded (audit-trail invariant unchanged). Self-reported "scratch cleared" without the on-disk assertion is forbidden (iter-2 F2: the report said PASS while iter-1 logs survived).

**Freshness gating (`adapters/_shared/src/smoke_child_capture.ts`).** `assertChainIntegrity` gains an optional `runStart` (epoch ms or Date): when provided, a capture whose `statSync(path).mtimeMs < runStart` yields `chainFinding(child, "capture stale (pre-run)")` before the content checks run. Callers: `/smoke-test` Phase 2.Y and `/conformance-loop` Phase A pass the run-start timestamp captured at Phase 0 acceptance. Defense-in-depth with the verified wipe — either alone closes the stale-log false-pass; both together survive a wipe bypass.

**F6 doc note (`.claude/skills/smoke-test/SKILL.md` Phase 1 step 6).** One sentence: the child model layer denies ALL `.claude/settings.json` writes — full-file Write AND append-only Edit (iter-2 confirmed no merge path exists at the child layer) — so the parent's pre-creation must carry the FULL final allow-list; children can never extend it.
