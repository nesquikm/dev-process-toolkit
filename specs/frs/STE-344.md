# STE-344: Token-usage capture layer — git-ignored ledger + SessionEnd hook + transcript parser {#STE-344}

## Requirement

The toolkit has no way to see how many tokens a given SDD skill burns. An operator running `/brainstorm`, `/spec-write`, `/implement`, `/tdd` etc. cannot tell which step is expensive (process-optimization) or roughly what a milestone cost to build (cost-awareness). This FR establishes the **capture half** of per-skill token stats — everything downstream (STE-345 render into FR files, STE-346 commit-boundary rollup) reads what this FR records.

**Empirical grounding (real transcripts, 2026-07-01).** The design rests on the observed Claude Code transcript shape, not on assumption:

- Every assistant line carries `message.usage.{input_tokens, output_tokens, cache_read_input_tokens, cache_creation_input_tokens}` and `message.model` (e.g. `claude-opus-4-8`).
- Every line carries a top-level `attributionSkill` (e.g. `dev-process-toolkit:spec-write`) + `attributionPlugin`. **Per-skill accounting is therefore a group-by `attributionSkill` × `model`** — there is no boundary-slicing problem.
- `context:fork` child skills (the four TDD forks, `spec-research`, `deps-research`) **inherit the parent's `attributionSkill`**, so they roll up into the parent aggregate for free. Verified: the `dev-process-toolkit:tdd` bucket already contains its four forks; no separate child buckets appear.
- `(no attributionSkill)` lines = main-loop / orchestrator turns outside any skill → captured under a `(main-loop)` sentinel bucket.

**Non-interference is the load-bearing constraint** (operator requirement: "it shouldn't affect other functionality at all"). The capture path writes **only** a git-ignored ledger, so it never dirties the working tree, never participates in a commit, and never blocks — it is invisible to `/gate-check`, `/pr`, `/implement` clean-tree checks, and every other skill.

## Acceptance Criteria

- AC-STE-344.1: **Ledger schema + location.** An append-only ledger at `<projectRoot>/.dev-process/token-ledger.jsonl`. Each line is one aggregate for a `(session_id, skill, model)` triple with fields `{schema, ts, session_id, git_branch, skill, model, input_tokens, output_tokens, cache_read_input_tokens, cache_creation_input_tokens, message_count}`, where `skill` is the `attributionSkill` string or the literal sentinel `(main-loop)`. A helper `ledgerPath(projectRoot)` and a typed `TokenLedgerRow` are exported from `adapters/_shared/src/token_usage.ts`. {#AC-STE-344.1}
- AC-STE-344.2: **Pure transcript parser.** `parseTranscriptTokenUsage(transcriptPath): TokenLedgerRow[]` (in `adapters/_shared/src/token_usage.ts`) reads the JSONL and, for every line carrying `message.usage`, accumulates the four token fields into a bucket keyed by `(attributionSkill ?? "(main-loop)", message.model)`; returns one row per non-empty bucket with `message_count` set. Fail-open (reusing the `templates/hooks/_lib/session.ts` posture): a missing, unreadable, or whitespace-only transcript returns `[]`; malformed individual lines are skipped, never thrown. {#AC-STE-344.2}
- AC-STE-344.3: **Fork roll-up + graceful degradation, proven by fixture.** A committed trimmed-real fixture transcript (a `context:fork` session) drives a test asserting: (a) a parent skill's row includes its `context:fork` child-skill tokens (inherited `attributionSkill`); (b) `(no attributionSkill)` lines land in the `(main-loop)` bucket; (c) when **no** line carries `attributionSkill` (older Claude Code), the parser degrades to a single `(main-loop)` bucket rather than dropping data. The `Agent`-tool subagent under-count (see Notes) is documented, not asserted. {#AC-STE-344.3}
- AC-STE-344.4: **Capture hook, idempotent per session.** A bundled hook (`templates/hooks/process/session-token-ledger.sh` + a TS module, wired into `plugins/dev-process-toolkit/hooks/hooks.json` on `SessionEnd` with `Stop` as an equivalent trigger) parses `transcript_path` from the stdin hook JSON (per the STE-290 contract, via `parseHookPayload`) and writes the session's rows to the ledger, **replacing** any existing rows whose `session_id` matches (re-fire never appends duplicates). Fail-open: any parse/IO error exits `0` with no write and no stderr gate. {#AC-STE-344.4}
- AC-STE-344.5: **`.gitignore` wiring.** `/setup` adds a `.dev-process/` entry to the consumer project's `.gitignore` (creating the file if absent; no-op if the entry is already present), existence-guarded + idempotent; and this FR adds the same entry to the toolkit's own root `.gitignore`. The bundled hook needs no `/setup` install step — `hooks/hooks.json` is auto-discovered (per M74). {#AC-STE-344.5}
- AC-STE-344.6: **Non-interference, proven by test.** A test asserts (a) `ledgerPath()` resolves under a path matched by the `.dev-process/` `.gitignore` entry (so `git status` stays clean after a hook run), and (b) the hook writes nothing outside `.dev-process/` and exits `0` even on a malformed stdin payload. No tracked file is ever modified by the capture path. {#AC-STE-344.6}

## Technical Design

**Scope: capture only.** No spec file is written or rendered here — that is STE-345 (FR body) and STE-346 (commit boundaries). This FR produces the ledger those two read.

**Files touched:**
- `plugins/dev-process-toolkit/adapters/_shared/src/token_usage.ts` — **new.** Exports `TokenLedgerRow`, `ledgerPath(projectRoot)`, `parseTranscriptTokenUsage(transcriptPath)`, and `writeSessionRows(projectRoot, sessionId, rows)` (read ledger → drop matching `session_id` → append fresh → write). Pure functions; the writer is the only IO.
- `plugins/dev-process-toolkit/templates/hooks/process/session-token-ledger.sh` — **new.** Thin stdin→module wrapper matching the existing `process/*.sh` shape (`"${CLAUDE_PLUGIN_ROOT}"` invocation); pipes hook stdin JSON to the TS module; always exits `0`.
- `plugins/dev-process-toolkit/templates/hooks/process/session-token-ledger.ts` (or `_lib/`) — **new.** Uses `parseHookPayload` from `_lib/session.ts` to read `transcript_path` + `session_id`, calls `parseTranscriptTokenUsage` + `writeSessionRows`, swallows all errors (fail-open exit 0).
- `plugins/dev-process-toolkit/hooks/hooks.json` — add a `SessionEnd` block (with `Stop` fallback) pointing at `session-token-ledger.sh`.
- `.gitignore` (repo root) — add `.dev-process/`.
- `plugins/dev-process-toolkit/skills/setup/SKILL.md` — add the idempotent `.dev-process/` `.gitignore` step.

**Reuse.** The parser reuses `session.ts`'s fail-open `readFileSync` + line-split idiom and `parseHookPayload` for the stdin contract — the same battle-tested primitives the Refusing hooks already depend on. The ledger writer is the only new IO surface, and it is confined to `.dev-process/`.

**Ledger row (illustrative):**
```jsonl
{"schema":"token-ledger/v1","ts":"2026-07-01T14:19:20Z","session_id":"9bced79a-…","git_branch":"chore/ste-344-…","skill":"dev-process-toolkit:spec-write","model":"claude-opus-4-8","input_tokens":9487,"output_tokens":7319,"cache_read_input_tokens":16432,"cache_creation_input_tokens":13974,"message_count":58}
{"schema":"token-ledger/v1","ts":"2026-07-01T14:19:20Z","session_id":"9bced79a-…","git_branch":"chore/ste-344-…","skill":"(main-loop)","model":"claude-opus-4-8","input_tokens":5673,"output_tokens":394086,"cache_read_input_tokens":0,"cache_creation_input_tokens":0,"message_count":200}
```

**Trigger choice.** `Stop` fires at end of each assistant turn and `SessionEnd` at session close; both re-derive the whole session's rows from the transcript and **replace** the session's ledger rows, so firing repeatedly is safe and idempotent. `Stop` is the robustness pick (survives an unclean exit); implementation confirms the exact event name against the live harness.

## Testing

- `tests/token-usage.test.ts` — `parseTranscriptTokenUsage` grouping by `(attributionSkill, model)`; the four token fields summed; fail-open on missing/whitespace/malformed; `(main-loop)` bucketing; degradation to `(main-loop)`-only when `attributionSkill` absent (AC-STE-344.2, .3).
- `tests/fixtures/token-usage/` — a trimmed real `context:fork` transcript proving fork roll-up into the parent bucket (AC-STE-344.3).
- Hook-module test — idempotent per-`session_id` replace (re-fire = no duplicate rows); fail-open exit 0 on malformed payload (AC-STE-344.4).
- Non-interference test — `ledgerPath()` under the `.dev-process/` ignore; no tracked-file writes (AC-STE-344.6).
- Gate: `bun test` (full gate from `plugins/dev-process-toolkit`) + `/dev-process-toolkit:gate-check` GREEN on this repo.

## Notes

- **Origin.** 2026-07-01 `/brainstorm` + `/spec-write` session. The brainstorm feared a hard "slice the transcript by skill boundary + map forked-agent files" problem; the 2026-07-01 empirical transcript analysis **dissolved it** — `attributionSkill` already tags every line, and `context:fork` children inherit the parent's tag. This FR is much simpler than the brainstorm anticipated.
- **Conscious stance reversal.** The repo previously chose "no budgets, token counts, or cost caps." This milestone deliberately reverses that for **measurement** (not enforcement — there are still no caps). The reversal is intentional and operator-approved.
- **Agent-tool subagent under-count.** `Agent`-tool subagents (e.g. `code-reviewer` spawned in `/implement` Phase 3) carry no `isSidechain`, no `attributionAgent`, and produce no `agent-*.jsonl` in this harness version — their per-message tokens are not separately attributable and are a documented under-count. `context:fork` skills are unaffected (they roll up via `attributionSkill`).
- **Version dependency.** `attributionSkill` is a harness-version-dependent field; the parser degrades to a `(main-loop)`-only ledger (never a crash) when it is absent, so older Claude Code still produces a valid, if coarse, ledger.
- **Foundation FR.** STE-345 (render + `/spec-write`) and STE-346 (commit-boundary render) both read this ledger; neither is implementable before it exists.
