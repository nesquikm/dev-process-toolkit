# STE-195: /smoke-test driver retry-with-rollback on stream-idle timeout for prompt-bearing child spawns {#STE-195}

## Requirement

When a `/smoke-test` Phase 2 child spawn (`/setup`, `/spec-write`, `/implement`) exits with the Anthropic stream-idle timeout signature (`API Error: Stream idle timeout - partial response received`), the driver attempts exactly one retry after rolling back partial state in the test project via `git clean -fdq -e .claude -e .mcp.json && git checkout -- .`. The rollback preserves driver-pre-created sensitive files (`.claude/settings.json`, `.mcp.json`) that the parent placed before invoking the child. A second consecutive timeout aborts the smoke run with NFR-10 canonical refusal naming the skill, both attempt timestamps, and the recipe.

The 2026-05-04 Jira smoke run hit the failure mode on `/setup`'s first attempt (partial state created `src/.placeholder.test.ts` but no `CLAUDE.md`, no `specs/` scaffold). The driver recovered manually using the same rollback recipe; building it in turns a smoke-blocker into a quiet retry.

Out of scope:

- More than one retry. Two attempts is the budget; the third instance is genuine and should surface as a smoke-test failure, not loop indefinitely.
- Retry-with-rollback for non-stream-idle exit signatures (segfault, OOM kill, etc.). Stream-idle timeout is the one Anthropic-API transient class with a known deterministic rollback; other exit modes warrant operator inspection, not auto-retry.
- Retry without rollback. Re-spawning a `/setup` child on a partially-scaffolded test project would likely fail or produce inconsistent state — the rollback is the deterministic precondition.

## Acceptance Criteria

- AC-STE-195.1: `.claude/skills/smoke-test/SKILL.md` Phase 2 child-spawn block documents the stream-idle timeout exit signature (`API Error: Stream idle timeout` substring in stderr or exit reason) and the retry-with-rollback recipe verbatim. {#AC-STE-195.1}
- AC-STE-195.2: On detection of the signature, the driver runs `git clean -fdq -e .claude -e .mcp.json && git checkout -- .` inside the test project's working directory (NOT the dpt repo cwd) before the second spawn attempt. The `-e .claude -e .mcp.json` excludes preserve the parent-pre-created sensitive files. {#AC-STE-195.2}
- AC-STE-195.3: Retry path emits a `child_stream_idle_retried` row in the Phase 2 per-skill log (`/tmp/dpt-smoke-<tracker>-<skill>.log`) with both attempt timestamps (UTC ISO-8601). {#AC-STE-195.3}
- AC-STE-195.4: Second consecutive timeout (retry also stream-idles) aborts the smoke run with NFR-10 canonical refusal naming: (a) the skill that timed out twice, (b) both attempt timestamps, (c) the rollback recipe operators can run manually if the third attempt is appropriate. {#AC-STE-195.4}
- AC-STE-195.5: Manual smoke chain validation. Operator-deferred — the failure mode is non-deterministic (depends on Anthropic API stream behavior). AC ships `[~]`; M54 retroactive validation when a real stream-idle surfaces. {#AC-STE-195.5}

## Technical Design

| File | Change |

### Detection signature

The driver inspects the child's exit reason / stderr for the substring `API Error: Stream idle timeout`. Match is substring (not exact); tolerates the trailing `- partial response received` and any minor wording drift in future Anthropic API versions.

### Rollback scope

The recipe runs inside the **test project's** working directory (e.g., `../dpt-test-project-linear`). It does NOT run inside the dpt repo cwd. The driver's per-spawn cwd handling already isolates the test project; the rollback inherits that scope.

### Recipe verbatim

```bash
git clean -fdq -e .claude -e .mcp.json && git checkout -- .
```

- `git clean -fdq -e .claude -e .mcp.json` — removes untracked files/directories EXCEPT `.claude/` and `.mcp.json`.
- `git checkout -- .` — reverts tracked-file modifications.
- Combined: returns the test project to its last-committed state plus the parent-pre-created sensitive files.

### Failure-mode surface in Phase 2 log

On retry success, append:

```
2026-05-04T06:42:11Z child_stream_idle_retried skill=/setup attempts=2
  attempt_1_started=2026-05-04T06:40:07Z attempt_1_exit=stream_idle
  attempt_2_started=2026-05-04T06:42:33Z attempt_2_exit=success
```

On double-timeout abort, NFR-10 canonical refusal text:

```
ABORT: /smoke-test Phase 2 spawn /<skill> stream-idle timeout twice
  attempt_1_started=<ts1> attempt_1_exit=stream_idle
  attempt_2_started=<ts2> attempt_2_exit=stream_idle
  rollback recipe: git clean -fdq -e .claude -e .mcp.json && git checkout -- .
```
