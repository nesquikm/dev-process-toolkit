# STE-215: /implement propagates file-removal to cross-cutting specs (technical-spec.md, testing-spec.md) {#STE-215}

## Requirement

When `/implement` Phase 4 removes a file (most commonly `src/.placeholder.test.ts`, the M1 placeholder shield deleted by the first real test), it updates `specs/plan/M<N>.md` (task checkbox + tasks block) and the `## 6 Traceability matrix` in `specs/requirements.md`, but it does NOT propagate the file removal to `specs/technical-spec.md` (directory-tree block + prose) or `specs/testing-spec.md` (directory-tree block + "What NOT to Test" section). The result is durable cross-cutting drift: the cross-cutting specs keep referencing files that no longer exist on disk.

**Findings bundled:** smoke #6 F4, smoke #8 F5, B-F3 from `/tmp/dpt-smoke-findings-2026-05-04-jira.md` (Run B v2.7.0). 4× recurring across smoke runs; the dev-process-toolkit's own `specs/technical-spec.md:13` + `specs/testing-spec.md:18` directory trees still list `.placeholder.test.ts` even though the placeholder shield was deleted in commit `9aabd6c`.

**Symptoms observed across multiple smoke runs:**

1. `specs/technical-spec.md:13` directory tree still lists `.placeholder.test.ts`.
2. `specs/technical-spec.md:53` prose still says "stays until the first real test ships".
3. `specs/testing-spec.md:18` directory tree still lists `.placeholder.test.ts`.
4. `specs/testing-spec.md:33` "What NOT to Test" section still mentions the placeholder.
5. Re-confirmed across smoke #6, smoke #8, and 2026-05-04 Jira Run B — without `/implement` propagating, the drift accumulates and is fixed only by manual sweeps.

## Acceptance Criteria

- AC-STE-215.1: After `/implement` Phase 4 commits a file deletion (any tracked file the implementation removed), it scans `specs/technical-spec.md` and `specs/testing-spec.md` for references to the removed path. Detection includes: lines inside triple-backtick code fences (directory-tree blocks) containing the basename or relative path of the removed file; prose mentions of the relative path. {#AC-STE-215.1}
- AC-STE-215.2: When references are found, `/implement` updates each affected spec file: directory-tree lines containing only the removed basename are deleted; prose lines that name the removed path are flagged for human review (commented-out diff lines preserved with a `# REMOVED: <path>` marker so the operator can decide whether the surrounding sentence still reads correctly post-deletion). Conservative editing: the propagation never silently rewrites prose, only cleans the directory-tree block where the path appears as a leaf entry. {#AC-STE-215.2}
- AC-STE-215.3: A single follow-up commit lands the propagation: `chore(specs): propagate <removed-path> removal to cross-cutting specs`. The commit fires after the file-removal commit but before the archival commit (Phase 4 step 16, between the implementation-commit step 15 and the milestone-archival step 17). {#AC-STE-215.3}
- AC-STE-215.4: When references are NOT found in either cross-cutting spec, the propagation step is a silent no-op (no follow-up commit, no closing summary row). The propagation only fires when at least one match is found in at least one of the two cross-cutting spec files. {#AC-STE-215.4}
- AC-STE-215.5: New `/gate-check` probe `cross-cutting-spec-stale-file-refs` scans `specs/technical-spec.md` and `specs/testing-spec.md` for path-references in directory-tree blocks that don't exist on disk; flags as ADVISORY (not GATE FAILED) on any miss. The probe is the safety net for paths that bypass `/implement`'s propagation step (manual deletions, `git rm`, downstream toolkit consumers). {#AC-STE-215.5}
- AC-STE-215.6: Test fixture: an `/implement` run that deletes `src/.placeholder.test.ts` produces clean technical-spec.md + testing-spec.md (no stale path refs in directory-tree blocks). The fix-it-now sweep also clears the existing 4 stale refs in this repo's own `specs/technical-spec.md:13` + `specs/testing-spec.md:18` (those edits ship as part of M55's implementation, not as a separate cleanup commit). {#AC-STE-215.6}

## Technical Design

**Phase 4 hook.** `/implement` Phase 4 file-removal step is the natural integration point: the implementer has just committed `git rm <path>` (or equivalent), the tree state is known, and the next step (archival) hasn't fired yet.

**Detection helper.** Add `scanCrossCuttingSpecRefs(removedPath, specsDir)` (location TBD — likely `adapters/_shared/src/`):

```typescript
function scanCrossCuttingSpecRefs(removedPath: string, specsDir: string): {
  technicalSpec: { line: number, snippet: string }[],
  testingSpec: { line: number, snippet: string }[],
} {
  // grep technical-spec.md and testing-spec.md for removedPath OR basename(removedPath)
  // distinguish: inside triple-backtick fence (directory-tree leaf) vs. prose mention
  // return per-file match list
}
```

**Edit policy.**
- **Directory-tree leaf** (line inside triple-backtick fence, content is a tree-character-prefixed leaf naming the path): delete the line.
- **Prose mention**: flag in the propagation commit message but leave untouched (prose context is unsafe to auto-rewrite — surrounding sentences may need restructuring). The commit message lists prose lines for the operator's awareness; the operator amends in a follow-up commit if needed.

**Probe `cross-cutting-spec-stale-file-refs`.** Walk `specs/technical-spec.md` and `specs/testing-spec.md`; for each tree-leaf line inside a triple-backtick fence, resolve to a relative path (using common-prefix conventions for the directory-tree blocks); assert the path exists on disk. Missing paths surface as ADVISORY rows.

## Testing

- Unit test: `scanCrossCuttingSpecRefs("src/foo.ts", specsDir)` against a fixture spec set returns the right tree-leaf and prose-mention sets.
- Unit test: directory-tree leaf deletion is idempotent (re-running on already-cleaned tree is a no-op).
- Integration test: `/implement` fixture with a placeholder-shield deletion exercises the propagation path; assert the follow-up commit lands; assert technical-spec.md / testing-spec.md show the leaf deleted.
- Smoke regression: post-fix smoke runs no longer surface technical-spec.md / testing-spec.md stale refs in their `/spec-review` output.
