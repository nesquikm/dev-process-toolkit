## Requirement

The iter-2 healthy `/setup` grandchild surfaced finding F5 (low): the documented Jira status-vocabulary fetch (`getJiraIssueTypeMetaWithFields` → `status.allowedValues`, locked by STE-303) returns no `allowedValues` for team-managed (next-gen, `"simplified": true`) projects; the child recovered by reading issue transitions. The transitions-derived path should be documented as the primary fallback — the exact candidate STE-303's deferred open question listed.

## Acceptance Criteria

- AC-STE-361.1: `adapters/jira.md`'s status-fetch section documents `getJiraIssueTypeMetaWithFields.allowedValues` as company-managed-only and the transitions-derived path (`getTransitionsForJiraIssue` → `to.name` / `to.statusCategory`) as primary for team-managed projects, with the probe order; the AC-STE-303.3 output contract (ordered verbatim string array) is preserved; the jira-md doc-conformance test family pins the section.
- AC-STE-361.2: Any skill-prose reference to the allowedValues fetch path (e.g., /setup step 7f tracker-config) names the team-managed fallback ordering; a meta-test pins it.

## Technical Design

**adapters/jira.md status-fetch section.** Rewrite the `list_project_statuses` documentation to a two-path probe order: (1) detect project style — `getVisibleJiraProjects` response carries `"simplified": true` for team-managed (next-gen) projects; (2) company-managed ⇒ `getJiraIssueTypeMetaWithFields` → `status.allowedValues` (the STE-303 path, unchanged); (3) team-managed ⇒ transitions-derived: `getTransitionsForJiraIssue` against a representative issue (or the project's create-meta default issue type when no issue exists) → collect `transitions[].to.name` in API order, dedup preserving first occurrence. Both paths return the AC-STE-303.3 contract shape — ordered string array, verbatim casing/whitespace — so `tracker-config.yaml` composition (STE-302) is untouched. Document the zero-issue edge (fresh team-managed project with no issues: fall back to the three statusCategory names with a `tracker_config_write_mcp_unavailable`-style advisory).

**Skill-prose sweep.** Grep skills/docs for references to the allowedValues fetch (`/setup` step 7f tracker-config, `docs/setup-tracker-mode.md` if present) and add the team-managed fallback ordering sentence at each site.

## Testing
