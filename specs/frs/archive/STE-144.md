# STE-144: Rename mode-none-regression fixture references → mode-none-baseline {#STE-144}

## Requirement

`specs/testing-spec.md:59,144` references `tests/fixtures/mode-none-regression/` as a Tier-3 regression fixture. The same reference appears in `specs/frs/archive/STE-86.md:93` and `specs/frs/archive/STE-76.md:33,78`. The fixture tree under `plugins/dev-process-toolkit/tests/fixtures/projects/` contains `mode-none-baseline`, `mode-none-archived`, `mode-none-fresh-setup`, but NO `mode-none-regression`. The closest existing fixture is `mode-none-baseline`. The references describe a baseline regression suite that simply isn't wired up — it's a documentation lie a contributor following the spec would trip over.

A contributor adding a new Tier-3 regression test would either create the missing fixture (duplicating `mode-none-baseline`) or chase ghosts. Fix by renaming the spec references to point at the real fixture.

Archived FRs (`STE-86.md`, `STE-76.md`) are immutable per the archive-immutability invariant — leave as-is, even though they carry the same drift. The active spec (`testing-spec.md`) is the surface that matters.

## Acceptance Criteria

- AC-STE-144.1: `specs/testing-spec.md:59` no longer references `tests/fixtures/mode-none-regression/`. The reference is updated to `tests/fixtures/projects/mode-none-baseline/` (or the canonical relative path used elsewhere in the spec).
- AC-STE-144.2: `specs/testing-spec.md:144` carries the same update — same canonical fixture name.
- AC-STE-144.3: Archived FRs (`specs/frs/archive/STE-86.md:93` and `specs/frs/archive/STE-76.md:33,78`) are NOT modified — archive immutability holds.
- AC-STE-144.4: `/gate-check` clean across the live repo. Specifically, the traceability-link probe (#23) returns zero violations for the affected lines (the new path resolves to a real fixture directory).
- AC-STE-144.5: `bun test` is green.
- AC-STE-144.6: A `grep -rn "mode-none-regression"` across `specs/` (excluding `specs/frs/archive/`) returns zero hits after the edit.

## Technical Design

Two-line edit in `specs/testing-spec.md`:

- Line 59: replace `mode-none-regression` → `mode-none-baseline` (path canonicalized to `tests/fixtures/projects/mode-none-baseline/` or whatever shape the surrounding prose uses).
- Line 144: same replacement.

Verify the surrounding prose doesn't reference behaviors specific to a "regression" framing — if line 59's sentence reads "the regression suite asserts byte-for-byte parity," consider whether the `mode-none-baseline` fixture supports the same assertion. If yes, the rename suffices; if no, the prose needs a one-sentence rewrite to match the actual fixture's contract.
