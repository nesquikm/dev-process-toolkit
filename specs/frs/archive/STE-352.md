## Requirement

Give `/smoke-test` the ability to detect the M94 symptom class — a child spawn that produced no output yet was reported green. The F2 text-mode capture blind spot (which hid F1) and the absence of a 0-byte-grandchild check are why the STE-350 bug survived weeks of green runs.

## Acceptance Criteria

- AC-STE-352.1: `/smoke-test` captures child output via `--output-format stream-json` (or otherwise guarantees per-probe rows and forked `tdd-result` fences are parseable), closing the F2 text-mode blind spot where the final message alone hid nested tokens.
- AC-STE-352.2: `/smoke-test` asserts each spawned child produced non-empty, non-denied output — a child whose log is 0 bytes, or whose result JSON carries a `permission_denials` entry for a `claude` spawn, is a hard (high-severity) finding, not a silent pass.
- AC-STE-352.3: A Phase 2.X-style regression fixture reproduces a nested `claude -p` spawn and asserts it completes with non-empty output under the patched allow-list; with the pattern removed the fixture fails (the runtime counterpart to STE-351.2's static probe).

## Technical Design

- **stream-json capture:** the canonical-chain children currently redirect stdout to `.log` in default text format (`smoke-test/SKILL.md:350-417`), so only the final message is captured — per-probe rows and `tdd-result` fences are absent (smoke F2). Switch the capture for the children whose nested tokens matter to `--output-format stream-json` and parse the NDJSON via the existing `parseStreamJsonTranscript` (`adapters/_shared/src/socratic_first_turn_stream.ts`), reused from Phase 8, which already proves the pattern.
- **non-empty / non-denied assertion:** after each child returns, assert `wc -c` on its log is `> 0` and the result JSON has no `permission_denials[].tool_input.command` matching a `claude` spawn. Either condition ⇒ emit a high-severity finding with a canonical diagnostic (`STE-350 regression: nested claude -p spawn denied/empty — <child>`). This is the direct detector for the exact 0-byte-grandchild symptom.
- **regression fixture:** add a Phase 2.X group (same shape as the STE-231 groups) that reproduces a nested spawn and asserts non-empty completion under the allow-list; a negative variant (pattern absent) asserts the denial is caught. Capture artifacts under `tests/fixtures/`.

## Testing

- `parseStreamJsonTranscript` over a captured stream-json fixture yields the expected probe rows / `tdd-result` fences (unit).
- non-empty/denial assertion: fixture with a 0-byte log ⇒ finding; fixture whose result JSON has a `permission_denials` entry for `claude` ⇒ finding; healthy log ⇒ pass (unit).
- Phase 2.X fixture (runtime, smoke-validated; may ship `[~]` pending a live run).

## Notes

FR3 closes the observability gap that let STE-350 hide: green checkmarks with no runtime evidence. It is the runtime complement to STE-351's static fence — the probe catches the config omission before a run, this catches a denied/empty spawn during a run. Relates to the 2026-05-26 Linear smoke findings (F1 probes LLM-emulated + F2 text-mode capture hid the finding).
