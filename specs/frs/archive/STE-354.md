## Requirement

`/smoke-test` Phase 0.5's stale-scratch cleanup must never delete the concurrent other leg's live artifacts. STE-186 (M52) deliberately widened the cleanup glob to `dpt-smoke-mcp-config-*.json` for stale-config cleanup; the 2026-07-02 tandem run proved the cross-tracker glob races the other leg's Phase 1 step 5 write — the Jira leg deleted the Linear leg's live MCP config seconds after it was written, and the first `/setup` child spawn failed with "MCP config file not found" (conformance finding F1, high). This contradicts § Operator-driven parallelism's isolation invariant ("never read the same MCP config … never touch this one").

## Acceptance Criteria

- AC-STE-354.1: Phase 0.5 removes only `dpt-smoke-mcp-config-<tracker>.json` (the resolved tracker's own file); no cross-tracker `mcp-config-*` glob remains in the cleanup block; a meta-test pins the scoped form and fails on any cross-tracker glob.
- AC-STE-354.2: The STE-186 stale-cleanup intent is preserved and documented as compatible — each leg cleans its own stale config, so staleness coverage is unchanged; § Operator-driven parallelism prose and the Phase 0.5 cleanup line agree; a meta-test asserts both.

## Technical Design

One-line change in `.claude/skills/smoke-test/SKILL.md` § Phase 0.5: the cleanup command drops `/tmp/dpt-smoke-mcp-config-*.json` in favor of `/tmp/dpt-smoke-mcp-config-<tracker>.json` (the resolved tracker token, consistent with the per-skill log glob `dpt-smoke-<tracker>-*.log` on the same line). The surrounding justification prose (smoke #9 / Linear F1 note, which documented the both-variants glob as a feature) is rewritten to state the per-tracker scoping and why: the cross-tracker form races a concurrent tandem leg's Phase 1 step 5 write (2026-07-02 F1). § Operator-driven parallelism gains one sentence noting Phase 0.5 cleanup is per-tracker-scoped by the same isolation invariant. Meta-test lands in the existing smoke-test conformance family under `plugins/dev-process-toolkit/tests/` (regex over the Phase 0.5 fence: the scoped literal present, the cross-tracker glob absent).

## Testing

- Meta-test: Phase 0.5 fence contains `dpt-smoke-mcp-config-<tracker>.json` and does NOT contain `dpt-smoke-mcp-config-*.json` (AC-STE-354.1).
- Meta-test: § Operator-driven parallelism carries the per-tracker cleanup sentence; the STE-186 staleness rationale is retained in the Phase 0.5 prose (AC-STE-354.2).
- Runtime (smoke-validated on the next tandem run): both legs complete Phase 1 without an mcp-config-not-found failure.
