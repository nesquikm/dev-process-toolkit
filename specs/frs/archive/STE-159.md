# STE-159: /smoke-test parallel-safe artifact isolation {#STE-159}

## Requirement

`/smoke-test` (project-local skill at `.claude/skills/smoke-test/SKILL.md`) is single-tracker per invocation but hard-codes every per-run artifact to a tracker-agnostic name: the test-project path is `../dpt-test-project` (basename hard-coded; pre-flight #6 enforces it), and `/tmp/dpt-smoke-*.{md,log,json,txt}` artifacts have no tracker suffix. Running Linear and Jira runs back-to-back is therefore **wall-clock serial**: ~10 minutes each, ~20 minutes total per release-candidate validation. Pre-flight #2 (`../dpt-test-project` already exists) hard-fails the second invocation if the first hasn't torn down yet.

The brainstorm decision (2026-04-30) selected operator-driven parallelism — two terminals, two `--tracker` values, no driver fan-out and no merged findings. To make that safe, every per-run artifact must be tracker-keyed so the two runs cannot collide on the filesystem or on stdout-redirected `/tmp` paths. No combined-mode flag, no parent-side fan-out, no merged-findings format — those were considered (approaches 2 and 3) and explicitly rejected as over-engineering for the wall-clock target.

Out of scope: any change to the canonical chain (Phases 0–5 stay byte-identical per tracker), any change to budget caps (each tracker run still has its own `--max-budget-usd 3` per skill), and rate-limit measurement (operator's problem; surfaces in the per-run trace if it bites).

## Technical Design

Single-file edit to `.claude/skills/smoke-test/SKILL.md`. Six touch-points:

1. **Argument parsing** (top of skill) — unchanged. `--tracker` already exists.
2. **Pre-flight #2** — replace literal `../dpt-test-project` with `../dpt-test-project-<resolved-tracker>`. Refusal message names the per-tracker path.
3. **Pre-flight #6 reference impl** — change `TEST_PATH="../dpt-test-project"` to `TEST_PATH="../dpt-test-project-${TRACKER}"`, change the basename-equality check to membership in `{dpt-test-project-linear, dpt-test-project-jira}`. Update the "verified 2026-04-27 against six adversarial cases" comment to note the M46 expansion (the canonical-good case now has two valid forms).
4. **Phase 1 step 1, step 5, step 6** — substitute the per-tracker basename and per-tracker `/tmp` paths.
5. **Phase 2** — per-skill log redirection picks up the tracker suffix.
6. **Phase 3 / Output / Phase 5 teardown** — findings filename, approval log filename, teardown checklist all reflect the renamed paths.

New "Operator-driven parallelism" subsection added after the "When to use" block, stating the supported parallel mode and the rate-limit caveat (one Anthropic key, two children billing concurrently, expected win <2×).

Rationale for not generalizing further: a `--tracker linear,jira` combined flag would force the driver to either fan out parent-side (approach 3 from brainstorm — high complexity for marginal gain over two terminals) or sequentialize Phase 2 chains (no win at all). Two terminals is mechanically simpler, gives the operator independent abort control per tracker, and isolates failure modes (a Linear-side hang doesn't stall the Jira chain). The tradeoff against approach 2 (one driver, fan-out in Phase 2 only) is real: approach 2 would give one approval and one teardown checklist, but at the cost of console-multiplexing logic, prefixed phase printing, and a per-chain budget cap rewrite. Approach 1's cost is one more terminal window — accepted.

## Testing

The skill is its own test. Verify by:

1. `/smoke-test --tracker linear` solo — runs into `../dpt-test-project-linear`, all artifacts carry `-linear` suffix.
2. `/smoke-test --tracker jira --jira-project DST` solo — runs into `../dpt-test-project-jira`, all artifacts carry `-jira` suffix.
3. **Tandem run** (AC-STE-159.6) — both invocations within seconds of each other; both canonical chains complete; no filesystem collision; two trace files land.

Adversarial probe of pre-flight #6's expanded allow-list: invoke with the path forced to `../dpt-test-project` (legacy bare basename), `../dpt-test-project-foo`, and `../dpt-test-project-LINEAR` (case mismatch) — all three must refuse with NFR-10 shape. The two canonical-good forms must pass.

## Notes

- The bare `../dpt-test-project` basename is intentionally **dropped** rather than preserved as a Linear-mode default. Per `project_no_users_yet.md`: the plugin has no installs and the smoke-test skill is project-local (the maintainer is the only operator). Back-compat scaffolding for the old basename is dead weight.
- Tandem-run failure modes worth surfacing in the trace: (a) Anthropic rate-limit throttling slowing one or both children below the solo-run baseline; (b) Linear/Jira MCP-side rate limits if both chains hit the same provider concurrently (Linear-only / Jira-only by design — the two runs target different MCP servers, so this is a hypothetical not an expected hit).
