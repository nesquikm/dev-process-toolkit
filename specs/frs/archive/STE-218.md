# STE-218: Remove ## smoke-test runs reference table; inline F-DR rationale into SKILL.md body {#STE-218}

## Requirement

`.claude/skills/smoke-test/SKILL.md` carries a `## smoke-test runs` reference table at the bottom of the file (lines 669–684 in the v2.7.0 shape) listing every smoke run chronologically with date, plugin version, tracker mode, outcome, and follow-up milestone. The body of the same SKILL.md carries 9 inline `run #<N> F-DR<X>` references (lines 12, 145, 222, 230, 298, 299, 300, 303 — exact line numbers verify pre-fix) that anchor driver-side workarounds to specific historical incidents documented in the table. User review during the M55 brainstorm concluded the table accretes per-run noise without earning its keep — readers don't navigate to it; per-run findings files in `/tmp/dpt-smoke-findings-<date>-<tracker>.md` are the persistent audit trail; the F-DR codes are jargon that requires a lookup-table to decode.

**Findings bundled:** user-requested during M55 brainstorm session (2026-05-04). The reasoning: "I don't think that it can be helpful at all" — the table is a duplicate index whose contents are fully captured in per-run findings files.

**Symptoms / cost:**

1. The runs table grew to 11+ rows over the project's lifetime; each ship adds a new row, growing the table without bounded retention.
2. The 9 inline `run #<N> F-DR<X>` references are unreadable without cross-referencing the table — the F-DR code (e.g., `F-DR3`) is a tag with no meaning outside the table's context.
3. Removing the table without inlining would orphan the 9 references; readers would see `run #1 F-DR3` and have no way to know what constraint that documents.
4. The fix is to delete the table AND replace each inline reference with a self-contained one-liner that explains the constraint without needing the F-DR code.

## Acceptance Criteria

- AC-STE-218.1: The `## smoke-test runs` section is removed from `.claude/skills/smoke-test/SKILL.md`. The deletion includes: the section heading; the introductory paragraph ("Reference runs in chronological order…"); the table header; every table row; any trailing notes specific to the runs table. After deletion, the file's last section is whatever section preceded `## smoke-test runs` (likely `## Threat model` or the last canonical-chain section). {#AC-STE-218.1}
- AC-STE-218.2: Each of the 9 inline `run #<N> F-DR<X>` references in the body is replaced with a self-contained one-liner explaining the constraint. After substitution: zero `F-DR` codes remain in the file; zero `run #<N>` references remain in the file. The 9 reference sites cover four distinct constraints (F-DR1: claude-st alias scope; F-DR2: disable-model-invocation skill loading; F-DR3: --plugin-dir shadows plugin-loaded MCPs; F-DR7: sensitive-path classification at child model layer); each substitution carries a 1–2 sentence inline explanation. {#AC-STE-218.2}
- AC-STE-218.3: After substitution, the SKILL.md body stands alone — reading top-to-bottom motivates every driver-side workaround without needing the runs table or the F-DR codes. The introduction sentence at line 12 ("See § smoke-test runs at the bottom of this file for the full reference-run list") is removed. {#AC-STE-218.3}
- AC-STE-218.4: Cross-references inside the file that point to `## Threat model` or other intra-file sections are preserved (those are intra-file section references, not run-table references). The deletion is scoped to the `## smoke-test runs` section and the 9 `run #<N> F-DR<X>` reference sites; nothing else changes. {#AC-STE-218.4}
- AC-STE-218.5: Test fixture: `grep -c "F-DR\|run #[0-9]\|Run #[0-9]\|## smoke-test runs" .claude/skills/smoke-test/SKILL.md` returns `0` post-fix. (The `## smoke-test runs` heading itself is removed; the `run #N` and `F-DR` patterns are scrubbed from the body.) {#AC-STE-218.5}
- AC-STE-218.6: Smoke driver behavior unchanged — the runs table was reference-only documentation; no skill logic, no `/smoke-test` invocation step, and no probe consults it. A `/smoke-test` run post-fix produces byte-identical Phase 0–5 behavior to a pre-fix run. {#AC-STE-218.6}

## Technical Design

**Pre-fix audit.** Before editing, locate every `run #<N>` and `F-DR` token in the file:

```
grep -n "F-DR\|run #[0-9]" .claude/skills/smoke-test/SKILL.md
```

This yields the 9 reference sites (line numbers approximate; may shift slightly between v2.7.0 and the implementation date — the recognizer is the grep, not the line numbers).

**Substitution table** (rationale derived from the surrounding prose at each site, NOT from the runs-table cells — the table is being deleted):

| F-DR code | Constraint | Inline replacement (illustrative) |
|-----------|------------|-----------------------------------|
| F-DR1 | The `claude-st` zsh alias does not expand inside the parent harness's Bash tool | "the parent invokes `CLAUDE_CONFIG_DIR=~/.claude-st claude -p …` directly because the `claude-st` zsh alias does not expand inside the parent harness's Bash tool" |
| F-DR2 | Plugin skills carry `disable-model-invocation: true`, which blocks the child's model from invoking them via the Skill tool; only user-typed slash commands trigger them | "Plugin skills carry `disable-model-invocation: true`, so the child's model cannot call them via the Skill tool — only user-typed slash commands trigger; the prompt-pre-bake puts the slash command as the literal first line of the user message" |
| F-DR3 | `--plugin-dir` (used to load the in-tree plugin under test) shadows plugin-loaded MCPs | "`--plugin-dir` (used to load the in-tree plugin under test) shadows plugin-loaded MCPs, so the active tracker MCP must be passed via `--mcp-config` from a per-tracker wrapper file written to `/tmp/`" |
| F-DR7 | The harness's sensitive-path classification of `.claude/settings.json` and `.mcp.json` survives `bypassPermissions` at the child's model layer; parent-side shell I/O via Bash heredoc is not subject to it | "the harness's sensitive-path classification of `.claude/settings.json` and `.mcp.json` survives `bypassPermissions` at the child's model layer, so the parent's Bash tool pre-creates them via heredoc (shell I/O is not subject to that classification)" |

The actual edit at each site preserves the surrounding sentence's structure — only the `run #<N> F-DR<X>` parenthetical or comma-clause is replaced. The 9 sites land 4 distinct rationales (some F-DR codes appear at multiple sites referencing the same constraint).

**Deletion of the runs table.** After the 9 substitutions, delete the `## smoke-test runs` section (heading + intro paragraph + table). Verify the file ends cleanly at whatever section preceded.

**Verification.** Post-edit, run the AC-STE-218.5 grep; the result must be `0`. Manual review of the file top-to-bottom confirms every driver-side workaround now reads coherently without lookup.
