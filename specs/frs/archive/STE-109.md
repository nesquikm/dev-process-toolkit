## Requirement

After v1.29.0's `/setup` + `/spec-write` + `/implement` + `/spec-archive` ran end-to-end on a fresh project, the working tree still had the /setup outputs untracked:

```
?? .claude/
?? .mcp.json
?? CLAUDE.md
?? specs/technical-spec.md
?? specs/testing-spec.md
```

`/setup` doesn't commit; `/implement` only commits FR-scoped changes; the bootstrap artifacts drift further from HEAD with every feature shipped. The user has no canonical moment to commit them, and they leak into the first feature PR — confusing reviewers.

The fix: `/setup` produces a single bootstrap commit at the end of its run (after writing all outputs), prompting the user for approval per the skill's `Human approval before commit` Core Principle.

## Acceptance Criteria

- AC-STE-109.1: After /setup writes its outputs, it stages the canonical /setup output set (`CLAUDE.md`, `.claude/settings.json`, `.mcp.json` if written, `specs/requirements.md`, `specs/technical-spec.md`, `specs/testing-spec.md`, `specs/plan/M1.md`, plus the empty `specs/frs/{,archive/}` and `specs/plan/archive/` dirs via `.gitkeep`) and prompts the user with the diff: "Commit these as `chore: bootstrap dev-process-toolkit`? (y/n/edit)".
- AC-STE-109.2: On `y`, the commit lands. On `n`, /setup logs a warning to its final report and to a new `## /setup audit` entry (per STE-108) reminding the user to commit the bootstrap before /spec-write. On `edit`, /setup hands control to the user with the files staged but uncommitted.
- AC-STE-109.3: In autonomous mode (no human present), /setup proceeds with the commit using the default message — this is the "default" mode for the bootstrap commit, not the "requires-input" mode (the bootstrap is mechanical; the diff is the entire /setup output and is reviewable post-commit).
- AC-STE-109.4: If the project is not a git repo, /setup runs `git init` first (per existing skill prose) and then commits. If the working tree has unrelated uncommitted changes when /setup starts, /setup refuses to commit (NFR-10 shape: "uncommitted changes exist; resolve before /setup") rather than risk bundling unrelated work.
- AC-STE-109.5: The commit message is `chore: bootstrap dev-process-toolkit (v<plugin-version>)`. The version comes from `plugins/dev-process-toolkit/.claude-plugin/plugin.json`. Body lists the files written, not the contents.
- AC-STE-109.6: `skills/setup/SKILL.md` adds a "## After /setup" section that documents this commit step and the conditions under which it runs.
- AC-STE-109.7: New `/gate-check` probe `setup-bootstrap-committed`: if CLAUDE.md is toolkit-managed (per the `<!-- generated by ... -->` marker from STE-108) AND the file is currently in the working tree, it MUST be committed (no `??` in `git status`). Probe lives at `plugins/dev-process-toolkit/adapters/_shared/src/setup_bootstrap_committed.ts` (flat path matches existing convention); integration test at `plugins/dev-process-toolkit/tests/gate-check-setup-bootstrap-committed.test.ts` per STE-82.
- AC-STE-109.8: Test fixture covers: clean repo (commit lands), repo with unrelated uncommitted changes (abort with NFR-10), non-repo (git init + commit), autonomous mode with default flags (commit lands without prompt).
- AC-STE-109.9: `bun test` green. CHANGELOG `### Added` entry cross-references this FR.

## Technical Design

**Skill change.** `skills/setup/SKILL.md` adds a Phase 8 (or similar terminal phase): "Bootstrap commit". The phase runs after all writes and the audit log are populated.

**Bootstrap file list.** Hardcoded in the skill prose (matches the canonical /setup output set); not parameterized. Adding a new file to /setup's output requires updating this list — caught by the new gate-check probe if missed.

**`.gitkeep` for empty dirs.** /setup writes `.gitkeep` files in `specs/frs/`, `specs/frs/archive/`, `specs/plan/archive/` so they survive the initial commit.

**Pre-flight check.** Before the commit, /setup runs `git status --porcelain` and ensures the only untracked or modified files are in the bootstrap set. Anything else → abort.

## Testing

`plugins/dev-process-toolkit/tests/gate-check-setup-bootstrap-committed.test.ts` per STE-82. End-to-end test that runs /setup against a fixture project and asserts the resulting commit shape.
