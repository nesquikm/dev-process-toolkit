## Requirement

The v1.29.0 smoke test ran the full chain `/spec-write → /implement → /spec-archive` against `STE-102`. After /spec-archive moved `specs/frs/STE-102.md` → `specs/frs/archive/STE-102.md`, `specs/requirements.md` line 55 still linked the live path:

```
| STE-102 | Greeting helper | [link](frs/STE-102.md) |
```

The link 404s. Either /spec-archive should rewrite references to point at the archive path, or the convention should be "always link the archive path" (and /spec-write should pre-write the archive-path link, since archive is the canonical post-merge home).

This FR takes the first approach: /spec-archive rewrites references at archive time. Rationale: the active FR period is short (one /implement run); the archived state is the long-tail one. Rewriting at archive time means active-period readers get a direct link, not an "expected here once shipped" stub.

## Acceptance Criteria

- AC-STE-111.1: When /spec-archive moves an FR file from `specs/frs/<id>.md` to `specs/frs/archive/<id>.md`, it scans these files for references to the original path and rewrites them to the archive path:
  - `specs/requirements.md`
  - `specs/plan/<M>.md` (active milestone plan)
  - `specs/plan/archive/<M>.md` (archived milestone plans)
  - `CHANGELOG.md` (if the FR is referenced in a recent unreleased section — historical entries frozen)
- AC-STE-111.2: The rewrite is mechanical (string replace `frs/<id>.md` → `frs/archive/<id>.md`); no semantic parsing required. Both Markdown link forms (`](frs/<id>.md)` and `](./frs/<id>.md)`) and bare path mentions are covered.
- AC-STE-111.3: The rewrite is performed in the same commit as the `git mv`, not a follow-up commit. Atomic.
- AC-STE-111.4: New `/gate-check` probe `traceability-link-validity`: every link in `specs/requirements.md` matrix and `specs/plan/<M>.md` FR-list pointing at a `specs/frs/...` path MUST resolve to an existing file. Broken links (`/frs/<id>.md` when the file is in `frs/archive/`) → fail. Probe at `plugins/dev-process-toolkit/adapters/_shared/src/traceability_link_validity.ts` (flat path matches existing convention); integration test at `plugins/dev-process-toolkit/tests/gate-check-traceability-link-validity.test.ts` per STE-82.
- AC-STE-111.5: Edge case — if /spec-archive is called on an FR whose path appears nowhere in the cross-cutting docs (orphan FR, no traceability row), the skill proceeds without error and emits an info-level note in its report.
- AC-STE-111.6: Edge case — `CHANGELOG.md` rewrite is scoped to the unreleased section + the version-section currently being assembled by /ship-milestone, if applicable. Released sections (under `## [X.Y.Z]` headers with a date) are frozen and never touched. Detection: scan top-of-file until first dated `## [X.Y.Z]` header.
- AC-STE-111.7: Test fixture covers: FR with traceability row in requirements.md (rewritten), FR referenced in active plan (rewritten), FR referenced in archived plan file (rewritten), FR referenced in unreleased CHANGELOG (rewritten), FR referenced in released CHANGELOG (NOT rewritten), orphan FR (no rewrite, no error).
- AC-STE-111.8: `bun test` green. CHANGELOG `### Fixed` entry cross-references this FR + smoke-test origin.

## Technical Design

**Skill change.** `skills/spec-archive/SKILL.md` adds a "Rewrite traceability links" step between the `git mv` and the commit. New helper at `plugins/dev-process-toolkit/adapters/_shared/src/spec_archive/rewrite_links.ts`:

```typescript
export function rewriteArchiveLinks(repoRoot: string, frId: string): RewriteResult {
  const targets = [
    `${repoRoot}/specs/requirements.md`,
    ...glob(`${repoRoot}/specs/plan/*.md`),
    ...glob(`${repoRoot}/specs/plan/archive/*.md`),
    `${repoRoot}/CHANGELOG.md`, // scoped to unreleased per AC-STE-111.6
  ];
  const before = `frs/${frId}.md`;
  const after = `frs/archive/${frId}.md`;
  const filesChanged: string[] = [];
  for (const path of targets) {
    if (replaceInFile(path, before, after, /* changelog scoping */)) {
      filesChanged.push(path);
    }
  }
  return { filesChanged };
}
```

**CHANGELOG scoping.** `replaceInFile` for CHANGELOG.md takes an extra arg that limits replacement to lines before the first dated `## [X.Y.Z] — YYYY-MM-DD` header. Pure string operation; no Markdown parser needed.

**Probe.** `traceability_link_validity` walks the same file set as the helper and flags any `frs/<id>.md` reference that doesn't resolve. Skipped when `mode: none` and the project has no `requirements.md` matrix.

- Phase C FR — independent of Phase A/B; can land in parallel with STE-110.
- The "rewrite at archive time" choice over "always link archive" was made because /spec-write is called before the FR is implemented (active period); making /spec-write link the archive path means active-period readers click into a non-existent file. Archive-time rewrite gives both periods a working link.
