## Requirement

The toolkit's autonomous-mode contract has a hole on the refusal side. When a Claude Code session carries the harness-injected system-reminder *"The user has asked you to work without stopping for clarifying questions"*, structurally interactive gates and `requires-input:` annotated steps in toolkit skills get silently bypassed: the model picks a value and writes it to the audit trail with a guess-flavored reason, indistinguishable from a deliberate user choice. v2.13.0 / 2026-05-06 incident gist caught this against `/setup` step 7b â€” model picked `tracker_mode=none` with audit reason *"autonomous run; user described a single-maintainer plugin project"* despite the skill text literally saying *"Auto Mode does NOT relax `requires-input`"*.

STE-226 closed the **default-apply side** (byte-checkable marker `<dpt:auto-approve>v1</dpt:auto-approve>` â‡’ default-apply). This FR closes the **refusal side**: without the marker AND without a live human answer, every `requires-input:` step refuses with NFR-10 canonical shape; every audit row carries `imputed: true|false` so model-imputed values are structurally distinguishable from user-confirmed ones; a single `/gate-check` probe enforces the contract across all skills with `requires-input:` annotations.

## Technical Design

(Brainstorm-approved Approach 2; gist https://gist.github.com/nesquikm/d6d7c8303b7b923ab0dfc4aa902ec5ec)

`docs/auto-mode-protocol.md` is the single source of truth; every skill cites it instead of redefining the rule. The `requireOrRefuse(...)` helper consolidates the four-outcome decision so future skills reuse it without copy-paste. `/setup` step 7b is the first refactor and the regression-coverage anchor. The audit-row extension piggybacks on STE-108's existing `appendAuditRow` helper â€” adds one column to the format, one parameter to the helper, one parser tolerance for legacy rows. The new gate-check probe is modeled on STE-226's `auto_approve_marker_in_canonical_spawns` probe (same globbing, same coverage-then-citation shape). `/smoke-test` Phase 0 is updated in lockstep so its own refusal path exercises the protocol it embeds in child spawns.
