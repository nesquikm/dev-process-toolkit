# STE-128: Test-layout policy — pick tests/ mirror or src/ co-location and reconcile templates {#STE-128}

## Requirement

The toolkit must commit to ONE canonical test layout — either `tests/`-mirror or `src/`-co-located — and reconcile every spec template, gate probe, and CLAUDE.md DO NOT line to that choice. Today the gate is permissive (probe #20 accepts any `*.test.ts` outside `node_modules`) but the templates are strict (testing-spec § 2 prescribes `tests/` mirror; CLAUDE.md DO NOT requires "a real test file in tests/" before placeholder deletion). That gap forces every per-FR opt-in into the FR's Notes block (see smoke-test STE-124 for an example) and silently rots the docs.

Recommended choice: **`src/`-co-located** — matches the gate's existing permissive behavior, removes the `tests/.placeholder.test.ts` workaround entirely, and mirrors mainstream Bun/TS convention.

**Smoke-test evidence.** Run #4 (2026-04-27, v1.34.0). The shipped STE-124 placed its test at `src/greet.test.ts` (co-located): passed gate probe #20, but required FR Notes to authorize the layout deviation, and triggered drift in three places (testing-spec § 2, CLAUDE.md DO NOT line, M1.md task-list still references `tests/.placeholder.test.ts`). Full report: `/tmp/dpt-smoke-findings-2026-04-27.md` § F5 + Cross-spec consistency drift.

## Acceptance Criteria

- AC-STE-128.1: One layout chosen (recommend co-location) and documented as a decision record in `docs/patterns.md`.
- AC-STE-128.2: `templates/spec-templates/testing-spec.md.template` § 2 reflects the chosen layout. If co-location: prescribe `src/foo.test.ts` mirroring `src/foo.ts`. If `tests/` mirror: keep current template.
- AC-STE-128.3: `templates/CLAUDE.md.template` DO NOT block reflects the same choice. If co-location: drop the placeholder rule entirely (or replace with "do not delete `<file>` until `git ls-files '*.test.ts'` returns at least one match").
- AC-STE-128.4: `/setup` no longer scaffolds `tests/.placeholder.test.ts` if the chosen layout is co-location. M1.md template's foundation tasks are updated to match.
- AC-STE-128.5: Gate probe #20 is updated to enforce the chosen layout (currently permissive — both layouts pass; after this FR, the wrong layout fails the gate). For co-location: probe rejects `tests/*.test.ts` unless the FR file declares an opt-out. For mirror: probe rejects `src/**/*.test.ts`.
- AC-STE-128.6: Existing FRs/templates that mention the old layout are migrated or marked legacy.

## Technical Design

- Decision lives in `docs/patterns.md` § "Test layout policy" with a short rationale (gate already permissive → co-location is path of least resistance).
- Probe #20 update: read the `## Testing Conventions` block in CLAUDE.md to extract the layout, then enforce it. Falls back to permissive if the block is absent.
- Per the "no users yet" memory, no migration shim for downstream projects — break backwards compat freely.

## Testing

- Test file: `tests/templates/test_layout_policy.test.ts`.
- Cases: (a) co-location chosen + project has only `src/*.test.ts` → probe #20 passes; (b) co-location chosen + project has `tests/foo.test.ts` → probe #20 fails with NFR-10 naming the offending file; (c) mirror chosen + project has only `tests/*.test.ts` → passes; (d) absent `## Testing Conventions` block → probe stays permissive.

## Notes

- Severity: low (gate doesn't fail today). But it's the long tail of every future smoke-test surfacing the same drift triplet (testing-spec / CLAUDE.md / plan task-list), so worth closing.
- This FR alone doesn't fix STE-129 (requirements.md scope); they're independent template-policy FRs that pair naturally.
