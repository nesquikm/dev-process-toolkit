# STE-186: /smoke-test widen Phase 0.5 cleanup glob to MCP config files {#STE-186}

## Requirement

Phase 0.5 of `.claude/skills/smoke-test/SKILL.md` cleans stale per-run scratch via:

```
rm -f /tmp/dpt-smoke-prompt-*.txt /tmp/dpt-smoke-<tracker>-*.log
```

Neither glob matches `/tmp/dpt-smoke-mcp-config-<tracker>.json`, the wrapped MCP config the driver writes in Phase 1 step 5. The file persists across runs by design (it's regenerated each run), but the parent's `Write` tool refuses to overwrite a file it has not read in this session. Phase 1 step 5 had to fall back to a Bash heredoc to produce the wrapped MCP config — functionally identical, but the SKILL.md reference to "Write" implicitly assumes the file is fresh.

The 2026-05-01 Linear smoke run § F1 surfaced this. The fall-back path produced the correct config and no run was actually blocked, but the contract drift between SKILL.md prose ("write via Write") and operator behavior ("must use Bash heredoc") is the real finding.

This FR widens the Phase 0.5 cleanup glob to **also** delete `/tmp/dpt-smoke-mcp-config-*.json`. Result: every Phase 1 step 5 starts from a clean slate; either Write-tool or Bash-heredoc forms work; SKILL.md prose stays consistent.

Out of scope:

- The Phase 1 step 5 prose itself (Write vs. heredoc choice). Both work post-cleanup; SKILL.md keeps its current reference.
- Other `/tmp/dpt-smoke-*` artifacts (findings file, approval record). Those are deliberately preserved across runs (audit-trail invariant); the cleanup glob is intentionally narrow.
- The `/tmp/dpt-smoke-prompt-*.txt` cleanup line. Covered by STE-185 (which retires the prompt-file path entirely).

## Technical Design

### Files touched

| File | Change |
|------|--------|
| `.claude/skills/smoke-test/SKILL.md` | Phase 0.5 `rm -f` command extended with `/tmp/dpt-smoke-mcp-config-*.json`; surrounding prose adds the rationale + audit-trail invariant. ~3-line edit. |

### Glob choice

`/tmp/dpt-smoke-mcp-config-*.json` matches both `linear` and `jira` variants without per-tracker branching. The trailing `.json` extension prevents over-broad matches (e.g., a hypothetical `/tmp/dpt-smoke-mcp-config-backup` directory). Concurrent two-terminal tandem runs (M46) are not affected — Phase 0.5 fires before Phase 1 in each terminal independently, and both cleanups race against artifacts from prior runs (not the concurrent run's artifacts, which are written later in Phase 1).
