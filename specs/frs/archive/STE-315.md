# STE-315: New /gate-check probe — public-surface count drift {#STE-315}

## Requirement

Add a new `/gate-check` probe (#57) `public_surface_count_drift` that asserts the documented skill / agent / probe count tokens in `README.md` and `CLAUDE.md` match the actual on-disk counts. Backfill the seven stale count surfaces verified by the 2026-05-20 repo review so the new probe passes on `main` at FR ship time.

Closes the largest documented drift class in the toolkit: every recently-shipped FR that adds or removes a skill, agent, or probe rots a public-surface count line, and human review has missed seven such drifts at once. Verified disk state on `main` @ `2875661`: 23 skills (`ls -d plugins/dev-process-toolkit/skills/*/ | wc -l`), 8 agents (`ls plugins/dev-process-toolkit/agents/*.md | wc -l`), highest probe `56` (`grep -nE "^[0-9]+\\. " plugins/dev-process-toolkit/skills/gate-check/SKILL.md | tail -1`). README.md L3 still says "15 commands, 5 agents", L10 says "51 numbered" probes, etc. The new probe makes those rots loud at the next `/gate-check` instead of waiting for the next human review.

## Acceptance Criteria

- AC-STE-315.1: A new probe module `public_surface_count_drift` lives at `plugins/dev-process-toolkit/adapters/_shared/src/public_surface_count_drift.ts` (sibling-of-probes canonical path; matches the 55-of-56 existing layout per finding A25). Probe is registered in `skills/gate-check/SKILL.md` as the 57th numbered probe with `PROBE_ID = "public_surface_count_drift"`, severity `error`.
- AC-STE-315.2: At runtime the probe computes three observed values from disk — skills count = `Array.from(globIterate("plugins/dev-process-toolkit/skills/*/")).length` (exclude non-directory entries); agents count = `Array.from(globIterate("plugins/dev-process-toolkit/agents/*.md")).length`; max probe number = the largest decimal prefix matched by `^(\\d+)\\.\\s` in `skills/gate-check/SKILL.md` — and a set of documented values by parsing the count-bearing lines in `README.md` and `CLAUDE.md` (the exact line-numbers and patterns the test fixtures pin down per AC-STE-315.4). - AC-STE-315.3: Each violation emits NFR-10 canonical refusal shape (`<file>:<line>:<column> — <reason>` with `Refusing: / Remedy: / Context:` sub-lines). Refusing names the documented token (e.g. `"15 commands, 5 agents"`); Remedy names the on-disk truth and instructs the operator to update the line; Context names the source-of-truth path that the probe ran on (`ls -d plugins/dev-process-toolkit/skills/*/`). ## Technical Design

Probe module exports `runPublicSurfaceCountDriftProbe(projectRoot: string): ProbeResult` matching the canonical signature of probes #55–#56. Reads exactly three files at probe time: `README.md`, `CLAUDE.md`, `skills/gate-check/SKILL.md`. Uses `Bun.glob` (or `fs.globSync` for portability) to enumerate skills + agents. Counts are reported as `{ documented: number, observed: number, lineNo: number, sourcePath: string }` per token.

Pattern detection uses anchored regexes keyed off the verified-on-2026-05-20 surface (e.g. `/^(\\d+) commands?, (\\d+) agents?$/m` against README L3, `/^- \\*\\*Deterministic quality gates\\*\\* — (\\d+) numbered/m` against L10). The probe is **byte-strict** at the canonical surfaces — adding or removing a count-bearing line in `README.md` or `CLAUDE.md` requires updating the probe's pattern list in the same commit. Pattern list lives as a const at the top of the probe module so contributors editing public surfaces see the contract.

Probe runs after the existing 56 probes; numbered #57. Adds itself to the gate-check probe ordering at the bottom (matches recent precedent — #55, #56 added under STE-313). Counts both shipped under M84 (probe #57) and the L214 forward-message ("probe 58+ must ship the matching test file") so contributors who add probe #58 next see the right reminder.

Spec-research seed (STE-230 AC-STE-230.6): `spec_research_invoked` — 3 precedents returned (STE-88 count-only convention, STE-138 release-checklist-consistency probe pattern, STE-91 README-walkthrough-vs-count-line invariant). STE-138's "enumerate canonical paths/sections and assert each resolves on disk" pattern is the closest existing fit; this probe extends the same shape to count tokens.

Deps-research vacuous (`specs/deps.yaml` absent — zero entries → fork skipped per /spec-write § 0b step 2.5b).

## Testing

`tests/gate-check-public-surface-count-drift.test.ts` exercises the probe in isolation against synthetic fixture trees (in-memory `tmp` dirs). Four cases per AC-STE-315.5; each asserts the canonical NFR-10 message shape via `expect(err.message).toMatch(/^README\\.md:3:\\d+ — .+\\nRefusing: .+\\nRemedy: .+\\nContext: .+$/)`.

End-to-end verification: post-backfill, `/gate-check` PASSes on `main` with probe #57 enabled; an injected single-token mutation in any of the six monitored files fails the gate with the exact violation message.

No `claude -p` regression coverage required — this probe is read-only and deterministic, not gate-of-an-LLM-decision.
