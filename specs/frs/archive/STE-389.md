# STE-389: Derived-only PR titles — remove /pr's [PR title] argument {#STE-389}

## Summary

/pr advertises a PR-title positional argument whose only contract is a buried Notes
bullet — an operator supplied a title and the skill ignored it. Rather than harden a
surface nobody needs, this FR removes it: PR titles become derived-only, always
mirroring the dominant commit's Conventional Commits subject. Free text after /pr is
acknowledged with an explicit redirect — never silently ignored. Approved at the
2026-07-16 brainstorm; silent removal and a deprecation window were rejected.

## Requirement

Remove the argument surface from `skills/pr/SKILL.md` (frontmatter `argument-hint` +
the `$ARGUMENTS` Notes bullet), harden the Step-5 Title bullet to derived-only, and
add an explicit redirect rule for free text after /pr. Update README's Commands-table
row and Args-column note, and amend the cross-cutting `specs/testing-spec.md` Tier-1
frontmatter row so /pr is not a standing violator. Pin everything with one new
doc-conformance meta-test.

## Acceptance Criteria

- AC-STE-389.1: **Hint removed.** `skills/pr/SKILL.md` frontmatter has no `argument-hint:` key; meta-test asserts absence. {#AC-STE-389.1}
- AC-STE-389.2: **`$ARGUMENTS` removed.** No `$ARGUMENTS` literal anywhere in `skills/pr/SKILL.md` (the "use it" Notes bullet is deleted); meta-test asserts absence. {#AC-STE-389.2}
- AC-STE-389.3: **Derived-only prose.** Step 5's Title bullet states the PR title is always derived from the dominant commit's Conventional Commits subject, with no user-supplied override path; meta-test pins the phrase. {#AC-STE-389.3}
- AC-STE-389.4: **Explicit redirect.** A Notes rule states: when the invocation carries free text after /pr, the skill explicitly replies "PR titles are derived from the commit subject; amend the commit to change the title" and proceeds; meta-test pins the line. The new prose adds zero `STE-<N>` tokens (skills token ceiling is at 245/245). {#AC-STE-389.4}
- AC-STE-389.5: **README.** The /pr row's Args cell renders `—`; the `[PR title]` literal is absent from README.md; the Args-column note gains "— marks a skill that takes no arguments"; meta-test pins cell + absence. {#AC-STE-389.5}
- AC-STE-389.6: **Cross-cutting row amended.** `specs/testing-spec.md` Tier-1 frontmatter row reads "argument-hint where the skill takes arguments" (or equivalent) so a hint-less /pr conforms. {#AC-STE-389.6}
- AC-STE-389.7: **Full gate green.** `bun test` from `plugins/dev-process-toolkit` GREEN; SKILL line caps untouched (~62–64 ≪ 351); the byte-pinned Ship-State Pre-Flight section unmodified. {#AC-STE-389.7}

## Technical Design

- Docs/prose-only change — no runtime TypeScript surface (/pr has no TS helpers for
  titles).
- One new doc-conformance meta-test (STE-370 AC-STE-370.5 pattern) carries all pins:
  frontmatter-absence, `$ARGUMENTS`-literal absence, derived-only phrase, redirect
  line, README cell + literal absence.
- Constraints from the pre-spec verification sweep: new prose must not cite STE
  tickets (skills token ceiling at exact capacity, 245/245) and must not paraphrase
  ticket-binding-resolver or /tdd stage vocabulary (the cross-skill contract-drift
  gate scans skills/pr/SKILL.md and README.md); the Ship-State Pre-Flight section is
  byte-pinned by `m99-ste-370-post-merge-ceremony.test.ts` and stays untouched.
- All existing callers invoke /pr bare (ship-milestone chain § 9/§ 10, setup workflow
  listing, simplify advisory, plan_ship_coherence ceremony recipe, templates) — zero
  caller updates.

## Testing

- Meta-test matrix per the ACs: argument-hint absence, `$ARGUMENTS` literal absence,
  derived-only phrase pin, redirect-line pin, README cell + `[PR title]`-absence pin.
- Deterministic, no network; full gate from `plugins/dev-process-toolkit`.

## Notes

- Origin incident (2026-07-16): operator-supplied title ignored — the "use it" rule
  was a buried Notes bullet the executing model missed.
- The redirect is prose-level (runtime LLM behavior); the meta-test pins prose
  presence, not runtime behavior — asymmetric-stakes tradeoff accepted at brainstorm.
- Forward-only; no deprecation window (the surface never worked reliably), though the
  first external user now exists (issue #39).
