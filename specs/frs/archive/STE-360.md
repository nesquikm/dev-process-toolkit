## Requirement

The iter-2 healthy `/setup` grandchild run surfaced finding F4 (medium): the scaffolded `pre-commit-tdd-orchestrator` hook classifies `src/.placeholder.test.ts` (the STE-113 Bun zero-match workaround — /setup's own canonical Bun deliverable) as tdd-required and blocks the bootstrap commit, leaving the placeholder staged with a manual-commit instruction. /setup's output cannot ride its own bootstrap commit while its own hook is active.

## Acceptance Criteria

- AC-STE-360.1: `templates/hooks/_lib/hooks/pre-commit-tdd-orchestrator.ts` exempts staged `src/.placeholder.test.ts` carrying the `Bun zero-match workaround` marker comment (marker grep primary, basename guard secondary); unit tests extend the existing suite: a placeholder-only commit passes without /tdd evidence, placeholder + real FR file still requires evidence, and placeholder DELETION commits pass (the STE-215/STE-222 lifecycle).
- AC-STE-360.2: /setup's bootstrap-commit prose documents the exemption (the placeholder rides the bootstrap commit again); a doc-conformance test pins it.

## Technical Design

**Hook exemption (`templates/hooks/_lib/hooks/pre-commit-tdd-orchestrator.ts`).** In the staged-file classification (fed by `git diff --cached --name-only`), a staged path is exempt when (a) its basename is `.placeholder.test.ts` AND (b) either the staged content carries the `Bun zero-match workaround` marker comment (grep on the staged blob for additions) or the path is staged as a deletion (content unavailable — the STE-215/STE-222 first-real-test-lands lifecycle). The dual key prevents gaming: a real test renamed to `.placeholder.test.ts` without the marker still requires /tdd evidence. Exempt-only-file commits pass without `Skill(/tdd)` evidence; mixed commits (placeholder + any tdd-required file) still require it — the exemption subtracts the placeholder from the required set, never waives the check for other files.

**/setup prose (`skills/setup/SKILL.md` bootstrap-commit step).** Document that the placeholder is hook-exempt via its marker comment and rides the bootstrap commit; remove/adjust any prose implying the operator must commit it manually.

## Testing

- Unit (extend `tests/hook-modules-pre-commit-tdd-orchestrator.test.ts`): placeholder-only staged (with marker) ⇒ pass without evidence; placeholder without marker ⇒ still tdd-required; placeholder + real FR file ⇒ evidence required; placeholder deletion ⇒ pass (AC-STE-360.1). Use the existing in-memory `mkdtempSync` fixture pattern (STE-113 AC.5 precedent, no on-disk fixtures).
- Doc-conformance: /setup bootstrap-commit prose documents the exemption (AC-STE-360.2).
