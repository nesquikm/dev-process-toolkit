## Requirement

Two files in `plugins/dev-process-toolkit/docs/` encode shipped-milestone history in their filenames:

- `v2-layout-reference.md` — named after a v1→v2 layout migration that has completed.
- `fr-39-sync.md` — named after FR-39, but the body now opens "STE-17 — Bidirectional AC Sync" (already drifted).

Both files mislead a reader who never lived through that history. Rename to role-based names and rewrite all inbound references on active surfaces. Body prose cleanup is split out to sibling FR STE-116 so the verification gates stay honest (this FR has a strict grep gate; the editorial FR is reviewer-gated).

## Acceptance Criteria

- AC-STE-115.1: `git mv plugins/dev-process-toolkit/docs/v2-layout-reference.md plugins/dev-process-toolkit/docs/layout-reference.md`. History preserved per file. {#AC-STE-115.1}
- AC-STE-115.2: `git mv plugins/dev-process-toolkit/docs/fr-39-sync.md plugins/dev-process-toolkit/docs/ac-sync.md`. History preserved per file. {#AC-STE-115.2}
- AC-STE-115.3: Every inbound reference to either old filename on the in-scope surfaces is rewritten to the new name. In-scope surfaces: `plugins/dev-process-toolkit/docs/*.md`, `plugins/dev-process-toolkit/skills/**/SKILL.md`, `plugins/dev-process-toolkit/agents/**/*.md`, root `README.md`, root `specs/{requirements,technical-spec,testing-spec}.md`. Out-of-scope (left stale by design): `**/archive/**`, `CHANGELOG.md`, `specs/plan/archive/`, `specs/frs/archive/`. {#AC-STE-115.3}
- AC-STE-115.4: Forward verification gate. `rg -n 'v2-layout-reference|fr-39-sync' plugins/dev-process-toolkit/ --glob '!**/archive/**' --glob '!CHANGELOG.md'` returns **zero hits**. Same grep across `README.md` and root `specs/*.md` (excluding `specs/**/archive/`) returns zero hits. {#AC-STE-115.4}
- AC-STE-115.5: `bun test` green. `/gate-check` green. {#AC-STE-115.5}
- AC-STE-115.6: **Double-check references pass.** After AC-STE-115.4 passes, run an inverse-grep + link-resolution check to confirm rewrites land on real files: (a) `rg -n 'layout-reference\.md|ac-sync\.md' plugins/dev-process-toolkit/ specs/ README.md` returns hits and every cited path resolves to an existing file (no broken paths from typos in the rewrite); (b) for each in-scope `.md` file touched by AC-STE-115.3, every relative markdown link of the shape `[...](...md)` or `[...](docs/....md)` resolves to an existing file under the repo root. Any unresolved link is a hard fail — fix before declaring the FR done. {#AC-STE-115.6}

## Technical Design

Mechanical refactor. Two `git mv` operations + an Edit pass across the in-scope surfaces. No code or schema changes.

**Discovery commands (Phase A step 3):**

```bash
rg -l 'v2-layout-reference' plugins/dev-process-toolkit/ specs/ README.md
rg -l 'fr-39-sync' plugins/dev-process-toolkit/ specs/ README.md
```

**Double-check commands (AC-STE-115.6):**

```bash
# (a) inverse grep against new names
rg -n 'layout-reference\.md|ac-sync\.md' plugins/dev-process-toolkit/ specs/ README.md
# (b) extract relative .md links from touched files; verify each resolves
#     (use a small ad-hoc script or rg + awk; no new tooling required)
```

**Excluded from rewrite (deliberate):**

- `specs/frs/archive/**` and `specs/plan/archive/**` — frozen per AC-STE-18.4.
- `CHANGELOG.md` — historical record describing past releases.

## Testing

No new automated tests. Verification is the AC-STE-115.4 forward grep gate, the AC-STE-115.6 double-check pass, and `/gate-check`.

## Notes

Sibling FR STE-116 covers body prose cleanup. The split is intentional: this FR's gate is mechanical (grep returns zero, links resolve), the sibling's is editorial (reviewer judgment).
