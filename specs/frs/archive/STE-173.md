# STE-173: /setup SKILL.md commit-msg hook prose — best-effort install + manual fallback {#STE-173}

## Requirement

Smoke #7 (Linear, 2026-04-30, v2.1.0) F6 noted that `/setup` step 6b silently degrades when the model-layer block fires on `.git/hooks/commit-msg` writes under `bypassPermissions`. The audit log captures the skip in `## /setup audit` and the run continues cleanly (the hook is a per-clone artifact, not part of the bootstrap set). But the SKILL prose never tells the operator the install is "best-effort under non-interactive runs" or names the manual install command.

Result: a downstream user running `/setup` non-interactively sees `setup: skipping commit-msg hook (model-layer block)` in the audit log and has no idea this is expected, no idea the contract is "best-effort", and no idea what to do about it.

This is a docs-only change to step 6b prose.

Out of scope:
- Removing the model-layer block — that's a harness-level decision, not the plugin's concern.
- Auto-retrying the install with elevated permissions — same reason.
- Changing the audit-log prose — already correct.

## Acceptance Criteria

- AC-STE-173.1: `plugins/dev-process-toolkit/skills/setup/SKILL.md` step 6b adds one prose sentence stating: "Hook install is best-effort under non-interactive runs — model-layer blocks on `.git/hooks/` writes are expected under `bypassPermissions` and are logged in `## /setup audit` rather than failing the run."
- AC-STE-173.2: The same step adds a one-liner manual fallback the user can copy-paste post-`/setup`: `cp plugins/dev-process-toolkit/templates/git-hooks/commit-msg.sh .git/hooks/commit-msg && chmod +x .git/hooks/commit-msg`.
- AC-STE-173.3: `grep -nE 'best-effort|chmod \+x .git/hooks/commit-msg' plugins/dev-process-toolkit/skills/setup/SKILL.md` returns at least 2 matches.
- AC-STE-173.4: `/setup` audit prose elsewhere (the existing `## /setup audit` block in CLAUDE.md template) is unchanged — only the SKILL.md operator-facing prose is touched.
- AC-STE-173.5: `bun test` green; `/gate-check` clean; `/spec-review M49` clean.

## Technical Design

Single-edit pass in `plugins/dev-process-toolkit/skills/setup/SKILL.md` step 6b:

Locate the existing prose:

> `default: shell` — install the POSIX-shell Conventional-Commits hook (zero-dep). `$ARGUMENTS` contains `--commitlint` ⇒ install the commitlint-delegating variant. Skip entirely when `.git/hooks/` is absent (log `setup: skipping commit-msg hook (not a git repo)`).

Append two lines:

- The "best-effort" disclaimer sentence.
- The manual fallback `cp` + `chmod` one-liner, in a code fence.

No code, test, or template change beyond the SKILL prose.
