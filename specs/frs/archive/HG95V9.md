## Requirement

**Description:** When `/implement` completes a milestone and the human approves the Phase 4 report, the milestone block and all ACs traced to that milestone are moved from live spec files into a single archive file under `specs/archive/`, replaced by one-line pointers. Keeps `plan.md` and `requirements.md` size bounded regardless of project age.

## Acceptance Criteria

- AC-16.1: `implement/SKILL.md` Phase 4 contains a subsection titled `### Milestone Archival` placed after `### Spec Deviation Summary` and before the final report rendering
- AC-16.2: The subsection specifies archival runs only after the human approves the Phase 4 report, never before
- AC-16.3: The archive target path format is `specs/archive/M{N}-{slug}.md` where `{slug}` is a lowercased, hyphen-separated milestone title
- AC-16.4: The subsection instructs the agent to **move** (not copy) the `## M{N}: ...` block out of `plan.md` into the archive file
- AC-16.5: The subsection instructs the agent to consult the `requirements.md` traceability matrix and move every AC whose row resolved in this milestone into the same archive file
- AC-16.6: FRs whose ACs are *all* archived collapse in `requirements.md` to a single-line pointer using Schema H; FRs with mixed status retain only their non-archived ACs
- AC-16.7: The subsection contains the literal instruction: `Archival is skipped if specs/archive/ does not exist`
- AC-16.8: The subsection contains the literal instruction: `technical-spec.md is never auto-archived`
- AC-16.9: The agent leaves a pointer line (Schema H) in `plan.md` at the exact location the milestone block was removed
- AC-16.10: The agent appends one row to `specs/archive/index.md` per archival (per FR-19)
