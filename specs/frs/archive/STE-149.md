# STE-149: /setup — relocate tracker-id placeholder row {#STE-149}

## Requirement

The /setup-generated `specs/requirements.md` includes a literal `AC-<tracker-id>.1 |  | |` example row inside the rendered Traceability Matrix (line 58 in v1.39.0 output). It is intentional template scaffolding, but it sits in the rendered (non-comment) table where a careful reader is briefly puzzled by the unresolved literal, and a future `requirements-md-no-placeholder` gate-check rule could flag it as drift. Move the example into the HTML comment block at the top of the file (where the convention is already documented), leaving the rendered table empty (header row only).

## Acceptance Criteria

- AC-STE-149.1: After /setup runs on a fresh project, `specs/requirements.md` contains zero `AC-<tracker-id>\.\d+` literal occurrences outside HTML comment blocks (`<!-- ... -->`).
- AC-STE-149.2: The convention documentation explaining the `AC-<tracker-id>.<N>` placeholder pattern is preserved, just relocated inside the HTML comment.
- AC-STE-149.3: The Traceability Matrix at /setup time has only the table header row — no example data rows.
- AC-STE-149.4: /setup re-run on a project where requirements.md is already canonical is a no-op (idempotent — does not re-introduce the example row, does not duplicate the comment).

## Technical Design

Touchpoint: `templates/spec-templates/requirements.md.template` — the file that /setup copies into `specs/requirements.md`. Edit the template directly:

1. Remove the `| AC-<tracker-id>.1 |  |  |` row from the rendered Traceability Matrix.
2. Fold the example into the existing HTML-comment header at the top of the file (alongside the lines that already document the placeholder convention).

No SKILL.md change required — /setup already copies the template verbatim, so editing the template is sufficient. Idempotency is preserved because /setup's idempotent-merge branch only writes the file when it does not already exist; an existing canonical requirements.md is left alone.

## Testing

- Snapshot: post-/setup `specs/requirements.md` matches a fixture with no rendered-table example row, and the new HTML comment lines are present.
- Idempotency: /setup re-run on the canonical file diffs to zero (no re-introduction of the example row, no comment duplication).

## Notes

Surfaced by /smoke-test 2026-04-28 finding F3 (low severity, cosmetic). Only matters if a future gate-check rule is added that flags `<tracker-id>` literals in rendered prose; until then it is a paper cut for first-time readers.
