# STE-169: /implement chain-failure error message — drop toolkit-internal path {#STE-169}

## Requirement

`/implement` `SKILL.md:265` (in the chain-failure error path that fires when a downstream skill is unavailable) instructs the operator to "check `plugins/dev-process-toolkit/.claude-plugin/plugin.json`". That path only exists inside the toolkit repo. End-users in their own project will look for a `plugins/` directory at their project root, not find one, and waste time before realising the message is toolkit-internal jargon.

Replace the message with a generic phrasing that doesn't assume host-project layout — e.g., "verify the skill is installed (it may be disabled or missing from the marketplace; check `claude /plugin list`)".

Same pass should spot-check `skills/implement/SKILL.md` for any other hard-coded `plugins/dev-process-toolkit/...` paths and remove them.

Out of scope:
- Other toolkit-internal references elsewhere in the plugin (audit cleared the other 12 skills; STE-168 covers `/gate-check`).
- Restructuring the chain-failure flow itself — only the error message text changes.

## Acceptance Criteria

- AC-STE-169.1: `plugins/dev-process-toolkit/skills/implement/SKILL.md` line 265 area (the chain-failure error message) is rewritten to a generic phrasing that does not reference `plugins/dev-process-toolkit/.claude-plugin/plugin.json` or any toolkit-internal path. The replacement names the failure (skill unavailable) and the user-facing remedy (`claude /plugin list` or equivalent) — no toolkit-internal layout assumption.
- AC-STE-169.2: Spot-check pass: `grep -nE 'plugins/dev-process-toolkit' plugins/dev-process-toolkit/skills/implement/SKILL.md` returns zero matches in the chain-failure / error-handling sections (legitimate references in skill-internal cross-links — e.g., "see `plugins/dev-process-toolkit/docs/X.md`" — are reviewed individually and rewritten if they fire on the operator-facing path).
- AC-STE-169.3: `bun test` green; `/gate-check` clean; `/spec-review M48` reports no deviations.

## Technical Design

Single-edit pass in `plugins/dev-process-toolkit/skills/implement/SKILL.md`:

- Read line 265 and surrounding context (chain-failure error message).
- Rewrite the path-bearing sentence. Suggested replacement:

  > "If a downstream skill is unavailable, verify the plugin is installed and the skill is enabled — `claude /plugin list` lists installed plugins and their skills."

- Run `grep -n 'plugins/dev-process-toolkit' plugins/dev-process-toolkit/skills/implement/SKILL.md` and review each match. Operator-facing error messages get rewritten; documentation-style cross-references (skill-internal) are evaluated individually.

No code changes. No test changes (the SKILL prose is what the LLM reads at runtime).
