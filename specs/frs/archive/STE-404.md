# STE-404: Close the first-turn tracker-create gap: fence /spec-write against the autonomous-reminder gate-bypass {#STE-404}

## Summary

The first-turn contract forbids the scaffolding tools — Write, Edit, NotebookEdit — before a skill has either asked its first question or refused. Creating a tracker ticket is just as consequential as scaffolding a file, but the arbiter's forbidden set never listed the tracker-create tools, so a skill that jumps straight to creating an issue slips through un-flagged.

The 2026-07-20 conformance re-run hit exactly that hole: under the harness autonomous-mode reminder and no marker, spec-write skipped its first-turn loop and its gates and made a real tracker write before asking anything. This FR adds the tracker-create tools to the forbidden-before-first-ask set in the arbiter, names them and the reminder-escalation explicitly in the skill contract, and pins the clause with a gate probe — closing the gap the arbiter and Phase 8 now detect deterministically.

## Requirement

The re-run's F4 (HIGH): under the autonomous-mode reminder + no auto-approve marker, `/spec-write` skipped the Socratic first-turn loop AND its draft/commit gates and issued a real `createJiraIssue` (DST-49) + FR scaffold — the magpie shape STE-237/294 fence. The escalation trigger is the reminder: the same spawn without it soft-halts and writes nothing (F2, medium).

The existing defenses are SKILL-prose + structural probes (they verify the SKILL text, which the runtime LLM ignored) plus two deterministic session-log/commit probes (#45 checks commits, #56 checks gate tokens). F4 slipped past all of them because it made a tracker WRITE (not a commit) and skipped the gates ENTIRELY (no token for #56 to arbitrate). The one detector that DID catch it — the `assertFirstTurnShape` arbiter (Phase 8) — caught the FR-scaffold Write, not the tracker create, because tracker-create MCP tools are absent from `SCAFFOLDING_TOOLS`.

The plugin cannot code-prevent an LLM with direct MCP access from calling a create tool. This FR closes the contract gap and strengthens deterministic detection: a tracker create before the first ask/refusal is now a first-turn violation, matching the "deterministic gates over LLM judgment" principle. Prevention remains bounded by the LLM's direct MCP access; detection is made precise and the contract explicit.

## Technical Design

**`adapters/_shared/src/socratic_first_turn.ts`.** Add an exported `TRACKER_CREATE_TOOLS = ["mcp__atlassian__createJiraIssue", "mcp__linear__save_issue"] as const`. In `assertFirstTurnShape`, extend the tool_use branch: a tool_use whose `name` is in `TRACKER_CREATE_TOOLS` before the first ask/refusal throws `SocraticFirstTurnViolationError` (same error, same NFR-10 message shape) — unconditionally (no `projectRoot` path check; creates have no path). Scaffolding (`SCAFFOLDING_TOOLS`) keeps its STE-399 path-scoping unchanged. The check ordering: refusal/ask short-circuit first (a create AFTER the ask is fine), then scaffolding (path-scoped), then tracker-create (name-only).

`mcp__linear__save_issue` covers both create and update, but before the first ask/refusal in `/spec-write` it can only be a create (there is no prior FR to update in the first turn) — so treating any first-turn `save_issue` as forbidden is correct and conservative. Document this in the code comment.

**`skills/spec-write/SKILL.md`.** Extend the § FIRST ACTION contract (line ~10) and the § Rules Socratic Loop Contract: the forbidden-before-first-ask set now includes tracker-create tools (`mcp__atlassian__createJiraIssue` / `mcp__linear__save_issue`), and an explicit sentence names the autonomous-mode reminder as the escalation trigger to resist (F4). Fold F2: at the § 0b draft-gate ABSENT + non-tty branch, reaffirm the ONLY compliant outcome is `RequiresInputRefusedError` — a soft-halt on the interactive prompt is non-compliant.

**Probe — `adapters/_shared/src/spec_write_first_turn_tracker_create_clause.ts` + register in `skills/gate-check/SKILL.md`.** Single-file scope (`/spec-write` SKILL), literal substring match, severity error, mirroring probe #70/#71 shape. Required literals: `mcp__atlassian__createJiraIssue`, `mcp__linear__save_issue`, and a first-turn anchor phrase. Bump probe-count pins 71 → 72 at the enumerated surfaces (README ×2, calibration tests, m109 cross-check).
