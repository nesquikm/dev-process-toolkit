## Requirement

The v1.29.0 smoke test generated `specs/frs/STE-102.md` with frontmatter:

```yaml
---
id: 01KQ1W6W6ND4CFJ2MWQSFF8G1Z
title: Greeting helper
milestone: M1
status: archived
archived_at: 2026-04-25T16:23:26Z
tracker:
  key: linear
  id: STE-102
  url: https://linear.app/stellarlab-mikes-sandbox/issue/STE-102/greeting-helper
created_at: 2026-04-25
---
```

Two issues with this:

1. **`id:` ULID is present in tracker mode.** AC-STE-86.5 (M21 "Bimodal") says `id:` is mode-conditional: required in mode-none, absent in tracker-mode. `/gate-check` probe-13 (`identity_mode_conditional`) already detects this as a warning. /spec-write itself is the source of the violation — its FR template emits `id:` unconditionally.
2. **`tracker:` block uses the verbose `{ key, id, url }` shape.** The toolkit's own M27 FRs (STE-103/104/105) use the compact `tracker: { linear: STE-103 }` shape. Two shapes in the wild = drift; downstream consumers / probes have to handle both.

This FR fixes the source of both shapes by branching the FR template on tracker mode.

## Acceptance Criteria

- AC-STE-110.1: `/spec-write` FR-template generation branches on tracker mode (read from CLAUDE.md `## Task Tracking`):
  - `mode: none` → emit `id: <ULID>` and no `tracker:` block.
  - `mode: linear|jira|custom` → omit `id:` entirely; emit `tracker:` block in the canonical compact shape `{ <mode>: <tracker-id> }`.
- AC-STE-110.2: The compact tracker-block shape is the only shape /spec-write emits in tracker mode. Specifically, the verbose `{ key, id, url }` shape from the smoke-test output is removed from any template path. URL can be reconstructed from `<tracker-id>` and the workspace slug if needed at read time.
- AC-STE-110.3: Existing tracker-mode FR files with `id:` are NOT touched — migration is out of scope here (M21 already covered the historical migration; remaining drift is in the template, which this FR fixes for new files).
- AC-STE-110.4: `/gate-check` probe-13 (`identity_mode_conditional`) is upgraded from warning to error after this FR ships. Existing toolkit FRs that still carry `id:` are either (a) archived under M16 or earlier (frozen, not subject to current rules) or (b) flagged in the gate-check report as actionable cleanup, not silently passed.
- AC-STE-110.5: New mode-none FR generation continues to emit `id:` (no regression). Test fixture covers both branches.
- AC-STE-110.6: Both frontmatter shapes are covered by the existing `tests/gate-check-identity-mode-conditional.test.ts` (probe-13) which exercises six probe-identity fixtures spanning the bimodal contract: `mode-none-valid` (id present), `tracker-mode-valid` (id absent), `mode-none-missing-id` (warn), `mode-none-wrong-id` (warn), `tracker-mode-has-id` (warn), `tracker-mode-malformed-id` (warn). M29 STE-110 flips the severity from warn to error; no separate `spec-write-frontmatter` fixture directory is needed — the probe-identity fixtures already cover both branches programmatically.
- AC-STE-110.7: `bun test` green. CHANGELOG `### Fixed` entry cross-references this FR + probe-13's pre-existing detection.

## Technical Design

**Template change.** `skills/spec-write/SKILL.md` — find the FR-frontmatter generation block and split into two branches:

```yaml
# tracker mode
---
title: <title>
milestone: <M>
status: archived
archived_at: 2026-04-25T16:23:26Z
tracker:
  <mode>: <tracker-id>
created_at: <ISO>
---
```

```yaml
# mode: none
---
id: <ULID>
title: <title>
milestone: <M>
status: archived
archived_at: 2026-04-25T16:23:26Z
created_at: <ISO>
---
```

**Probe-13 upgrade.** `plugins/dev-process-toolkit/adapters/_shared/src/identity_mode_conditional.ts` — change severity from "warn" to "error" (flat path matches the existing M21 ship location; M29 implementation kept the file in place rather than moving it). Existing test `tests/gate-check-identity-mode-conditional.test.ts` extended to cover the error case.

**No migration.** Existing tracker-mode FRs that carry `id:` were generated by the pre-fix template; they're either archived (frozen) or active toolkit FRs that get fixed by hand during /implement. M21 already migrated the historical bulk — this FR closes the regeneration leak.
