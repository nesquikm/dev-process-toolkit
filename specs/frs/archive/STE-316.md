# STE-316: Rewrite tracker-mode docs for v2 file-per-FR layout {#STE-316}

## Requirement

Rewrite the five tracker-mode docs that still teach the obsolete v1 `### FR-N` layout (ACs in `specs/requirements.md` FR-block bullet lists) to teach the canonical v2 file-per-FR layout (one FR per `specs/frs/<id>.md`, ACs in the file's `## Acceptance Criteria` section). The v2 layout has been canonical since STE-26 (M13, archived); v1→v2 migration scaffolding was retired by STE-96 (M26, archived). The five docs in scope teach v1 prose unchanged from before that migration — a new operator following them lands FR files that the `requirements-md-no-placeholder` probe (#29) hard-rejects.

Drift surface verified against `main` @ `2875661` (v2.28.1) on 2026-05-20:

| Doc                                            | Lines             | Stale teaching                                                                                  |
| ---------------------------------------------- | ----------------- | ----------------------------------------------------------------------------------------------- |
| `docs/ac-sync.md`                              | 20-22, 71, 89-90, 118 | ACs parsed from `### FR-{N}` block in `specs/requirements.md`; local-side write rewrites `requirements.md`. |
| `docs/implement-tracker-mode.md`               | 49-50             | Reads local AC list from `requirements.md` FR-block bullet list.                                |
| `docs/spec-review-tracker-mode.md`             | 6-9, 15, 31-34, 37 | Reads ACs from `requirements.md` FR sections; traceability matrix maps `FR-{N}` → ticket IDs; references retired DD-12.2 decision doc. |

## Acceptance Criteria

- AC-STE-316.1: After this FR ships, `git grep -nE "(\\\\\\\\#FR-[0-9]+|### FR-[0-9]+|FR-\\\\{N\\\\}|FR-\\\\{[0-9]+\\\\})" plugins/dev-process-toolkit/docs/{ac-sync,implement-tracker-mode,spec-review-tracker-mode,spec-write-tracker-mode,layout-reference,setup-tracker-mode}.md` returns zero matches. The replacement teaches: per-FR file at `specs/frs/<tracker-id>.md` (tracker mode) or `specs/frs/<short-ULID>.md` (mode-none), with ACs in the file's `## Acceptance Criteria` section using AC-prefix derived via `acPrefix(spec)` (tracker-id in tracker mode, short-ULID tail in mode-none).
- AC-STE-316.2: `docs/layout-reference.md` L62 is rewritten so the `id:` field is documented as mode-conditional (required in `mode: none`, absent in tracker mode — refers operators to probe #13 `identity_mode_conditional` and `skills/gate-check/SKILL.md:26`). L69 is rewritten to drop the v1→v2 `requirements.md#FR-N` rewrite claim; the only rewrite the toolkit ships is `frs/<id>.md` → `frs/archive/<id>.md` (per `adapters/_shared/src/spec_archive/rewrite_links.ts`).
- AC-STE-316.3: All five DD-12.x decision-record references (`docs/setup-tracker-mode.md:20` DD-12.9; `docs/setup-tracker-mode.md:80` DD-12.8; `docs/spec-review-tracker-mode.md:37` DD-12.2) are removed or rewritten to refer to current FR provenance (the originating tracker IDs visible via CHANGELOG). The DD-12 scheme is fully retired from `specs/` (verified: `grep -c "DD-12" specs/{technical,requirements}*.md` = 0); the doc surfaces are the last bound.
- AC-STE-316.4: Bidirectional consistency — for any `<tracker-id>` substituted into one of the six docs, a `/gate-check` run with probe #29 (`requirements-md-no-placeholder`) on a freshly-scaffolded project following the docs' new instructions passes (no `### FR-N` blocks land in `specs/requirements.md`). Reviewer-prose check: the rewritten paragraphs read coherently end-to-end; cross-references between the six docs all resolve.
- AC-STE-316.5: Add a one-line cross-reference at the top of each rewritten doc citing `docs/layout-reference.md` as the canonical authority on FR file shape. Prevents future v1↔v2 split-brain by anchoring all five doc surfaces to a single source of truth that already exists.

## Technical Design

Pure doc edits — no code, no tests, no probe changes. The doc-level Pattern from spec-research seed (STE-115 / STE-116 split): mechanical grep gate paired with reviewer-prose gate.

Rewrite strategy per doc:

- **`docs/ac-sync.md`** (lines 20-22, 71, 89-90, 118) — replace every `specs/requirements.md` AC-reference with `specs/frs/<tracker-id>.md ## Acceptance Criteria`. The bidirectional sync target is now the per-FR file, not the central requirements.md. Local-side write becomes a targeted edit to `specs/frs/<id>.md` instead of "rewriting the FR block's AC list".
- **`docs/implement-tracker-mode.md`** (lines 49-50) — replace "Read the local FR's AC list from `specs/requirements.md` (parsed from the `### FR-{N}:` block's bullet list)" with "Read `specs/frs/<tracker-id>.md` ## Acceptance Criteria section".
- **`docs/spec-review-tracker-mode.md`** (lines 6-9, 15, 31-34) — replace "reads ACs from `specs/requirements.md` FR sections" with "reads ACs from each `specs/frs/<tracker-id>.md` file's `## Acceptance Criteria` section". Drop the "traceability matrix maps `FR-{N}` → ticket IDs" prose; the FR file's filename + frontmatter IS the traceability now.
- **`docs/spec-write-tracker-mode.md`** (lines 41-44, 63-72) — replace "After each FR-level AC save in `specs/requirements.md`" with "After each AC-list save in `specs/frs/<tracker-id>.md`". Drop the FR-N → ticket-id resolver matrix prose; the filename `<tracker-id>.md` already encodes the binding.
- **`docs/layout-reference.md`** (lines 62, 69) — per AC-STE-316.2.
- **`docs/setup-tracker-mode.md`** (lines 20, 80) — replace DD-12.9 / DD-12.8 references with `STE-<N>` citations or drop the citation entirely (the surrounding prose already states the requirement).
