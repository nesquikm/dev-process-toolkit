## Requirement

**Description:** Detect and flag tests that pass but assert nothing meaningful.

## Acceptance Criteria

- AC-8.1: `tdd/SKILL.md` RED phase contains a subsection titled `### Assertion Quality` with guidance that tests must assert on output/behavior
- AC-8.2: The Assertion Quality subsection lists at least 3 concrete anti-patterns: `expect(fn).not.toThrow()` alone, `assert result is not None` alone, type-only checks without value verification
- AC-8.3: `implement/SKILL.md` Phase 3 Stage A contains a check titled `Assertion quality check` referencing the same anti-patterns as AC-8.2
- AC-8.4: The anti-pattern list in both files is identical (same items, same order) to prevent drift between skills
