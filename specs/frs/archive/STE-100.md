## Requirement

Two user-facing docs reference `bun run adapters/<tracker>/src/<helper>.ts` without the `${CLAUDE_PLUGIN_ROOT}/` prefix:

- `docs/setup-tracker-mode.md:151-152` — Jira AC custom-field discovery step.
- `docs/tracker-adapters.md:14` — narrative template.

A user copy-pasting these from a consumer install runs them from project root, where `adapters/` doesn't exist (it lives at `${CLAUDE_PLUGIN_ROOT}/adapters/`).

`tests/skill-path-portability.test.ts` exists and scans `skills/**/SKILL.md` for the portable prefix but does NOT scan `docs/**/*.md`. The two hits above sit outside the test's scope.

## Acceptance Criteria

- AC-STE-100.1: `docs/setup-tracker-mode.md:151-152` `bun run` invocation is prefixed with `${CLAUDE_PLUGIN_ROOT}/`. {#AC-STE-100.1}
- AC-STE-100.2: `docs/tracker-adapters.md:14` `bun run` template is prefixed with `${CLAUDE_PLUGIN_ROOT}/`. {#AC-STE-100.2}
- AC-STE-100.3: `tests/skill-path-portability.test.ts` extended to scan `plugins/dev-process-toolkit/docs/**/*.md` for `bun run adapters/` patterns lacking the portable prefix. Test fails when a doc uses the bare path. {#AC-STE-100.3}
- AC-STE-100.4: The extended test passes after AC-STE-100.1 and AC-STE-100.2 land. {#AC-STE-100.4}
- AC-STE-100.5: No legitimate same-directory `bun run` (e.g., test scripts inside `adapters/_shared/`) is incorrectly flagged. The test scope is `docs/**`, not the entire repo. {#AC-STE-100.5}

## Technical Design

Two prose edits + one test extension:

1. `docs/setup-tracker-mode.md:151-152` — change `bun run adapters/jira/src/discover_field.ts` to `bun run ${CLAUDE_PLUGIN_ROOT}/adapters/jira/src/discover_field.ts`.
2. `docs/tracker-adapters.md:14` — change template `bun run adapters/<tracker>/src/<helper>.ts` to `bun run ${CLAUDE_PLUGIN_ROOT}/adapters/<tracker>/src/<helper>.ts`.
3. `tests/skill-path-portability.test.ts` — add a `describe("docs path portability")` block that walks `plugins/dev-process-toolkit/docs/**/*.md` and asserts no line matches `bun run adapters/` without the `${CLAUDE_PLUGIN_ROOT}/` prefix.

## Testing

The test extension itself IS the test. After it lands, `bun test` enforces the invariant going forward.
