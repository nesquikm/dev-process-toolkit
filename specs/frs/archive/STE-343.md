# STE-343: /gate-check probe — every referenced design-image path resolves on disk {#STE-343}

## Requirement

A `## Design References` entry that points at a moved, renamed, or never-committed image is a silent rot: the spec claims a design target that no longer exists, and nothing catches it until a human notices. The toolkit's first principle is "deterministic gates override LLM judgment" — and image-path existence is perfectly deterministic.

This FR adds a `/gate-check` probe that scans every spec file for `## Design References` entries and **GATE FAILED**s on any repo-root-relative path that does not resolve on disk. It is the deterministic guard that makes STE-341's immutable-reference promise enforceable rather than aspirational.

## Acceptance Criteria

- AC-STE-343.1: **Forward-resolution probe.** A new `/gate-check` probe scans `specs/requirements.md`, `specs/frs/**/*.md` (active **and** `archive/`), `specs/technical-spec.md`, `specs/testing-spec.md`, and `specs/plan/**/*.md` for `## Design References` entries, extracts each repo-root-relative path, and resolves it against the repo root. Any path that does not exist on disk → **GATE FAILED**, naming `file:line` + the missing path in NFR-10 canonical shape. {#AC-STE-343.1}
- AC-STE-343.2: **Vacuous when unused.** When no spec file contains a `## Design References` section (or all such sections are empty), the probe passes vacuously — zero findings, no error. This is the state of every project that hasn't adopted design images, including this repo at ship time. {#AC-STE-343.2}
- AC-STE-343.3: **Archive immutability proven by test.** Unit tests cover: (a) a present path → pass; (b) a missing path → GATE FAILED naming `file:line` + path; (c) no `## Design References` section → vacuous pass; (d) an **archived** FR (`specs/frs/archive/<id>.md`) whose `## Design References` entry resolves correctly → pass, demonstrating that the same repo-root-relative reference is valid after the FR moved into `archive/` (the immutability guarantee). {#AC-STE-343.3}

## Technical Design

**Files touched:**
- `plugins/dev-process-toolkit/adapters/_shared/src/scan_design_references.ts` — new pure helper: `scanDesignReferences(projectRoot) → { file, line, path, resolves }[]`. Walks the spec-file glob, parses `## Design References` sections (list entries of the documented shape — backtick-wrapped repo-root-relative path + optional caption), and `existsSync(join(projectRoot, path))` for each. Returns structured rows; the probe formats failures.
- `plugins/dev-process-toolkit/skills/gate-check/SKILL.md` — wire the helper into a new numbered probe (next available number), **severity: error / GATE FAILED**, with the vacuous-pass and archive-inclusive scan rules in prose.
- `plugins/dev-process-toolkit/tests/gate-check-design-references-resolve.test.ts` — the AC-STE-343.3 cases.

**Parsing contract:** an entry is a list item under a `## Design References` heading whose first backtick-wrapped token is a repo-root-relative path (matching the STE-341 documented shape). Non-path prose lines under the heading are ignored. The probe is path-existence only — it does not validate image format, dimensions, or that the file is actually an image (out of scope).

**Severity rationale:** GATE FAILED (not advisory). A dangling design reference is an objective, deterministic defect — exactly the class the toolkit's deterministic-gate principle says must hard-fail. Contrast with Deliverable B's *visual* conformance check (later milestone), which is LLM judgment and therefore advisory.

## Testing

- `tests/gate-check-design-references-resolve.test.ts`: present-path pass; missing-path fail (asserts `file:line` + path in the message); no-section vacuous pass; archived-FR-with-valid-ref pass. Fixtures use a temp spec tree (active + `archive/`) so the archive-immutability case is exercised against a real moved-file layout.
- Removing the `existsSync` check makes the missing-path case go green — a true guard, not a tautology.
- Gate: `bun test` (full gate from `plugins/dev-process-toolkit`, incl. `tests/` meta-tests) + `/dev-process-toolkit:gate-check` GREEN on this repo (vacuous — this repo ships zero design references).

## Notes

- **Depends on STE-341** for the `## Design References` section shape + repo-root-relative path convention this probe parses.
- **Forward-direction only.** The probe asserts referenced paths exist; it does **not** flag orphan images (files in `specs/design/` referenced by no spec). Orphan detection is deliberately out of scope — it is a hygiene nicety, not a correctness gate, and the ducks' blob-accumulation concern is better served by a future advisory if it proves needed.
- **Pairs with STE-342.** The capture step writes paths that this probe is guaranteed to resolve at write time; the probe catches subsequent drift (a manually moved/deleted image, a hand-edited reference).
