# STE-229: feat(skills/report-issue): bug-report skill bundling curated context + opt-in transcript into a secret gist {#STE-229}

## Requirement

A new `/dev-process-toolkit:report-issue` skill ships with the plugin so any toolkit user (or the maintainer self-debugging) can capture a structured incident report — dev narrative + curated repo state — and optionally the current Claude Code session's JSONL transcript, package the result into a **secret GitHub gist** via `gh gist create -s`, and print the URL for sharing with the maintainer or for downstream consumption by `/dev-process-toolkit:brainstorm <gist-url>`.

Default invocation is privacy-first (curated only, no transcript); `--full` and an in-flow `[y/N]` prompt offer the opt-in transcript path for high-signal cases. All payloads pass through a redaction pass that strips known secret patterns (Anthropic / OpenAI / GitHub / AWS keys, generic `api_key|token|password|secret` assignments) from settings + transcript before any upload. "Secret" gist semantics on GitHub are unlisted-but-URL-shareable, **not** truly private; the redaction pass is the privacy guarantee, not the gist's privacy flag.

After STE-229 ships:

- A toolkit user hitting an unexpected behaviour runs `/report-issue`, answers four Socratic prompts, sees a preview, approves, and gets a gist URL back.
- The maintainer pastes the URL into `/brainstorm <gist-url>` to triage with full context (curated state + optionally transcript).
- The same dev can also self-debug by feeding their own URL into `/brainstorm` in a fresh session — the round-trip is two operator gestures with no intermediate state.
- Plugin maintainers no longer rely on Slack screenshots and partial transcripts to diagnose issues.

The skill writes nothing under VCS — it is a pure outbound operation against `gh`. It is therefore exempt from STE-228's `commit_producing_skill_branch_gate` probe; the probe is updated to read an explicit non-commit-producing allowlist rather than parsing universally for `git commit` literal references.

## Acceptance Criteria

- AC-STE-229.1: Skill file at `plugins/dev-process-toolkit/skills/report-issue/SKILL.md` with frontmatter `name: report-issue`, `description: <one-liner>`, `argument-hint: '[--full]'`. Auto-discovered by Claude Code from `skills/` (no `plugin.json` change — the current convention has no `skills:` array; only `name`, `version`, `description`, `author`, `homepage`, `repository`, `license`, `keywords` are present).
- AC-STE-229.2: Hard prerequisite — the skill probes `gh auth status` before any data collection. On failure, emits an NFR-10-shape refusal naming the failed probe + remedy `gh auth login`, exits non-zero, and writes nothing (no temp directory created, no narrative collected).
- AC-STE-229.3: Default-mode (no `--full`) curated payload includes:
  - Plugin version (read from `plugins/dev-process-toolkit/.claude-plugin/plugin.json` `version` field).
  - `git rev-parse HEAD`.
  - `git status --porcelain`.
  - `git log -10 --oneline`.
  - Last `/gate-check` log if discoverable (probe `.claude/last-gate-check.log` and equivalents from the toolkit's gate-check skill output convention; skip if absent — do not fail the skill).
  - Redacted `.claude/settings.json` and `.claude/settings.local.json` (per AC-STE-229.6).
  - List of skills invoked in the current session — derived by reading the JSONL session file (per AC-STE-229.4) and extracting the `name` field of every `Skill` tool-use record.
  - The dev's narrative (per AC-STE-229.5).
- AC-STE-229.4: Full-mode is triggered by EITHER the `--full` CLI flag OR an in-flow `Include full transcript? [y/N]` prompt during preview (default `N` interactively; auto-`N` in auto-mode unless the marker per AC-STE-229.8 is present, in which case the marker drives whatever flag was passed). When triggered, the payload additionally bundles the current session's JSONL at `~/.claude/projects/<cwd-slug>/<session>.jsonl`. `<cwd-slug>` is derived from `pwd` by replacing every `/` with `-`. The current session is selected as the most-recent-mtime JSONL in the slug directory (rationale: skills cannot read their own session UUID from environment; mtime is the deterministic proxy and is documented as such inline in `find_current_session.ts` and SKILL.md).
- AC-STE-229.5: Narrative collection is Socratic — one prompt at a time (composes with `docs/patterns.md § Pattern 26: Socratic Prompting`). The four prompts in canonical order:
  1. "What happened? (one or two sentences describing the unexpected behaviour)"
  2. "What did you expect to happen instead?"
  3. "Severity? (low / medium / high)"
  4. "Reproducible? If so, list the steps. If not, type 'unsure'."
  Doc-conformance test asserts the four lines are present in this order in SKILL.md.
- AC-STE-229.6: Redaction — before any payload is uploaded, every text artifact runs through `scrubSecrets(text)` from a new module `plugins/dev-process-toolkit/adapters/_shared/src/scrub_secrets.ts`. The `SECRET_PATTERNS` constant exports these seven patterns:
  1. Anthropic API keys: `sk-ant-[A-Za-z0-9_-]{20,}`
  2. OpenAI API keys: `sk-[A-Za-z0-9]{20,}` (matches OpenAI shape; Anthropic shape per #1 takes precedence by virtue of being matched first via longer-prefix-first ordering)
  3. GitHub PATs: `gh[pous]_[A-Za-z0-9]{20,}` (covers `ghp_`, `gho_`, `ghu_`, `ghs_`)
  4. AWS access keys: `AKIA[0-9A-Z]{16}`
  5. Generic key=value: `(?i)(api[_-]?key|token|password|secret)\s*[:=]\s*['"]?[^\s'"]{8,}` — replaced as `<key>=***REDACTED***` to preserve key visibility for triage context
  6. JWT tokens: `eyJ[A-Za-z0-9_-]+\.[A-Za-z0-9_-]+\.[A-Za-z0-9_-]+`
  7. AWS secret keys (heuristic): `(?i)aws[_-]?secret[_-]?access[_-]?key\s*[:=]\s*['"]?[A-Za-z0-9/+=]{40}`
  Substitution is the literal `***REDACTED***` (except for pattern #5 which preserves the key name). Returns `{ scrubbed: string; matches: { pattern: string; count: number }[] }`. Settings files (`.claude/settings.json` / `.claude/settings.local.json`) are scrubbed even if they parse as YAML/JSON — the regex passes treat them as plain text to avoid leaking via key-substring match.
- AC-STE-229.7: Multi-file gist payload — three files maximum:
  - `report.md` — narrative + curated context inline; markdown for human readability; sections clearly headed (`## Narrative`, `## Severity`, `## Reproducibility`, `## Curated Context`, `## Redaction Summary`).
  - `metadata.json` — machine-readable: `{ timestamp, plugin_version, git_head, severity, redaction_summary: { pattern, count }[], full_transcript_included: boolean }`.
  - `transcript.jsonl` — included only in full mode; the scrubbed transcript.
  Canonical create command: `gh gist create -s -d "<title>" report.md metadata.json [transcript.jsonl]`. `-s` is the secret/unlisted flag. Title format: `dev-process-toolkit issue: <severity> — <one-line narrative head, ≤72 chars>` (truncated at 72 with ellipsis if longer).
- AC-STE-229.8: Preview gate — before `gh gist create`, the skill prints a preview block listing each file with its byte size + the redaction summary, then prompts `Push to gist? [y / n / edit]`. **Default-apply `y` when the prompt body contains the literal line `<dpt:auto-approve>v1</dpt:auto-approve>` (byte-grep, no inference)** — emits a `report_issue_default_applied` capability row in the closing summary. Marker is the single deterministic mechanism per STE-226's contract (legacy `Auto Mode Active` system-reminder detection is **not** used). Interactive `n` decline emits `report_issue_declined`, deletes the temp working directory (`mktemp -d`-created), exits non-zero. `edit` re-opens narrative collection with the previous answers pre-filled (best effort; transcript and curated context are not editable — the preview is the dev's last chance to abort before upload).
- AC-STE-229.9: Closing summary on every successful run (≥100 bytes; per-skill console-status contract used by `/spec-write`, `/setup`, `/implement`, `/gate-check`, `/spec-review`, `/simplify`). Contents:
  - Rendered gist URL.
  - File list with byte sizes.
  - Redaction-match counts grouped by pattern key.
  - Severity (echoed from narrative).
  - A `Next:` line offering both paths verbatim:
    - `Share this URL with the plugin maintainer for triage.`
    - `Or run /dev-process-toolkit:brainstorm <gist-url> to self-debug from the captured context.`
- AC-STE-229.10: Branch-gate exemption — the skill writes no files outside `mktemp -d` (which it deletes on every exit path) and never invokes `git commit`. SKILL.md documents this exemption explicitly. STE-228's `commit_producing_skill_branch_gate` probe is updated to consult an explicit allowlist constant `NON_COMMIT_PRODUCING_SKILLS = ["report-issue", ...]` rather than parsing for `git commit` literal references universally; `/report-issue` is added to that allowlist.
- AC-STE-229.11: Capability rows added to the static plain-language map at `/spec-write` § 7 (toolkit-wide map, inherited by every skill that emits a summary). Three new keys, in alphabetical order alongside existing entries:
  - `report_issue_default_applied` → `/report-issue gist auto-pushed (marker present in prompt body) — verify the gist contents before sharing`
  - `report_issue_declined` → `/report-issue gist declined — temp directory deleted, no upload`
  - `report_issue_redacted_payload` → `/report-issue scrubbed <N> secret-pattern match(es) before upload — see metadata.json for breakdown` (fires unconditionally on every successful push so operators see the scrub summary in console; `<N>` substitutes the total match count across all files and patterns)
- AC-STE-229.12: Smoke test at `plugins/dev-process-toolkit/tests/report-issue.smoke.test.ts` (Bun-mocked). The smoke MUST cover:
  1. gh-auth-fail refusal path (mock `gh auth status` to exit non-zero; assert NFR-10-shape refusal + non-zero exit + no temp directory created).
  2. Default-mode payload shape — assert `report.md` + `metadata.json` are produced; `transcript.jsonl` is absent.
  3. Full-mode payload shape — assert all three files produced; `transcript.jsonl` content matches the mocked JSONL.
  4. Redaction — fixture settings.json contains seven planted secrets (one per `SECRET_PATTERNS` entry); assert none appear verbatim in the captured upload payload.
  5. Preview-gate `n`-decline path — assert no `gh gist create` invocation, temp directory removed, exit non-zero.
  6. Closing-summary byte-floor (`wc -c` ≥ 100 on captured stdout).
  7. Marker-present auto-push — prompt body contains `<dpt:auto-approve>v1</dpt:auto-approve>`; assert `gh gist create` called with the exact canonical args + `report_issue_default_applied` row in stdout.
- AC-STE-229.13: `/brainstorm` accepts a gist URL as argument — when the first positional argument matches `^https://gist\.github\.com/[^/]+/[a-f0-9]{8,}/?$`, the skill fetches each file via `gh gist view <id-or-url> --raw -f <filename>` (or `gh gist view <id-or-url>` for the full payload), treats the combined text as the brainstorm seed, and proceeds with the existing Socratic flow. Cross-cutting edit to `plugins/dev-process-toolkit/skills/brainstorm/SKILL.md`. The `/report-issue` ↔ `/brainstorm` round-trip closes the design loop.

## Technical Design

**New skill module.** `plugins/dev-process-toolkit/skills/report-issue/SKILL.md` — single file, follows the `simplify` skill's shape (no helper subdirectory unless one is needed for the cross-cutting allowlist constant). Frontmatter:

```yaml
---
name: report-issue
description: <one-liner — TBD by author>
argument-hint: '[--full]'
---
```

**New helper modules.**

- `plugins/dev-process-toolkit/adapters/_shared/src/scrub_secrets.ts` — pure, deterministic, no I/O. Exports `scrubSecrets(text: string)` and `SECRET_PATTERNS: { key: string, pattern: RegExp, replacement?: string }[]` for test introspection. Patterns ordered longest-prefix-first to avoid Anthropic `sk-ant-…` keys being incorrectly flagged as OpenAI `sk-…` keys.
- `plugins/dev-process-toolkit/adapters/_shared/src/find_current_session.ts` — exports `findCurrentSession(cwd: string): string | null`. Algorithm: convert cwd to slug (replace `/` with `-`), `readdir` `~/.claude/projects/<slug>/`, filter `*.jsonl`, sort by `mtime` descending, return the first or null. Slug-dir-missing returns null cleanly (no throw).

**Composition with existing helpers.** None required — the skill stands alone. It does not call `requireCommittableBranch` (per AC-STE-229.10), does not call `Provider.sync` (no FR write), and does not call `buildFRFrontmatter` (no spec write).

**`gh` shell-out.** Only `gh auth status` (probe) and `gh gist create -s -d <title> <files...>` (publish). No `gh` API SDK. Preview/redaction logic is pure-text and runs before the publish shell-out. Exit code from `gh gist create` is captured; non-zero is surfaced as an NFR-10-shape refusal naming the underlying `gh` stderr (e.g., rate-limited / network-error / auth-revoked-mid-flight).

**Cross-cutting edit to `/brainstorm`.** `plugins/dev-process-toolkit/skills/brainstorm/SKILL.md` adds a new step before the existing "1. Clarify the Problem" — when the first positional argument matches the gist URL regex, fetch the gist payload via `gh gist view`, parse `report.md` for the narrative + curated context, parse `metadata.json` for severity + redaction summary, parse `transcript.jsonl` if present, then proceed into the existing Socratic flow with this material as the seed. Doc-conformance test asserts the gist-URL branch is documented and the regex matches the canonical GitHub gist URL shape.

**Probe update for STE-228.** The `commit_producing_skill_branch_gate` probe (introduced by STE-228 AC-STE-228.9) is updated to consult an explicit allowlist constant rather than parsing universally for `git commit` references. New constant `NON_COMMIT_PRODUCING_SKILLS = ["report-issue"]` is added to the probe source. SKILL.md prose for `/report-issue` documents the exemption + the allowlist's location.

**Temp directory hygiene.** Every successful or aborted run executes `rm -rf "$TMPDIR_VAR"` in a `trap … EXIT` (or the equivalent JS `finally` block, depending on the implementation language used by the surrounding runner). No payload artifacts persist on disk after the skill exits.

## Notes

- Audience: primarily plugin maintainer triage (toolkit user → maintainer return-channel); secondarily self-debug (dev → fresh `/brainstorm` session).
- "Secret" gist semantics: GitHub secret/unlisted gists are **not** truly private — anyone with the URL can read. The redaction pass is the privacy guarantee, not the gist's privacy flag. SKILL.md states this explicitly so the dev knows the risk surface before approving the preview gate.
- Per `project_no_users_yet`, no migration / opt-out path for the redaction defaults — they ship enabled with no flag to disable. If a toolkit user has a context where they want raw transcripts (e.g., debugging a bug *in* the redaction logic itself), they can call `gh gist create` manually with the same payload shape; the skill is convenience over capability.
- The mtime heuristic for current-session selection in `find_current_session.ts` is documented inline as "best-known approximation" — if Claude Code later exposes the session UUID via env var (e.g., `CLAUDE_SESSION_ID`), the helper SHOULD prefer that and fall back to mtime. The interface is stable across the change.

## Implementation notes

- The OpenAI `sk-…` regex in `SECRET_PATTERNS` was widened from `[A-Za-z0-9]{20,}` to `[A-Za-z0-9_-]{20,}` during code review to catch the v2 project-key shape `sk-proj-…` (whose hyphenated body would otherwise truncate the alphanumeric run at 4 chars and slip past unredacted). Anthropic still runs first per longest-prefix-first, so `sk-ant-…` keys are attributed to `anthropic_api_key`. Backfill: a future spec edit may want to enumerate v2 explicitly; the implementation contract is verified by the new "OpenAI v2 project key (sk-proj-…) is redacted" test.
- `find_current_session.ts` honors `process.env.CLAUDE_CONFIG_DIR` (e.g., operators running `~/.claude-st`); when unset, falls back to `<homedir()>/.claude`. The original AC text used `~/.claude/projects/` literally; SKILL.md prose and the helper's signature both surface the env-var path. Test coverage: three `defaultConfigDir` cases (env set / unset / empty).
- The three new capability rows (`report_issue_default_applied` / `_declined` / `_redacted_payload`) are co-housed in a single `report_issue_*` row in `/spec-write` § 7 (precedent: `branch_gate_*`). All three keys + their canonical prose appear verbatim in the row body; doc-conformance asserts each. Compression keeps `/spec-write/SKILL.md` under the NFR-1 ≤300-line cap.
- `/report-issue` SKILL.md step 7 (preview gate) is the canonical site of the `[y/N]` opt-in for transcript inclusion when `--full` was not on the CLI. Step 5 explicitly defers to step 7; the Rules section is consistent.
- Plan ACs 3 + 4 (live manual `/report-issue` and `/brainstorm <gist-url>` runs) ship `[~]` deferred — operator-observed verification at `/ship-milestone M62` time per plan task 8 ("operator-observed; capture the gist URL into the M62 ship-checklist").
