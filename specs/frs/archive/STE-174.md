# STE-174: /simplify SKILL.md no-op gate-skip behavior prose {#STE-174}

## Requirement

Smoke #6 (Jira, 2026-04-30, v2.1.0) F6 and smoke #7 (Linear, 2026-04-30, v2.1.0) F7 — same shape, both runs reproduced — surfaced that `/simplify` skips the gate re-run on no-op runs (correct behavior — re-running a clean gate against an unchanged tree is wasted tokens). But `SKILL.md` "Verify" step doesn't document the conditional, so a reader (and the LLM running the skill) has to reverse-engineer the rule from the prompt.

Result: when the `/simplify` log carries `Gate verification: Skipped — no changes were made…`, a future smoke-test driver auditing the chain has no canonical reference for whether the skip is expected. Both smoke runs flagged it as worth documenting.

Docs-only change to the "Verify" step.

Out of scope:
- Changing the actual no-op behavior — re-running a clean gate against an unchanged tree IS wasted tokens; the conditional is correct.
- Adding a new flag to force a gate re-run — no need.

## Acceptance Criteria

- AC-STE-174.1: `plugins/dev-process-toolkit/skills/simplify/SKILL.md` "Verify" step prose explicitly states: "If no refactors are warranted (the diff is already minimal), the gate re-run is skipped — re-running an unchanged tree against a clean gate is wasted tokens. Consult the most recent `/implement` or `/gate-check` log for the active gate stamp."
- AC-STE-174.2: `grep -nE 'no refactors are warranted|gate re-run is skipped' plugins/dev-process-toolkit/skills/simplify/SKILL.md` returns at least 2 matches.
- AC-STE-174.3: The "Verify" step's existing prose ("Re-run `bunx tsc --noEmit && bun test` after any changes") is preserved unchanged — the new sentence is a clarifying addendum, not a replacement.
- AC-STE-174.4: `bun test` green; `/gate-check` clean; `/spec-review M49` clean.

## Technical Design

Single-edit pass in `plugins/dev-process-toolkit/skills/simplify/SKILL.md`:

Locate the "Verify" step (currently a single line: `4. **Verify** — Run gate check to ensure nothing broke`). Append a parenthetical or follow-up sentence with the no-op conditional. The exact prose is at AC-STE-174.1.

No code, test, or fixture change.
