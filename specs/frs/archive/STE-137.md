# STE-137: Strip internal namespace from shipped templates + examples (F8-Tier-1 + G0) {#STE-137}

## Requirement

The plugin is meant to install into other projects (those projects use their own tracker — Linear with their own prefix, Jira with their own keys, or `mode: none`). Shipped templates and examples must therefore not leak this-repo's internal `STE-N` / `M<N>` namespace. Today they do, and the leak is most damaging in surfaces that get COPIED into user projects.

A user adopting the plugin into a Jira project gets a `CLAUDE.md` peppered with hardcoded `STE-42` / `Refs: M7` examples they have no context for. Per the user's stated principle (2026-04-27): "we need to ship 'clean' plugin." This FR is the Tier-1 cut that addresses every shipped artifact whose content lands directly in the user's filesystem.

Bundle G0 (broken `docs/patterns.md` link in `testing-spec.md.template:11` that copies into every user project) — same class of leak, fix in the same edit pass.

Adapter `.md` files, skill prose, and the `docs/` sweep (~700 references) are explicitly OUT of scope and land in M39 STE-143.

## Acceptance Criteria

Acceptable replacements: `<TICKET-ID>`, `LIN-42`, `M1`, `<short-ULID>`, or deletion of the parenthetical when the surrounding prose stands without it.
- AC-STE-137.2: `templates/spec-templates/requirements.md.template` — all 9 STE-N hits (STE-50/60/66×2/80/122×3/129) and 2 M-N hits replaced by generic placeholders. Same treatment.
- AC-STE-137.3: `templates/spec-templates/plan.md.template` — all 2 STE-N + 10 M-N hits replaced by generic placeholders.
- AC-STE-137.4: `templates/spec-templates/testing-spec.md.template:11` — the broken `docs/patterns.md` reference is fixed: either inline the layout-policy rule (it's two sentences) or reword to "See the plugin's documentation" without a path. The line MUST NOT carry a relative path that does not resolve in an adopting project.
- AC-STE-137.5: `templates/docs-architecture.md.template:12` — the single STE-N reference (inside an HTML comment) is removed or replaced with a placeholder. Even though it's in a comment, it's noise the LLM reads on every render.
- AC-STE-137.6: `examples/bun-typescript.md` — the 3 STE-N hits (`STE-128` ×2, `STE-106`) are replaced with generic stack-relevant placeholders (e.g., `LIN-128`) or deleted along with the surrounding example sentence if it doesn't survive removal.
- AC-STE-137.7: A new test (or extension of an existing template-shape test) asserts that the listed template + example files carry no `STE-\d+` or `M\d+` literal references. Regex-driven, runs as part of `bun test`.
- AC-STE-137.8: `bun test` passes after the edits. `/gate-check` clean (no probe regressions).
- AC-STE-137.9: `/setup` end-to-end run on a fresh project (smoke-test skill or manual reproduction) produces a generated `CLAUDE.md` that carries no `STE-\d+` or this-repo `M\d+` references.

## Technical Design

Mostly hand-edits across 5 files — 8 lines in `CLAUDE.md.template`, ~11 STE-N + 12 M-N in spec templates, 1 line in `docs-architecture.md.template`, 1 line in `testing-spec.md.template`, 3 in `examples/bun-typescript.md`. Total ~30-40 line changes.

The new test (AC-STE-137.7) lives at `plugins/dev-process-toolkit/tests/templates-no-internal-namespace.test.ts` (or analogous). For each listed template path:
1. Read the file.
2. Strip fenced code blocks (templates legitimately demonstrate frontmatter snippets that include `<tracker-id>` placeholders, but the placeholder shape is `<tracker-id>`, never `STE-N`).
3. Assert the residual content matches no `\b(STE|M)\d+\b` pattern.

Edge cases the test must allow:
- Literal `<tracker-id>` placeholder (the canonical replacement).
- Adapter-rendered placeholders like `STE-<N>` (literal `<N>` token, not a real digit).
- HTML comments may carry FR cross-reference IDs only if the implementation chooses to keep them — but for Tier-1 the cleaner default is to strip them.

For G0 (testing-spec template fix), the simplest replacement:

```
**Layout policy: `src/`-co-located** — tests live next to the source they exercise (toolkit default).
```

drops the path entirely. Two sentences inline if a longer rationale is needed.

## Testing

- **Unit / lint** — the new template-namespace test catches future drift.
- **Manual** — render a `/setup` against a clean fixture project; confirm the generated `CLAUDE.md` and spec files carry no `STE-N` / `M<N>` literals.
- **Gate check** — `/gate-check` runs clean post-edit.
