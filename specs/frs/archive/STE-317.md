# STE-317: Explicit-subcommand example permission blocks {#STE-317}

## Requirement

Rewrite the four stack-example permission blocks under `examples/<stack>/` to use explicit-subcommand allowlists matching `templates/permissions.json` exactly, replacing the obsolete glob-shaped `Bash(<cmd> *)` rules that the harness denies as self-modification. Same root cause STE-209 (M54) closed at the canonical `templates/permissions.json` surface; the four example files were never updated and still teach the failing shape to anyone copy-pasting from them.

## Technical Design

Pure example-file edits — four files, surgical replacements of the glob-shaped permission rules with explicit-subcommand allowlists from `templates/permissions.json`.

Source of truth: `templates/permissions.json` exports `_common` (cross-stack read-only commands), `_writes` (destructive ops, documented but not pre-emitted), and `stacks.<stack>` (per-stack invocations). Each example block emits `_common + stacks.<stack>` flattened in deterministic order (preserving the JSON file's order — STE-252 AC.1 contract).

Verification flow per file: read `templates/permissions.json` at edit time, project `_common + stacks.<stack>` to a markdown JSON code-fence, replace the existing block in the example doc. Per-stack subset:

- **bun-typescript**: `_common` + `stacks.bun` (`bun install`, `bun test`, `bun run`, `bun --version`, `bunx`).
- **typescript-node/gate-commands.md**: `_common` + `stacks.node` (the canonical key in `templates/permissions.json` — NOT `stacks.npm`). If absent at edit time, fall back to `_common` only + a one-line note. Verify against templates/permissions.json at edit time.
- **python/gate-commands.md**: `_common` + `stacks.python` (if defined) or per-tool entries from the file's existing list (pytest, mypy, ruff, etc.) rewritten as explicit subcommands.
- **flutter-dart/gate-commands.md**: `_common` + `stacks.flutter` (if defined) or per-tool entries.

If a stack key is missing from `templates/permissions.json`, AC-STE-317.3's escape hatch applies: use `_common` only with a one-line note. The FR does NOT add new stack keys to `templates/permissions.json` — that's a separate change (different release-target collision potential).
