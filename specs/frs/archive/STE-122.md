## Requirement

`templates/spec-templates/requirements.md.template` lines 32-34 (`AC-1.1` / `AC-1.2` / `AC-1.3`) and lines 78-79 (traceability matrix `AC-1.1` / `AC-1.2`) hardcode literal `AC-1.<N>` placeholders. The LLM-as-runtime running `/spec-write` fills these in verbatim instead of substituting the canonical `AC-<tracker-id>.<N>` (tracker mode) or `AC-<short-ULID>.<N>` (mode-none) form via `acPrefix(spec)` from `adapters/_shared/src/ac_prefix.ts` (STE-50).

Smoke-test 2026-04-27 reproduced this on v1.31.0: a freshly /spec-written tracker-mode FR landed with ACs `AC-1.1, AC-1.2, AC-1.3, AC-1.4` (where `1` matches the milestone `M1`) instead of `AC-STE-120.1` etc. Probe-15 (`guessed_tracker_id`) catches it at gate time → exit 1.

Same M31-class discretionary-check pattern as STE-117/118/119/121. Fix: tighten the template to surface the canonical placeholder shape AND add a post-write self-check inside `/spec-write` that runs probe-15 logic against the just-touched files before returning.

**Scanner module:**

```typescript
// adapters/_shared/src/guessed_tracker_id_scan.ts

const FENCED_FENCE = /^```/;
const INDENTED_FENCE = /^    /;
const GUESSED_AC = /\bAC-\d+\.\d+\b/g;

export interface GuessedTrackerIdViolation {
  file: string;
  line: number;
  column: number;
  match: string;
  message: string;
}

export function scanGuessedTrackerIdLiterals(files: string[]): GuessedTrackerIdViolation[] {
  const violations: GuessedTrackerIdViolation[] = [];
  for (const file of files) {
    const content = readFileSync(file, "utf-8");
    const lines = content.split("\n");
    let inFence = false;
    for (let i = 0; i < lines.length; i++) {
      const line = lines[i] ?? "";
      if (FENCED_FENCE.test(line)) {
        inFence = !inFence;
        continue;
      }
      if (inFence || INDENTED_FENCE.test(line)) continue;
      // Strip inline backticks (single-line code spans)
      const stripped = line.replace(/`[^`]*`/g, "");
      let m: RegExpExecArray | null;
      const re = new RegExp(GUESSED_AC.source, "g");
      while ((m = re.exec(stripped)) !== null) {
        violations.push({
          file,
          line: i + 1,
          column: m.index + 1,
          match: m[0]!,
          message: `guessed_tracker_id: literal ${m[0]} found at ${file}:${i + 1}:${m.index + 1} — substitute <tracker-id> via acPrefix(spec) and retry`,
        });
      }
    }
  }
  return violations;
}
```

**`/spec-write` post-write self-check call site:** after step 5 "Cross-check consistency" returns and before step 6 "Risk scan", invoke `scanGuessedTrackerIdLiterals` against `[FR file path, specs/requirements.md]`. On any violation, refuse with NFR-10 canonical shape; the file write has happened (atomic-rename pattern), so the user must re-run /spec-write or manually substitute.

**Why scanner not just `acPrefix(spec)` substitution alone:** the LLM-as-runtime can ignore SKILL.md prose telling it to call `acPrefix`. The scanner is the system-enforced backstop — even if substitution silently drops, the scan refuses the write before /spec-write returns clean.

## Notes

**Sequencing constraint:** Independent of STE-121 and STE-123. STE-121 and STE-122 both edit `skills/spec-write/SKILL.md` but at different sections (STE-121 → step 0b frontmatter, STE-122 → step 1+ AC-prefix); lockstep merge inside the milestone is straightforward.

**Probe-15 stays at gate-time:** the post-write scan is the in-band first line of defense; probe-15 stays at gate-check time as the safety net for any path that bypasses /spec-write (e.g., manual FR-file edits, copy-paste from old templates).

**Why scope to `requirements.md` and the FR file:** the smoke-test reproduced the failure in both surfaces — the FR's own `## Acceptance Criteria` section AND `specs/requirements.md`'s traceability matrix. Test files (`tests/greet.test.ts` in the smoke-test report) propagate from the FR ACs and are out of scope here — they're TDD outputs, not /spec-write outputs.

**Mode-none fixture:** the scanner has to distinguish `AC-1.1` (bad — milestone-numbered) from `AC-VDTAF4.1` (good — short-ULID-prefixed) from `AC-STE-122.1` (good — tracker-prefixed). The regex `\bAC-\d+\.\d+\b` excludes both good forms because both contain at least one letter in the prefix segment. Test fixture asserts this discrimination.

**Fenced-block edge cases:** nested fences (a `\`\`\`` inside a `\`\`\`\`\`\`\`\`` outer fence) are uncommon in markdown but theoretically possible. The scanner uses a simple toggle that flips on every `\`\`\`` line — pathological cases (mismatched fences) would either over- or under-strip. The scope is /spec-write outputs, which the toolkit controls; pathological inputs are out of scope.
