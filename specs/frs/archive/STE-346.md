# STE-346: Commit-boundary token-stats render — /implement Phase 4 + /ship-milestone rollup {#STE-346}

## Requirement

STE-345 renders the FR-level `## Token Stats` block during `/spec-write`. This FR closes the loop at the two remaining moments where a spec file is already being written and committed — so the numbers stay current through the build and roll up to the milestone — **without ever creating a standalone dirty-tree event**.

- `/implement` **Phase 4** owns the archive commit; that is where the FR's implement + `/tdd` tokens finally exist, so the FR block is re-rendered there, inside that commit.
- `/ship-milestone` owns the release commit; that is where a milestone-level rollup belongs, inside that commit.

The non-interference guarantee from STE-344/345 is preserved exactly: renders happen **only** inside commits the operator already approves, never as a loose working-tree mutation.

## Acceptance Criteria

- AC-STE-346.1: **`/implement` Phase 4 re-render.** During the Phase 4 milestone-archival step, each in-scope FR's `## Token Stats` block is re-rendered from the ledger (via STE-345's `upsertTokenStatsBlock`) so it now includes that FR's `implement` + `tdd` rows, and the change is staged into the **same archive commit** — no separate commit, no post-commit dirty tree. Vacuous when the ledger has no rows for the FR. {#AC-STE-346.1}
- AC-STE-346.2: **`/ship-milestone` rollup.** `/ship-milestone` renders a `## Token Stats` rollup into `specs/plan/M<N>.md` — one subtotal row per in-scope FR, a `(main-loop)` row (the `(no attributionSkill)` orchestrator tokens for the milestone's sessions), and a milestone `total` row, each broken down by model — and stages it into the **release commit**. Sentinel-fenced + idempotent, identical mechanics to the FR block. {#AC-STE-346.2}
- AC-STE-346.3: **Milestone "design/exploration" bucket.** `brainstorm` ledger rows on the milestone's branch lineage that were never claimed by any FR (STE-345 AC-STE-345.6) roll up into a dedicated `design/exploration` line in the milestone block, so exploratory brainstorm cost is visible rather than silently dropped. {#AC-STE-346.3}
- AC-STE-346.4: **Idempotent + non-interfering, proven by test.** Re-running either render replaces the fenced region in place (no duplicate block); a test asserts that neither render produces a working-tree mutation **outside** the archive/release commit it belongs to (the render is always part of an already-approved commit, never a standalone change). {#AC-STE-346.4}

## Technical Design

**Files touched:**
- `plugins/dev-process-toolkit/skills/implement/SKILL.md` — Phase 4 archival: after moving the FR markdown, call `upsertTokenStatsBlock` on each in-scope FR from the ledger and `git add` the change into the archive commit (before the commit is created). One-line ordering note: the token-stats re-render happens after `git mv` / flip and before the commit, so it lands in the same commit.
- `plugins/dev-process-toolkit/skills/ship-milestone/SKILL.md` — add a milestone-rollup render step to the release-commit assembly, reading the `## Release Files` flow; render the `M<N>.md` block from the ledger and stage it into the release commit.
- `plugins/dev-process-toolkit/adapters/_shared/src/token_stats_render.ts` (from STE-345) — extend with `renderMilestoneRollup(rows, { frOrder })` producing the per-FR subtotal + `(main-loop)` + `design/exploration` + `total` table.
- Milestone plan template — document the optional `## Token Stats` rollup section in `specs/plan/M<N>.md`.

**Ordering discipline.** Both renders are strictly *pre-commit-of-an-already-happening-commit*: `/implement` Phase 4 already stages + commits the archive move, and `/ship-milestone` already stages + commits the release bump. The token-stats write is folded into those existing staged sets — it introduces **no new commit and no post-commit dirty tree**, which is what AC-STE-346.4 pins down.

**`(main-loop)` attribution.** The `(no attributionSkill)` rows are real orchestrator tokens (often the largest bucket). At FR level they are ambiguous, so they surface only at milestone level as a single `(main-loop)` line — honest about "this is coordination overhead, not one skill's cost."

## Testing

- `/implement` Phase 4 prose-contract meta-test — re-render folded into the archive commit, vacuous-when-empty (AC-STE-346.1).
- `/ship-milestone` prose-contract meta-test — rollup folded into the release commit (AC-STE-346.2).
