## Requirement

During the v1.29.0 smoke test, `/setup` omitted the `## Docs` section from CLAUDE.md entirely when not asked about docs mode. `/docs` parses that section to determine which mode is enabled — absent section means /docs becomes a silent no-op. The skill should emit the section in its canonical shape with all flags `false` rather than dropping it, so /docs has something to read and the user has something to flip later.

This is a "silent feature drop" failure mode: the absence of the section is indistinguishable from "I never ran /setup" vs. "/setup ran but skipped the docs question". Always-emit makes the skill self-describing.

## Acceptance Criteria

- AC-STE-107.1: After /setup runs, CLAUDE.md contains a `## Docs` section regardless of whether the user was asked about docs mode.
- AC-STE-107.2: When the user declines docs mode (or /setup runs autonomously and the docs question is skipped), the section contains the canonical default flags all set to `false`. Per STE-68 the third key's canonical name is `changelog_ci_owned` (not `changelog_mode` — the AC text in this FR draft predated alignment with STE-68's Schema X; the implementation correctly uses `changelog_ci_owned` and is the source of truth):

  ```
  ## Docs

  user_facing_mode: false
  packages_mode: false
  changelog_ci_owned: false
  ```

- AC-STE-107.3: When the user accepts a docs mode, the corresponding flag is `true`. Flags are independent (any subset can be true).
- AC-STE-107.4: New `/gate-check` probe `claudemd-docs-section-present`: if CLAUDE.md exists, it MUST have a `## Docs` section. Probe lives at `plugins/dev-process-toolkit/adapters/_shared/src/claudemd_docs_section.ts` (flat path matches existing convention); integration test at `plugins/dev-process-toolkit/tests/gate-check-claudemd-docs-section.test.ts` per STE-82.
- AC-STE-107.5: `templates/CLAUDE.md.template` is updated to include the `## Docs` block with all-false defaults so the skill's own template advertises the canonical shape.
- AC-STE-107.6: Test fixture covers: docs mode never asked (all-false), docs mode asked and declined (all-false), docs mode asked and `packages_mode` accepted (one true), all three accepted.
- AC-STE-107.7: `bun test` green. CHANGELOG `### Fixed` entry cross-references this FR.

## Technical Design

**Template change.** `plugins/dev-process-toolkit/templates/CLAUDE.md.template` gains a `## Docs` section with the three flags. Skill prose in `skills/setup/SKILL.md` is updated so that the docs question is "which flags do you want true?" rather than "do you want a docs section?".

**Probe.** Trivial — read CLAUDE.md, check section exists. Sibling probe to existing `## Task Tracking` checks.

**Migration for existing CLAUDE.md files.** Out of scope — no users yet (per `project_no_users_yet.md` memory). Existing toolkit-internal CLAUDE.md gets the section added by hand during /implement.
