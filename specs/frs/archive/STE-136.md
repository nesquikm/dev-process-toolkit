# STE-136: Append ## Docs section to repo CLAUDE.md + add probe self-check {#STE-136}

## Requirement

`/Users/ns/workspace/dev-process-toolkit/CLAUDE.md` (the dogfood repo's root CLAUDE.md) is missing the `## Docs` section. `/gate-check` probe #18 (`claudemd-docs-section-present`, STE-107 AC-STE-107.4) fires at **error** severity when CLAUDE.md exists but lacks a non-commented `## Docs` heading. Direct probe invocation against the live repo returns one violation. `bun test` does not surface this because `tests/gate-check-claudemd-docs-section.test.ts` exercises the probe against synthetic `mkdtempSync()` fixtures only — no repo-self-check.

The plugin's own `templates/CLAUDE.md.template:194` carries the section correctly. The dogfood repo just never had the section added; it predates STE-107 (M29) by several milestones.

The fix is two parts: (1) append the section to `CLAUDE.md` so the next `/gate-check` run is clean; (2) backfill a repo-self-check row to the probe test so live-repo drift is visible at `bun test` time.

## Acceptance Criteria

- AC-STE-136.1: `/Users/ns/workspace/dev-process-toolkit/CLAUDE.md` carries a `## Docs` heading. The body declares the dogfood-correct flags: `user_facing_mode: false`, `packages_mode: false`, `changelog_ci_owned: false`. (The plugin doesn't have user-facing docs of its own — the docs ARE the plugin.)
- AC-STE-136.2: `/gate-check` probe #18 (`claudemd-docs-section-present`) returns zero violations against the live repo. Verified by direct probe invocation post-edit.
- AC-STE-136.3: `tests/gate-check-claudemd-docs-section.test.ts` carries a repo-self-check row that calls `runClaudeMdDocsSectionProbe(repoRoot)` against the live `CLAUDE.md` and asserts an empty `violations` array. The shape mirrors `tests/gate-check-root-hygiene.test.ts:262-308` (`describe("AC-STE-107.X — /gate-check runs clean on this repo's baseline", …)`).
- AC-STE-136.4: `bun test` is green (1071 → ≥1072 passing).
- AC-STE-136.5: The new self-check fails loudly if anyone removes the `## Docs` section from the repo's CLAUDE.md in the future.

## Technical Design

**File 1 edit** — `CLAUDE.md`: append a section at the bottom (after `## Task Tracking`) following the canonical shape:

```
## Docs

user_facing_mode: false
packages_mode: false
changelog_ci_owned: false
```

The flag values reflect this repo's posture:
- `user_facing_mode: false` — the plugin's docs ARE the artifact; there is no separate user-facing-docs build target for the dev-process-toolkit project itself.
- `packages_mode: false` — single package, no per-package docs sub-mode.
- `changelog_ci_owned: false` — `CHANGELOG.md` is hand-maintained per the Release Checklist; not auto-generated by CI.

**File 2 edit** — `tests/gate-check-claudemd-docs-section.test.ts`: append a `describe(..., () => { test(...) })` block at the bottom that:
1. Reads `repoRoot = join(import.meta.dir, "..", "..", "..")` (mirroring the root-hygiene shape).
2. Calls `runClaudeMdDocsSectionProbe(repoRoot)`.
3. Asserts `result.violations.length === 0`.

If the existing test file doesn't already import `repoRoot`-style helpers, adopt the import shape from `tests/gate-check-root-hygiene.test.ts:262-308` directly.

## Testing

- **Repo-self-check** added per AC-STE-136.3 — guards regressions.
- **Manual:** invoke probe directly via `bun -e "import { runClaudeMdDocsSectionProbe } …"` to confirm zero violations post-edit.
- **`bun test`** is the gate; passes after both file edits.

No new fixture is needed — the live repo's CLAUDE.md is the fixture.
