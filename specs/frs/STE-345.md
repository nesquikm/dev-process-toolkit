# STE-345: Render ## Token Stats into FR files + /spec-write integration {#STE-345}

## Requirement

Given the STE-344 ledger, this FR is the **render half** for FR-level stats: a machine-managed `## Token Stats` block, projected from the ledger into per-FR spec files, plus the convention test + gate-check tolerance that keep it from drifting, plus `/spec-write` wiring so the block is refreshed as a natural byproduct of writing an FR.

**Placement is chosen for non-interference.** The block is sentinel-fenced and sits **last** in the FR body (after `## Notes`), so it never interrupts a human-authored section and never trips a section-scoped `/gate-check` probe. This mirrors M91's `## Design References` precedent — an additive, machine-managed FR section guarded by its own convention test.

**Two lines, not a sum.** Per the 2026-07-01 design decision, `brainstorm` and `spec-write` render as **separate rows plus a subtotal** — collapsing them to one number would throw away the "which of the two is expensive" signal that motivated the feature. `context:fork` children are already folded into each parent row by STE-344's attribution, so the rows are naturally rolled-up.

## Acceptance Criteria

- AC-STE-345.1: **Block shape.** A `## Token Stats` section fenced by literal `<!-- token-stats:begin -->` / `<!-- token-stats:end -->` markers, placed **last** in the FR body (after `## Notes`). Between the fences is a per-skill × per-model markdown table (columns: skill, model, input, output, cache-read, cache-creation) with the FR's `brainstorm` and `spec-write` rows shown separately and a `subtotal` row; the render is byte-stable for a given ledger input. {#AC-STE-345.1}
- AC-STE-345.2: **Render helper, idempotent.** A pure `renderTokenStatsBlock(rows): string` and an `upsertTokenStatsBlock(frBody, rows): string` (in `adapters/_shared/src/token_stats_render.ts`) that inserts the fenced block after `## Notes` when absent, or **replaces the fenced region in place** when present — re-rendering never appends a second block and never mutates any other section. {#AC-STE-345.2}
- AC-STE-345.3: **Convention test.** `tests/token-stats-convention.test.ts` (modeled on `design-references-convention.test.ts`) asserts the fenced-block shape, the `after ## Notes` placement, the separate brainstorm/spec-write rows + subtotal, and the idempotent update-in-place (double render == single render). {#AC-STE-345.3}
- AC-STE-345.4: **gate-check tolerance.** Any `/gate-check` probe that enforces the FR body section set/order admits the optional `## Token Stats` section (additive, like `## Design References`) — no false-positive **GATE FAILED** on an FR carrying it. If no such probe constrains the set, an explicit note in the FR-section contract records that the section is allowed. {#AC-STE-345.4}
- AC-STE-345.5: **`/spec-write` integration.** After `/spec-write` writes or updates an FR file, it renders that FR's `## Token Stats` block from the ledger, **riding its own FR-file write** (no separate dirty-tree event, no new commit). The run emits a literal capability row (`token_stats_rendered`) in its Step 7 closing summary; vacuous (no row, no block) when the ledger is absent or has no rows for the FR. {#AC-STE-345.5}
- AC-STE-345.6: **Brainstorm→FR bridging.** The ledger's `session_id` + `git_branch` fields let `/spec-write` attach pre-FR `brainstorm` tokens to the FR it creates: same-session brainstorm rows are used directly; a detached brainstorm is bridged by claiming the most-recent **unclaimed** `dev-process-toolkit:brainstorm` rows on the current branch lineage. Rows never claimed by any FR are left for STE-346's milestone "design" bucket. A claimed row is marked (a `claimed_by` field or a sibling claim-ledger) so no FR double-counts it. {#AC-STE-345.6}

## Technical Design

**Files touched:**
- `plugins/dev-process-toolkit/adapters/_shared/src/token_stats_render.ts` — **new.** `renderTokenStatsBlock(rows)` (pure table render), `upsertTokenStatsBlock(frBody, rows)` (idempotent fenced-region replace), and a `filterRowsForFR(ledger, { branch, sessionLineage, brainstormClaim })` selector that resolves which ledger rows belong to the FR (AC-STE-345.6).
- `plugins/dev-process-toolkit/skills/spec-write/SKILL.md` — § 0b / § 7: after the FR-file write, render the block from the ledger; emit the `token_stats_rendered` capability row; add the row to the § 7 static capability map.
- `plugins/dev-process-toolkit/docs/layout-reference.md` — document the optional `## Token Stats` section (fenced, last, machine-managed) alongside `## Design References`.
- `plugins/dev-process-toolkit/skills/gate-check/SKILL.md` + the relevant probe — admit the optional section (AC-STE-345.4).

**Claim semantics (AC-STE-345.6).** A brainstorm row is "claimed" by exactly one FR. The simplest durable marker is a `claimed_by: <tracker-id>` field written back into the (git-ignored) ledger row when `/spec-write` folds it into an FR; `filterRowsForFR` skips already-claimed rows. Same-branch + most-recent-unclaimed is the heuristic; abandoned (never-claimed) rows fall through to STE-346.

**Render is a projection, never a source.** The `.dev-process/` ledger is the source of truth; the `## Token Stats` block is a rendered, regenerable projection. Editing the block by hand is pointless (the next render overwrites the fenced region) — documented so operators don't treat it as authored content.

## Testing

- `tests/token-stats-render.test.ts` — `renderTokenStatsBlock` byte-stability; `upsertTokenStatsBlock` insert-after-Notes, replace-in-place, double-render idempotency (AC-STE-345.1, .2).
- `tests/token-stats-convention.test.ts` — shape/placement/rows/subtotal + idempotency contract (AC-STE-345.3).
- `/spec-write` prose-contract meta-test — render-after-FR-write + `token_stats_rendered` capability-row emission + vacuous-when-empty (AC-STE-345.5).
- Bridging unit test — same-session direct use, detached most-recent-unclaimed claim, no double-count across two FRs (AC-STE-345.6).
- gate-check tolerance test — an FR with a `## Token Stats` block does not trip the section-set probe (AC-STE-345.4).
- Gate: `bun test` + `/dev-process-toolkit:gate-check` GREEN.
