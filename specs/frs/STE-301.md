# STE-301: Dependency docs catalog + topic-aware retrieval {#STE-301}

## Requirement

The toolkit already generates packages-mode docs via `/docs --full`. Consumer projects that depend on several private packages (typical for a mobile app with internal SDKs) currently have no way to fold those packages' docs into design context — `/brainstorm` and `/spec-write` only see the consumer's own spec set, so the LLM rediscovers conventions, re-invents APIs, and proposes designs that conflict with existing private-package surface area.

This FR adds: a git-tracked manifest of sibling packages at `specs/deps.yaml`, a `/deps` management skill (add/edit/delete/list/sync subcommands), a read-only `deps-researcher` Haiku subagent invoked via a paired forked skill `/deps-research`, and injection seeds in `/brainstorm` Step 1.5b + `/spec-write` § 0b step 2.5b. The subagent emits a `deps-research-result` fenced block — closed-schema clone of STE-230's `spec-research-result` shape — capped at 25 lines with three canonical sections (`## Relevant Packages`, `## API Surface Highlights`, `## Reusable Patterns`) plus an optional `## Missing deps` subsection. Default checkout convention is sibling-directory (`../<name>`), git-tracked manifest with relative paths only.

This is a strict architectural extension of STE-230: `deps-researcher` is structurally identical to `spec-researcher` (Haiku, read-only tools, fixed-shape block, NON_COMMIT_PRODUCING_SKILLS allowlist, fork-only invocation), retargeted from the FR archive to the dep manifest.

## Technical Design

### Architecture

```
specs/deps.yaml
   │
   ▼
adapters/_shared/src/deps_manifest.ts   ← parser + helpers + sibling-path resolver
   │
   ├─► skills/deps/SKILL.md             ← user-invocable management surface
   │     ├─ add  → Socratic prompts → git clone → validate Diátaxis → writeManifest
   │     ├─ list → tabular stdout
   │     ├─ edit → Socratic prompts → mutate entry → writeManifest
   │     ├─ delete → confirmation → removeEntry → writeManifest
   │     └─ sync → walk entries → clone-prompt missing-with-origin
   │
   └─► skills/deps-research/SKILL.md    ← forked, read-only retrieval (Haiku)
         └─ agents/deps-researcher.md
              └─ emits `deps-research-result` fenced block
                   │
                   └─► injected by:
                         brainstorm/SKILL.md      § Step 1.5b  (after spec-research)
                         spec-write/SKILL.md      § 0b step 2.5b
```

### Entry schema

```yaml
# specs/deps.yaml
deps:
  - name: my-internal-sdk
    path: ../my-internal-sdk
    origin: git@github.com:acme/my-internal-sdk.git
    ref: main
    kind: toolkit-docs
  - name: shared-models
    path: ../shared-models
    origin: https://github.com/acme/shared-models.git
    kind: toolkit-docs
```

- `name`: unique within manifest; matches `[a-z][a-z0-9-]*`.
- `path`: relative, MUST start with `../` (sibling-only). Resolves against the consumer repo root via `resolveSiblingPath(repoRoot, entry)`.
- `origin`: optional URL or local-path; required if the operator wants `/deps sync` to clone the missing checkout.
- `ref`: optional git ref. M78 records it for documentation; no auto-pull / staleness check.
- `kind`: `"toolkit-docs"` only in M78. Slot for future formats.

### Retrieval contract

The `deps-research-result` fenced block clones STE-230's `spec-research-result` shape — same banner-line + canonical-sections + overflow-marker + line-cap mechanics. Three sections in fixed order:

1. `## Relevant Packages` — entries whose docs matched the topic (1-line summary each, name + purpose).
2. `## API Surface Highlights` — verbatim signature snippets lifted from each matched package's `docs/reference/` (per STE-72: LLM never paraphrases signatures; ground-truth-only).
3. `## Reusable Patterns` — decisions / patterns lifted from each matched package's `docs/explanation/`.

Optional fourth `## Missing deps` subsection naming entries whose local path is absent. The read-only fork cannot clone — the operator runs `/deps sync` to remediate.

### Missing-checkout handling

`deps-researcher` only has Read/Grep/Glob. When a manifest entry's resolved sibling path does not exist on disk, it silently skips that entry (no error, no fork failure) and lists the name in the `## Missing deps` subsection. The 25-line cap covers banner + sections + optional `## Missing deps` + fence; truncation marker applies symmetrically.

### Out of scope (deferred)

- Non-toolkit doc formats (third-party libs, raw READMEs, vendor docs) — `kind: "toolkit-docs"` only in M78.
- Auto-pull / staleness detection on sibling checkouts — `/deps sync` is clone-only; operator-driven updates.
- Per-dep auth handling — relies on operator's existing `git` / `gh` credentials.

## Testing

Framework: `bun test`. Test files mirror existing patterns in `plugins/dev-process-toolkit/tests/`.

**Manifest helper tests** (`tests/deps-manifest.test.ts`):

- `readManifest` on missing file → returns empty manifest (no throw).
- `readManifest` on malformed YAML → throws `DepsManifestShapeError`.
- `readManifest` on valid manifest → parses entries with all fields.
- `addEntry` on collision → throws shape error.
- `removeEntry` on missing name → throws shape error.
- `resolveSiblingPath` returns absolute path; rejects non-`../` paths.
- Entry validator rejects `kind` other than `toolkit-docs`.

**Output-shape parser tests** (`tests/deps-research-result.test.ts`):

- Valid 3-section block → parses cleanly.
- Valid 3-section + optional `## Missing deps` → parses cleanly.
- Block missing banner → format violation naming "banner".
- Block over 25 lines → format violation naming "line cap".
- Wrong section order → format violation naming "section order".
- Wrong section name → format violation naming the offending name.
- Two fenced blocks → format violation (exactly-one rule).

**Integration tests** (`tests/deps-research-injection.test.ts`):

- Brainstorm Step 1.5b with absent `specs/deps.yaml` → no fork invocation (vacuous).
- Brainstorm Step 1.5b with empty manifest → no fork (vacuous).
- Brainstorm Step 1.5b with non-empty manifest → fork invoked, block parsed + injected.
- Spec-write § 0b step 2.5b: symmetric to brainstorm tests.
- Shape-violation block → dropped silently, parent skill continues; `deps_research_shape_violation` capability emitted.
- `--no-tech` flag → injection skipped (no Step 2 to seed).

**Gate-check probe tests** (`tests/gate-check-deps-researcher-invariants.test.ts`):

- Conforming `agents/deps-researcher.md` + `skills/deps-research/SKILL.md` → probe passes.
- Subagent frontmatter missing `tools: Read, Grep, Glob` → probe fails, names the violation.
- Subagent frontmatter has `Write` in tools → probe fails.
- Child skill frontmatter missing `context: fork` → probe fails.
- Child skill includes `Agent` in allowed-tools → probe fails.

**Out of test scope (manual smoke / deferred):**

- End-to-end usage against a real Flutter / TypeScript consumer repo with 2-3 private sibling packages — covered by the M78 manual smoke pass. Ship `[~]` per `feedback_smoke_post_ship_retroactive.md` if the smoke is deferred to a follow-up retroactive-validation milestone.

## Notes

**Design heritage.**

- STE-230 (archived) — `spec-research` forked subagent for `/brainstorm` + `/spec-write`. Direct architectural twin: read-only Haiku fork, ≤ 25-line fenced block, NON_COMMIT_PRODUCING_SKILLS allowlist, identical injection points. `deps-research` is structurally identical, retargeted from the FR archive to `specs/deps.yaml`.
- STE-72 (archived) — Mechanical API signature extraction. Ground-truth-only invariant: the LLM never paraphrases signatures; `## API Surface Highlights` lifts verbatim from each package's `docs/reference/`.
- STE-69 / STE-70 (archived) — `/docs` Diátaxis tree contract. `deps-researcher` depends on the canonical `docs/` shape that `/docs --full` emits in each consumed package — `kind: "toolkit-docs"` is exactly that contract.
- STE-229 (archived) — `/report-issue` forked skill; sibling precedent for curated-context read-side fork on the NON_COMMIT_PRODUCING_SKILLS allowlist.

**What this FR explicitly does NOT change:**

- `/spec-research` (STE-230) is untouched. Both forks co-exist and are invoked in sequence in `/brainstorm` Step 1.5 → 1.5b and `/spec-write` § 0b step 2.5 → 2.5b.
- `/docs` (STE-70) is untouched. M78 reads `/docs --full` output of sibling packages; no changes to the generator.
- `packages_mode` (STE-69) is untouched. M78 is consumer-side only; producer-side `/docs --full` behavior is unchanged.

**Open issues for /implement to decide:**

- Whether `/deps add` should auto-detect existing sibling checkouts (e.g., if `../my-sdk` is already a git repo with a matching origin, skip the clone and just record the entry). Probable answer: yes, with a confirmation prompt — saves manual setup for operators who already cloned manually. Decide in implementation.
- Whether `## API Surface Highlights` should support a Diátaxis-aware extraction layer (e.g., parse `docs/reference/` as structured signature output rather than raw markdown). Probable answer: M78 reads raw markdown (simpler); structured extraction is a future enhancement.
- Whether `/deps sync` should warn when a present checkout's HEAD does not match the entry's `ref`. Probable answer: yes, surface as a warning row in the closing summary but do not auto-fix.
