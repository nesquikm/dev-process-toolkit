# STE-309: Remove Flutter/Dart empty_test.dart placeholder from /setup {#STE-309}

## Requirement

`/setup`'s Flutter/Dart stack-specific guidance (`plugins/dev-process-toolkit/skills/setup/SKILL.md` line 99) currently reads `flutter create ., add bloc_test + mocktail, verify test/ dir exists`. Past LLM runs interpreted "verify test/ dir exists" defensively — writing `test/empty_test.dart` with body `void main() { test('placeholder', () => expect(1, 1)); }` to guarantee the gate (`flutter analyze && flutter test`) reports green before any real tests exist.

Evidence the placeholder is cruft on current SDKs:

- Flutter 3.41.6 `flutter test` exits 0 against an empty `test/` directory (warning only, no failure).
- `flutter create .` already writes `test/widget_test.dart`, so `test/` is never empty in fresh bootstraps.
- Five sibling StellarLab packages (reader, stellar-app-data-analytics, stellar-app-ui-common, stellar-app-core-theme, stellar-app-data-oauth) carried the byte-identical placeholder from their `chore: bootstrap dev-process-toolkit` commits; each just removed it in a one-line deletion PR. Reviewer @ArkhipenkaPiotr flagged it on the reader PR ("Why?").

Bun's `.placeholder.test.ts` is **load-bearing** (STE-113 M29) — `bun test` exits 1 on zero matches with no `--passWithNoTests` flag. That placeholder stays. Other stacks (Vitest, Jest, pytest, `go test`) default-pass on empty test dirs and need no placeholder.

Tighten the Flutter/Dart bullet so the next `/setup` run cannot rationalize writing a placeholder, and ship a regression test that catches a future drift back to placeholder-emitting prose.

## Acceptance Criteria

- AC-STE-309.1: `plugins/dev-process-toolkit/skills/setup/SKILL.md`'s Flutter/Dart stack-specific guidance bullet is rewritten to: (a) state explicitly that `flutter create .` writes `test/widget_test.dart` automatically; (b) include the literal directive "do NOT add an extra placeholder file (e.g. `test/empty_test.dart`)"; (c) note that `flutter test` exits 0 against an empty `test/` directory on current Flutter SDKs.
- AC-STE-309.2: A new bun test `plugins/dev-process-toolkit/tests/setup-skill-flutter-no-placeholder.test.ts` reads `skills/setup/SKILL.md`, locates the Flutter/Dart stack-specific guidance bullet, and asserts: (a) the bullet does NOT contain any of the substrings `empty_test`, `expect(1, 1)`, or `expect(1,1)`; (b) the bullet does NOT contain any `test/<name>.dart` reference other than `test/widget_test.dart`; (c) the bullet positively contains both "do NOT add" and "placeholder" tokens (case-insensitive). The test is wired into the existing `bun test plugins/dev-process-toolkit/tests/` run and passes on first invocation against the post-edit SKILL.md.
- AC-STE-309.3: The v2.27.0 CHANGELOG entry (under the M80 section, `### Fixed` or `### Changed` per Keep-a-Changelog category) gains a line referencing this FR, stating the `/setup` behavior change and the cleanup callout: "Downstream Flutter/Dart packages carrying `test/empty_test.dart` from older `/setup` runs should delete the file manually — `flutter test` exits 0 on empty `test/` on current SDKs." No automated codemod, no `/gate-check` probe, no version-sniffing logic ships with this FR.

## Technical Design

**SKILL.md edit** — single-line rewrite at the Flutter/Dart bullet in `### 2. Scaffold new projects` § 2b § Stack-specific guidance:

Before:

```
- **Flutter/Dart:** `flutter create .`, add bloc_test + mocktail, verify test/ dir exists
```

After:

```
- **Flutter/Dart:** `flutter create .` (writes `test/widget_test.dart` automatically), add bloc_test + mocktail. Do NOT add an extra placeholder file (e.g. `test/empty_test.dart`) — `flutter test` exits 0 against an empty `test/` directory on current Flutter SDKs, and `widget_test.dart` from `flutter create` keeps `test/` non-empty in the canonical bootstrap path.
```

**Regression test shape** — `plugins/dev-process-toolkit/tests/setup-skill-flutter-no-placeholder.test.ts`:

```ts
import { test, expect } from "bun:test";
import { readFileSync } from "node:fs";
import { join } from "node:path";

const SKILL_PATH = join(
  import.meta.dir,
  "..",
  "skills",
  "setup",
  "SKILL.md",
);
const FLUTTER_BULLET_RE = /^\s*-\s+\*\*Flutter\/Dart:\*\*[^\n]+(\n  [^\n]+)*/m;

test("SKILL.md Flutter/Dart bullet emits no placeholder-test instructions", () => {
  const md = readFileSync(SKILL_PATH, "utf8");
  const m = md.match(FLUTTER_BULLET_RE);
  expect(m, "Flutter/Dart bullet not found in SKILL.md").not.toBeNull();
  const bullet = m![0];
  expect(bullet).not.toMatch(/empty_test/);
  expect(bullet).not.toMatch(/expect\(\s*1\s*,\s*1\s*\)/);
  const dartFiles = bullet.match(/test\/[A-Za-z0-9_.-]+\.dart/g) ?? [];
  for (const path of dartFiles) {
    expect(path).toBe("test/widget_test.dart");
  }
  expect(bullet.toLowerCase()).toContain("do not add");
  expect(bullet.toLowerCase()).toContain("placeholder");
});
```

**Out of scope** (explicit, per brainstorm Q4):

- Bun's `src/.placeholder.test.ts` stays (STE-113 — load-bearing, `bun test` exits 1 on zero matches).
- No version-sniffing / conditional emission logic (brainstorm clarifier 2: "drop unconditionally").
- No codemod, no `/dev-process-toolkit:cleanup-placeholders` skill, no new `/gate-check` probe for downstream cleanup (brainstorm clarifier 3: "leave to drift, doc-only callout").
- No audit of other speculative `/setup` scaffolding (brainstorm clarifier 4: "just the placeholder, narrow FR").
